# 仓库执行入口

## 仓库定位

本仓库是面向计算机科学研究的本地工作区，不是 Codex Skill，也不依赖隐藏提示词、
外部注册格式或自动状态机。Agent 通过读取工作区中的真实文档判断任务状态，并按用户
确认推进学习、研究、写作和投稿准备。

## 规范分工

- `AGENTS.md`：定义仓库概念、启动顺序和最高级行为边界。
- `WORKFLOW.md`：定义生命周期阶段、阶段状态、按需能力、初始化、路由和转移。
- `指导/使用说明.md`：定义阶段与能力共同遵守的输入、协作、证据、停止和回写方法。
- `指导/` 中的其他文件：定义某个阶段或能力的专门方法、特有输入、产物和完成条件。

发生冲突时，先遵守本文件，再遵守 `WORKFLOW.md`，最后执行通用协议和专项指导。
专项指导可以增加约束，但不能绕过上层证据边界和用户确认。

## 仓库概念

- **仓库根目录**：包含本文件、`WORKFLOW.md`、`指导/`、`workspace/`、`labs/` 和可选
  CLI。
- **工作区**：`workspace/`，存放研究文档和跨阶段交接材料。
- **任务目录**：扁平的 `workspace/<中文任务名>/`。一个目录承载一个可独立理解的
  学习任务、研究项目或论文任务，不增加固定类型中间层。
- **生命周期阶段**：会改变任务主线状态的工作，编号为 L1-L2、R1-R8。
- **按需能力**：可在任意阶段或独立调用的专项操作。调用结束后返回原任务，不改变
  生命周期阶段，也不能把未执行阶段标为完成。
- **阶段文档**：记录目标、证据、判断、用户决定和交接结果的 Markdown 文件。
- **任务内产物目录**：按需创建的 `论文资料/`、`实验记录/`、图表或稿件目录。
- **实验仓库容器**：根目录 `labs/`，只放用户手动准备的独立实验代码仓库，且被本
  研究仓库忽略。

人可见目录、研究文档和论文处理输出优先使用中文名称。同名任务存在时，让用户选择
有意义的中文限定词，不自动追加编号或日期。文档随实际进展创建，不预建空文件。

实验仓库必须由用户手动准备在 `labs/<实际仓库目录>/`。Agent 不得替用户创建
`labs/`、执行 clone、初始化 Git 仓库或生成仓库名称。进入实验仓库后，先读取该仓库
自己的 `AGENTS.md` 和工程规范，并把它视为独立 Git 工作区。本仓库只约束研究目标、
Claim、证据和交接记录，不规定实验仓库内部布局。

## Agent 启动顺序

1. 读取 `WORKFLOW.md`，理解阶段、能力、状态、初始化和路由规则。
2. 确认用户指向新任务、已有任务目录，还是一次性独立能力。
3. 对已有任务，通读 `workspace/<中文任务名>/` 下现有 Markdown；按需检查与当前目标
   直接相关的材料、稿件和实验记录。
4. 根据用户目标、有效产物、已跳过决定和未决确认，推导任务结果、当前阶段及状态。
5. 读取 `指导/使用说明.md`，再读取当前阶段或能力的专项指导。
6. 实验工作还要核验任务文档记录的 `labs/<仓库目录>/`，并读取该仓库规范。
7. 输入和确认条件满足后执行；结束时回写真实产物、决定、限制和返回位置。

## 最高级边界

- **中文优先**：正式英文论文形成前，研究构思、叙事、实验设计、结果分析和中间文档
  使用中文。
- **人类决策优先**：Agent 负责检索、规整、反驳、结构化和执行；方向、Story、实验
  范围、结果口径、阶段跳过和重大投稿决定由用户确认。
- **证据优先**：外部事实、相关工作、实验结论和投稿政策必须可追溯；不确定内容标为
  未核验、推断或待验证。
- **不伪造**：不得伪造论文、作者、机构、引用、原文位置、代码能力、实验数据或结果。
- **不粉饰**：不得通过换名、换壳、修辞或隐藏撞车工作、反例、负结果强化 Idea。
- **不绕过访问控制**：不得绕过登录、验证码、付费墙、许可证、数据授权或其他限制。
