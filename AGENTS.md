# 学习工作台使用入口

当前仓库只作为论文学习工作台使用。学习项目、论文记录、筛选状态和 PDF 资源均以服务器为唯一权威；不要创建并行的 Markdown 清单，不要直接读写 SQLite 或 `backend/data/`。

## 每次开始

1. 阅读 `API.md`，接口契约以服务器的 `/openapi.json` 为准。
2. 请求 `GET /api/health` 确认服务可用。
3. 请求 `GET /api/workspace`，一次读取全部项目、论文、筛选状态、PDF 资源和本地工具状态。
4. 根据用户当前目标选择已有项目；只有范围确实不属于现有项目时才创建新项目。

不要读取 `ROUTER.md`。它是保留的旧路由草案，当前学习工作台不使用。

## 文献检索与收录

检索前阅读 `指导/学习/文献检索与收录.md`。遵守其中的检索覆盖、来源顺序、核验和规模要求。

- 每篇候选必须补全 `GET /api/schema/paper` 返回的当前契约；字段语义参考 `指导/参考/论文记录字段.md`。
- 先根据工作区快照中的稳定键和论文身份查重，再批量提交目标项目。
- 新候选默认使用 `discovered`。这表示等待用户判断，不等于 Agent 已替用户收录。
- 用户明确决定后，才把状态更新为 `included`、`excluded` 或 `archived`；“不留”更新为 `excluded`，不删除记录。
- 临时 JSONL 只能放在被忽略的 `tmp/`，仅用于一次请求组装，请求结束即删除。

## 论文资源获取

获取前阅读 `指导/学习/论文资源获取.md`。

- 论文必须已经存在于服务器；没有记录时先完成检索与收录。
- 获取合法、完整且身份一致的 PDF 后，上传为该论文的 `original` 资源。
- 上传前检查已有资源；除非用户明确要求替换，否则复用正确原文。
- 下载只使用系统临时目录，上传后清理；不要在仓库里建立论文目录或保存长期副本。

## 页面负责的动作

- PDF2zh 和 TeX 在首页检查并安装，固定放在仓库 `.tools/` 下。
- 完整数据备份、下载和恢复在首页管理；恢复只接受服务器自己保存的当前格式快照。
- 用户在论文页面启动翻译并下载原文、中文或双语 PDF。
- 除非用户明确要求，Agent 不替用户安装工具或启动翻译。

## 完成任务

执行写操作后重新请求相关资源；任务结束前再次读取 `GET /api/workspace`，确认服务器中的真实状态与报告一致。只报告已核验、已提交或已上传的结果，失败和缺口必须明确说明。
