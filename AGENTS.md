# 仓库执行入口

## 仓库定位

本仓库是面向计算机科学研究的本地工作区, 不是 Codex Skill, 也不依赖隐藏提示词,
外部注册格式或自动状态机, Agent 通过读取工作区中的真实文档判断任务状态, 并按用户确认推进学习, 研究, 写作和投稿准备

## 规范分工

- `AGENTS.md`: 定义仓库概念, 启动顺序和最高级行为边界
- `WORKFLOW.md`: 定义生命周期阶段, 阶段状态, 按需能力, 初始化, 路由和转移
- `指导/使用说明.md`: 定义阶段与能力共同遵守的输入, 协作, 证据, 停止和回写方法
- `指导/` 中的其他文件: 定义某个阶段或能力的专门方法, 特有输入, 产物和完成条件
- `开发指南.md`: 面向维护者解释设计理由和修改入口, 不是高于上述活动规范的新规则层

发生冲突时, 先遵守本文件, 再遵守 `WORKFLOW.md`, 最后执行通用协议和专项指导
专项指导可以增加约束, 但不能绕过上层证据边界和用户确认

## 仓库概念

- **仓库根目录**: 包含本文件, `WORKFLOW.md`, `指导/`, `scripts/`, `workspace/`
  和 `labs/`
- **工作区**: `workspace/`, 其直接子目录是中文领域目录或中文研究项目目录, 不增加
  `学习/`, `研究/` 等固定类型中间层
- **领域目录**: `workspace/<中文领域名>/`, 保存学习规划, 领域知识地图, 资源记录,
  以及该领域下按论文组织的档案
- **论文目录**: `workspace/<中文领域名>/<论文中文短标题>/`, 一篇论文的原始 PDF,
  中文 PDF, 双语对照 PDF, 元数据, 处理记录, 阅读笔记和评议集中在同一目录, 不再
  增加 `论文资料/` 中间层
- **研究项目目录**: `workspace/<中文研究项目名>/`, 保存 Idea, Story, Claim, 证据,
  实验交接和稿件, 项目引用领域论文档案, 不重复复制整套论文材料
- **生命周期阶段**: 会改变任务主线状态的工作, 编号为 L1-L2, R1-R8
- **按需能力**: 可在任意阶段或独立调用的专项操作, 调用结束后返回原任务, 不改变
  生命周期阶段, 也不能把未执行阶段标为完成
- **控制文档**: 记录目标, 状态, 证据索引, 判断, 用户决定和交接结果的 Markdown,
  例如 `论文信息.md`, `论文处理记录.md`, 阶段文档和研究记录
- **内容产物**: 论文正文, 译文, 稿件, PDF, Tex, 图片和大体量原始结果, 它们可以
  被工具处理, 但默认不进入 Agent 的模型上下文
- **外部工具缓存**: PDF2zh 等工具可在用户目录维护自身缓存, 但缓存不是论文档案,
  研究证据或 Agent 阅读材料, 不复制进论文目录
- **项目内产物目录**: 研究项目中按需创建的 `实验记录/`, 图表或稿件目录
- **实验仓库容器**: 根目录 `labs/`, 只放用户手动准备的独立实验代码仓库, 且被本
  研究仓库忽略

人可见目录, 研究文档和论文处理输出优先使用中文名称, 同名领域, 项目或论文存在时,
让用户选择有意义的中文限定词, 不自动追加不透明编号, 文档随实际进展创建, 不预建空文件

实验仓库必须由用户手动准备在 `labs/<实际仓库目录>/`, Agent 不得替用户创建
`labs/`, 执行 clone, 初始化 Git 仓库或生成仓库名称, 进入实验仓库后, 先读取该仓库自己的 `AGENTS.md` 和工程规范, 并把它视为独立 Git 工作区, 本仓库只约束研究目标,
Claim, 证据和交接记录, 不规定实验仓库内部布局

## Agent 启动顺序

启动读取和路由完成前, 不得预设或向用户宣布将使用的具体工具, Skill, 执行流程和输出目录, 首次进度说明只应说明正在读取仓库规范, 检查工作区和识别任务类型, 不能把尚未核验的方案说成既定安排

1. 读取 `WORKFLOW.md`, 理解阶段, 能力, 状态, 初始化和路由规则
2. 读取 `指导/使用说明.md`, 确认输入, 确认, 阻塞和落盘协议
3. 确认用户指向新领域, 已有领域, 研究项目, 还是一次性独立能力
4. 对已有目标, 先列出文件并通读相关控制文档, 论文 PDF, 稿件和大体量结果只在当前
   任务确实需要语义判断时按最小范围读取
5. 根据用户目标, 有效产物, 已跳过决定和未决确认, 推导任务结果, 当前阶段或能力
6. 读取当前阶段或能力的专项指导, 完成这些读取后, 才形成并说明具体执行方案
7. 孤立论文需要归档时, 先检查已有领域目录, 用户未指定归属时, 提出候选并询问应加入
   哪个已有领域或创建哪个中文领域目录, 不得自行决定
8. 实验工作还要核验任务文档记录的 `labs/<仓库目录>/`, 并读取该仓库规范
9. 输入, 路径和确认条件满足后执行, 结束时回写真实产物, 决定, 限制和返回位置

## 最高级边界

- **中文优先**: 正式英文论文形成前, 研究构思, 叙事, 实验设计, 结果分析和中间文档
  使用中文
- **人类决策优先**: Agent 负责检索, 规整, 反驳, 结构化和执行, 方向, Story, 实验
  范围, 结果口径, 阶段跳过和重大投稿决定由用户确认
- **证据优先**: 外部事实, 相关工作, 实验结论和投稿政策必须可追溯, 不确定内容标为
  未核验, 推断或待验证
- **不伪造**: 不得伪造论文, 作者, 机构, 引用, 原文位置, 代码能力, 实验数据或结果
- **不粉饰**: 不得通过换名, 换壳, 修辞或隐藏撞车工作, 反例, 负结果强化 Idea
- **不绕过访问控制**: 不得绕过登录, 验证码, 付费墙, 许可证, 数据授权或其他限制
- **模型上下文隔离**: 工具读取全文不等于 Agent 阅读全文, 归档, PDF 翻译, 编译,
  哈希和文件检查等机械任务, 默认由外部工具或脚本在模型上下文之外处理
  命令输出只返回状态, 计数, 哈希和错误摘要, 不回显正文, 只有摘要, 问答, 证据核验,
  审稿, 写作或用户明确要求的内容检查才读取正文, 并限制到完成任务所需的最小片段
- **论文处理自动化优先**: 用户确认输入, 目录和外部传输后, Agent 应调用
  `scripts/translate-pdf.ps1` 并等待 PDF2zh 的机器结果, 不逐页介入, 不得主动打开,
  抽查或修正原文, 中文稿和双语稿, 也不把视觉或语义抽查加入机械处理, 管线失败时
  报告错误并停止, 不得改写 PDF2zh 源码或让 Agent 手工接管正文处理
