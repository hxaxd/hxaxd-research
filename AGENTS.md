# 仓库开发指南

本仓库是单用户本地文献工作台。产品代码只位于 `backend/` 和 `frontend/`；`.tools/`、运行数据、快照和构建产物不得提交。

## 架构边界

- 后端按领域纵向组织。目录、筛选、文件库、任务、智能体与外部集成各自拥有模型和命令；只在组合根接线。
- 数据库和文件系统只能由后端领域服务访问。前端、智能体和集成适配器必须调用类型化命令，不得直接操作存储。
- 候选是独立暂存实体。智能体可以创建候选，但 `included`、`excluded`、`archived` 只能由明确的用户命令产生。
- 智能体的任务上下文由后端组装为用户消息；权限、幂等、审计和状态约束必须由代码保证，不能依赖提示词。
- Zotero 同步由确定性代码规划和执行；智能体只能提出冲突处理建议。
- PDF、TeX 和派生文件使用内容哈希核验。多输出操作必须原子登记，失败不得留下半完成记录。

## 数据变更

- 不为旧接口或旧前端保留兼容层，但必须保留现有文献、项目关系、附件、哈希和筛选决定。
- 结构迁移先在影子数据库完成并验证，再原子切换；不要原地破坏活动数据库。
- 不要手工读取或修改 `backend/data/`。数据核验通过服务公开的完整性接口和快照完成。

## 开发命令

- 后端：在 `backend/` 运行 `uv run pytest` 与 `uv run ruff check .`。
- 前端：在 `frontend/` 运行 `npm test`、`npm run typecheck`、`npm run lint` 和 `npm run build`。
- 后端 OpenAPI 是字段契约的唯一权威。前端的手写类型只允许集中在
  `frontend/src/shared/api/contracts.ts` 这一处，并由契约测试防止漂移；不要维护第二份说明文档。

## 完成标准

新行为需要覆盖领域测试和接口测试。涉及数据、任务、工具或智能体时，还要验证失败恢复、取消、幂等、审计事件和重启后的状态。
