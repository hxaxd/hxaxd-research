# 仓库执行入口

## 仓库定位

本仓库是面向计算机科学研究的本地工作区, 不是 Skill, 也不依赖隐藏提示词, 外部注册格式或自动状态机, Agent 通过读取工作区中的真实文档判断任务状态, 并按用户确认推进学习, 研究, 写作和投稿准备

## 规范分工

-`ROUTER.md`: 阶段, 能力, 初始化和路由
-`指导/使用说明.md`: 所有阶段与能力共同遵守的输入, 协作, 证据和执行协议
-`指导/` 中的其他文件: 某个阶段或能力的专门方法

发生冲突时, 本文件优先于`ROUTER.md`, 再优先于通用协议和专项指导, 专项指导可以增加约束, 但不能绕过上层证据边界和用户确认

## 仓库概念

- **工作区**:`workspace/` 的直接子目录是中文领域目录或中文研究项目目录, 不增加`学习/`,`研究/` 等固定类型中间层
- **领域目录**:`workspace/<中文领域名>/`, 保存学习规划, 领域概览和按论文组织的档案
- **论文目录**:`workspace/<中文领域名>/<论文中文短标题>/`, 一篇论文的原始 PDF, 中文 PDF, 双语对照 PDF 和元数据集中在同一目录
- **研究项目目录**:`workspace/<中文研究项目名>/`, 保存 Idea, Story, Claim, 证据, 实验交接和稿件, 引用领域论文档案而不重复复制
- **实验仓库容器**: 根目录`labs/`, 只放用户手动准备的独立实验代码仓库, 被本研究仓库忽略

人可见目录和研究文档优先使用中文名称, 同名时让用户选择有意义的限定词, 不自动追加不透明编号

## Agent 启动顺序

启动读取和路由完成前, 不得预设或向用户宣布将使用的具体工具, Skill, 执行流程和输出目录

1. 读取`ROUTER.md`, 理解阶段, 能力, 初始化和路由规则
2. 确认用户指向新领域, 已有领域, 研究项目, 还是一次性独立能力
3. 对已有目标, 先列出文件并通读目录中的 Markdown 记录
4. 根据用户目标推导当前阶段或能力
5. 读取`指导/使用说明.md`, 确认输入, 确认和落盘协议
6. 读取当前阶段或能力的专项指导, 完成后才形成并说明具体执行方案
7. 输入, 路径和确认条件满足后执行, 结束时回写真实产物

## 最高级边界

- **中文优先**: 正式英文论文形成前, 研究构思, 叙事, 实验设计, 结果分析和中间文档使用中文
- **人类决策优先**: Agent 负责检索, 规整, 反驳, 结构化和执行, 方向, Story, 实验范围, 结果口径, 阶段跳过和重大投稿决定由用户确认
- **证据优先**: 外部事实, 相关工作, 实验结论和投稿政策必须可追溯, 不确定内容标为未核验, 推断或待验证
- **不伪造**: 不得伪造论文, 作者, 机构, 引用, 原文位置, 代码能力, 实验数据或结果
- **不粉饰**: 不得通过换名, 换壳, 修辞或隐藏撞车工作, 反例, 负结果强化 Idea
- **模型上下文隔离**: 归档, PDF 翻译, 编译和文件检查等机械任务由外部工具在模型上下文之外处理, 命令输出只返回状态和摘要, 不回显正文, 只有用户明确要求摘要, 审稿或内容检查时, 才按最小范围读取正文
- **论文翻译自动化**: PDF 翻译由`scripts/translate-pdf.ps1` 在独立进程中完成, Agent 不逐页介入, 不抽查译文, 不修正输出, 管线失败时报告错误并停止, 不得改写 PDF2zh 源码或手工接管正文处理
- **实验仓库隔离**: 实验仓库必须由用户手动准备在`labs/<实际仓库目录>/`, Agent 不得替用户创建, clone 或初始化, 进入后视为独立 Git 工作区, 本仓库只约束研究目标, Claim, 证据和交接记录, 不规定其内部布局
- **文档即研究记录**: 所有文档必须是人类可读的研究记录, 不创建仅用于 Agent 跟踪状态的元文档, 任务进度由已有研究文档的内容自然推导
