# 仓库执行入口

## 仓库定位

本仓库是面向计算机科学研究的本地工作区, 同时包含研究工作台 Web 服务。Agent 通过服务器中的学习数据和工作区中的真实研究文档理解任务状态, 并按用户确认推进学习、研究、写作和投稿准备。

## 规范分工

- `ROUTER.md`: 所有阶段、能力、初始化、证据、协作、执行和暂停协议
- `指导/` 中的其他文件: 某个阶段或能力的专门方法
- `backend/` 与 `frontend/`: 研究工作台服务, 不是研究文档

## 仓库概念

### 学习数据与论文资源

- **服务器唯一权威**: 学习项目、论文记录、筛选状态和 PDF 资源统一由研究工作台后端管理。SQLite 和服务器资源目录是实现细节, Agent 不直接读写。
- **学习项目**: “学习”是 Web 工作台的固定功能区, 其下一层直接是项目。项目从 `GET /api/projects` 读取, 新建前检查已有名称和范围, 需要时通过服务器接口执行。
- **论文记录**: 文献检索为每篇候选补全当前服务器契约要求的全部字段, 立即以`discovered`状态提交数据库。用户在页面查看后通过对话让 Agent 更新为`included`、`excluded`或`archived`。
- **临时 JSONL**: 只用于一次批量提交, 放在被忽略的`tmp/`; 不是候选列表或长期记录, 提交完成后删除。Agent 不维护与服务器等价的学习 Markdown。
- **论文 PDF**: 原文、中文译文和双语版本由服务器论文资源接口管理。Agent 获取或接收原文后上传为`original`; 不在`workspace/`建立领域目录或论文目录, 不直接向`backend/data/`复制文件。
- **翻译**: 用户在 Web 页面中启动后台翻译。Agent 不把 PDF 翻译当作学习能力, 不安装或直接调用翻译脚本。
- **领域概览**: HTML 领域概览尚未接入当前工作台, 暂不执行。现有领域概览指导只作为未来迁移草案。
- **无兼容导入**: 当前学习数据只通过现行服务器契约创建和修改, 不实现旧 Markdown 导入、数据库迁移或历史格式兼容。

### 研究工作区

- **研究项目目录**: `workspace/<中文研究项目名>/`, 保存 Idea、Story、Claim、证据、实验交接、稿件和实际投稿材料。
- **实验仓库容器**: 根目录`labs/`, 只放用户手动准备的独立实验代码仓库, 被本研究仓库忽略。
- **临时目录**: `tmp/`只放任务级临时机械文件, 不承担长期研究责任。

人可见研究目录和研究文档优先使用中文名称。同名时让用户选择有意义的限定词, 不自动追加不透明编号。

## Agent 启动顺序

启动读取和路由完成前, 不得预设或向用户宣布将使用的具体工具、Skill、执行流程和输出目录。

1. 读取`ROUTER.md`, 理解共同协议、阶段、能力、初始化和路由规则。
2. 确认用户指向服务器中的新学习项目、已有学习项目、新研究项目、已有研究项目, 还是一次性独立能力, 不混淆研究与学习。
3. 对已有学习目标, 先读取服务器项目列表和相关论文记录; 对已有研究目标, 先列出文件并通读目录中的 Markdown 记录。
4. 根据用户目标推导当前阶段或专项能力。如果对应能力不存在、暂停或尚未验证, 先说明缺少稳定专项指导; 低风险一次性任务可按用户原请求继续, 会产生长期产物或影响重大决策时暂停并由用户确认是否继续。
5. 读取当前阶段或能力的专项指导, 完成后才形成并说明具体执行方案。
6. 输入、服务器状态、路径和确认条件满足后执行, 结束时回读真实服务器记录或回写真实研究产物。
