# hxaxd-research

一个可以直接在任何你喜欢的 Harness+Model 中使用的本地计算机科学研究工作区, 面向领域学习、Idea 审查、论文叙事、实验证明、实验执行、论文写作和投稿准备。

仓库中的 Markdown 文档是公开、可修改的执行规范。

## 快速开始

1. 克隆仓库, 用任何你喜欢的 Harness+Model 打开仓库根目录
2. 如果你的 Harness 不会默认加载`AGENTS.md`, 请在请求中引用`@AGENTS.md`
3. 描述一个新任务, 或指定已有的领域、论文目录或研究项目目录
4. 根据 Agent 给出的影响说明, 确认方向和范围, Agent 会边做边写, 与你迭代直到认可

例如:

```text
我想学习 LLM Agent // 领域学习
```

```text
我有一个 idea ... // 新建研究
```

```text
请继续 {某个工作区目录} 的研究工作 // 继续研究
```

Agent 会依次读取 AGENTS → ROUTER → 判断你的意图 → 检查工作区 → 路由到对应阶段或能力 → 读取使用说明和专项指导 → 与你对齐关键决定 → 逐步写入真实产物。

## 工作方式

仓库围绕两件事组织: **领域学习**和**研究推进**。你可以根据当前输入直接进入任意阶段; 缺少输入时, Agent 会说明缺口和继续风险。

### 领域学习

系统掌握一个领域, 并收录你可能反复引用的论文:

| 顺序 | 做什么 | 产物 |
| ---- | ---- | ---- |
| 1 | 文献检索与收录 | `论文列表.md`, 按输入生成`原文.pdf` |
| 2 | 建立领域概览 | `领域概览.md` |
| 3 | 获取合法完整 PDF | `原文.pdf` |
| 4 | 按需翻译已有原文 | `中文译文.pdf`, `双语对照.pdf` |

### 研究推进

| 阶段 | 做什么 | 产物 |
| ---- | ---- | ---- |
| R1 | 对齐 Idea, 写成自然语言研究想法和四段式核心主张; 撞车分析和硬度审查只在对话中完成 | `研究想法.md` |
| R2 | 先寻找并确认中心叙事, 再从叙事中抽取承重主张 | `叙事定稿.md` |
| R3 | 把经验性主张转化为证明责任, 固定共同实验协议、替代解释控制和结果交付 | `实验证明.md` |
| R4 | 在用户准备的独立实验仓库中按冻结协议实现、正式运行并回填真实结果 | 更新后的`实验证明.md`和`labs/<仓库>/`内产物 |
| R5 | 围绕真实 Story 与证据直接维护唯一权威 TeX 论文稿件 | `论文稿件.tex`及其真实依赖 |
| R6 | 按当前官方规则生成投稿包, 并按需处理返修与终稿 | 实际投稿材料、审稿回复、修订稿或终稿 |

### 正式能力

以下能力不绑定研究阶段, 可按需独立调用:

- 科研配图
- 学术审阅与反向论证
- 文献检索与收录
- 论文资源获取
- PDF 翻译
- 领域概览

### 工作区结构

`workspace/` 顶层扁平——领域目录和研究项目目录直接并列:

```text
workspace/
├─ <中文领域名>/
│  ├─ 论文列表.md
│  ├─ 领域概览.md
│  └─ <论文中文短标题>/
│     ├─ 原文.pdf
│     ├─ 中文译文.pdf
│     └─ 双语对照.pdf
│
└─ <中文研究项目名>/
   ├─ 研究想法.md
   ├─ 叙事定稿.md
   ├─ 实验证明.md
   └─ 论文稿件.tex 及其实际依赖和投稿材料
```

### 你会关心的几件事

**论文目录在取得原文时创建。** 目录至少包含`原文.pdf`; 调用 PDF 翻译后再成对增加中文译文和双语对照, 目录内容遵守固定文件集合。

**领域归属由你决定。** 如果你直接提供一篇尚未归类的 PDF, Agent 会进入文献检索与收录: 先检查已有领域并提出候选, 由你确认归属后把论文完整记录写入`论文列表.md`, 再顺手将 PDF 保存为对应论文目录中的`原文.pdf`。因为原文已经由你提供, 不再调用论文资源获取。

**任务状态由真实文档体现。** Agent 以当前研究文档和实际产物继续工作。

**PDF 翻译是机械过程, 正文不进入 Agent 上下文。** PDF2zh 在独立进程中处理全文, Agent 只接收文件状态、大小、哈希和错误摘要。只有你明确要求摘要或内容检查时, 它才读取完成任务所需的局部内容。

## 实验仓库

实验代码不放入中文研究项目目录。需要实验工程时, 由用户把独立代码仓库手动准备到根目录的`labs/<实际仓库目录>/`, 研究文档记录该路径和真实产物位置。

`labs/` 被本仓库忽略。Agent 不会替用户创建、初始化或拉取实验仓库, 也不会规定独立仓库内部的代码布局; R3 固定会影响研究结论的实验协议, R4 自由决定工程实现, 并在独立仓库保存机器可读运行清单、日志和真实结果。研究项目只维护证明责任、结果摘要和可追溯路径。

## PDF 翻译脚本

> 仓库通过外部开源项目 [`PDFMathTranslate-next`](https://github.com/PDFMathTranslate/PDFMathTranslate-next) 处理 PDF。

### 安装

**PDF2zh 的安装、环境变量和首次资源下载建议由 Agent 引导完成**, 不要跳过 Agent 手工配置:

```text
安装 PDF2zh 环境
```

- Agent 会按配置指导完成环境安装与验证; 首次翻译可能下载额外资源, Agent 会提前说明。

### 校验

配置完成后, 尝试翻译一篇已经获取的论文 PDF:

```text
翻译 .../原文.pdf
```

- 详细参数、密钥解析顺序和行为边界见[`指导/学习/PDF 翻译脚本配置.md`](指导/学习/PDF 翻译脚本配置.md)
- 完整处理流程见[`指导/学习/PDF 翻译脚本使用.md`](指导/学习/PDF 翻译脚本使用.md)
