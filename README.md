# hxaxd-research

一个可以直接在任何你喜欢的 Harness+Model 中使用的本地计算机科学研究工作区, 面向领域学习、Idea 审查、实验证明和实验执行。写作与投稿目前只保留为未验证占位, 不作为稳定流程承诺。

本项目不是 Skill, 仓库中的 Markdown 文档就是公开、可修改的执行规范。

## 快速开始

1. 克隆仓库, 用任何你喜欢的 Harness+Model 打开仓库根目录
2. 如果你的 Harness 不会默认加载`AGENTS.md`, 请在请求中引用`@AGENTS.md`
3. 描述一个新任务, 或指定已有的领域、论文目录或研究项目目录
4. 根据 Agent 给出的影响说明, 确认方向和范围, Agent 会边做边写, 与你迭代直到认可

例如:

```text
我想学习 LLM Agent // 领域学习
```

```text
我有一个 idea ... // 新建研究
```

```text
请继续 {某个工作区目录} 的研究工作 // 继续研究
```

Agent 会依次读取 AGENTS → ROUTER → 判断你的意图 → 检查工作区 → 路由到对应阶段或能力 → 读取使用说明和专项指导 → 与你对齐关键决定 → 逐步写入真实产物。

## 工作方式

仓库围绕两件事组织: **领域学习**和**研究推进**。推荐路线不是门禁, 你可以直接进入任意阶段; 缺少输入时, Agent 会说明缺口和继续风险。

### 领域学习

系统掌握一个领域, 并归档你可能反复引用的论文:

| 顺序 | 做什么 | 产物 |
| ---- | ---- | ---- |
| 1 | 文献检索与归档: 发现论文, 策划列表并归档 | `论文列表.md` |
| 2 | 建立领域概览 | `领域概览.md` |
| 3 | 获取全文并翻译重点论文 | `原文.pdf`, `中文译文.pdf`, `双语对照.pdf` |

### 研究推进

新版研究主线:

| 阶段 | 做什么 | 产物 |
| ---- | ---- | ---- |
| R1 | 对齐 Idea, 写成自然语言研究想法和四段式核心主张; 撞车分析和硬度审查只在对话中完成 | `研究想法.md` |
| R2 | 根据研究想法展开论文叙事, 拆出分点核心主张 | `叙事定稿.md` |
| R3 | 为需要经验支持的分点主张写分点证明方案和结果交付要求 | `实验证明.md` |
| R4 | 在用户准备的独立实验仓库中实现、运行并回填真实结果、状态和产物路径 | 更新后的`实验证明.md`和`labs/<仓库>/`内产物 |
| R5 | 论文写作占位, 当前没有稳定专项指导 | 由用户请求决定 |
| R6 | 投稿准备占位, 当前没有稳定专项指导 | 由用户请求决定 |

实验阶段不再拆出独立中间计划、运行归档或单独事后判断文档。R3 先声明证明目标和交付要求, R4 由 coding agent 在独立实验仓库中自由落实。

### 正式能力

以下能力不绑定研究阶段, 可按需独立调用:

- 文献检索与归档
- 论文资源获取
- 论文归档与 PDF 翻译
- 领域概览

### 工作区结构

`workspace/` 顶层扁平——领域目录和研究项目目录直接并列:

```text
workspace/
├─ <中文领域名>/
│  ├─ 论文列表.md
│  ├─ 领域概览.md
│  └─ <论文中文短标题>/
│     ├─ 原文.pdf
│     ├─ 中文译文.pdf
│     └─ 双语对照.pdf
│
└─ <中文研究项目名>/
   ├─ 研究想法.md
   ├─ 叙事定稿.md
   ├─ 文献证据库.md
   ├─ 实验证明.md
   └─ 稿件与其他研究产物
```

### 你会关心的几件事

**论文目录按需创建, 不预建空文件。** 一个论文档案集中保存原始 PDF 和译文, 研究项目通过路径引用领域论文档案, 不重复复制。

**领域归属由你决定。** 如果你只提供一篇尚未归类的 PDF, Agent 会先检查已有领域并提出候选, 但最终归属由你确认, Agent 不会静默替你分类。

**任务状态由真实文档推导。** 仓库不依赖隐藏字段或额外状态文件。

**PDF 翻译是机械过程, 正文不进入 Agent 上下文。** PDF2zh 在独立进程中处理全文, Agent 只接收文件状态、大小、哈希和错误摘要。只有你明确要求摘要或内容检查时, 它才读取完成任务所需的局部内容。

## 实验仓库

实验代码不放入中文研究项目目录。需要实验工程时, 由用户把独立代码仓库手动准备到根目录的`labs/<实际仓库目录>/`, 研究文档记录该路径和真实产物位置。

`labs/`被本仓库忽略。Agent 不会替用户创建、初始化或拉取实验仓库, 也不会规定独立仓库内部的代码、日志和结果布局; 它只在研究文档中维护实验目标、结果交付摘要和可追溯路径。

## PDF 翻译工具

> 仓库通过外部开源项目 [`PDFMathTranslate-next`](https://github.com/PDFMathTranslate/PDFMathTranslate-next) 处理 PDF, 固定使用`pdf2zh-next==2.9.0`和 DeepSeek V4 Flash。

### 安装

**PDF2zh 的安装、环境变量和首次资源下载建议由 Agent 引导完成**, 不要跳过 Agent 手工配置:

```text
安装 PDF2zh 环境
```

- Agent 会处理`uv`、Python、固定版本安装、`PDF2ZH_DEEPSEEK_API_KEY`环境变量的设置与验证, 首次翻译还可能下载模型和字体, Agent 会提前说明。

### 校验

配置完成后, 尝试翻译并归档一篇论文的 PDF:

```text
翻译并归档 .../原文.pdf
```

- 详细参数、密钥解析顺序和行为边界见[`指导/学习/论文处理工具配置.md`](指导/学习/论文处理工具配置.md)
- 完整处理流程见[`指导/学习/论文处理工具使用.md`](指导/学习/论文处理工具使用.md)
