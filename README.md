# hxaxd-research

一个可以直接在 Codex 以及任何你喜欢的 Harness+Model 中使用的本地计算机科学研究工作区, 覆盖领域学习, Idea 审查, 实验, 论文写作, 审稿和投稿准备

本项目不是 Skill, 仓库中的 Markdown 文档就是公开, 可修改的执行规范

## 快速开始

1. 克隆仓库, 并用 Codex 以及任何你喜欢的 Harness+Model 打开仓库根目录
2. 在请求中引用 `@AGENTS.md`
3. 描述一个新任务, 或指定已有的领域, 论文目录或研究项目目录
4. 根据 Codex 给出的影响说明, 确认方向, 范围和是否落盘

例如:

```text
@AGENTS.md
我想系统学习 Coding Agent // 领域学习
```

```text
@AGENTS.md
我有一个 idea ... // 新建研究
```

```text
@AGENTS.md
请继续 {某个工作区目录} 的研究工作 // 继续研究
```

```text
@AGENTS.md
请帮我给 {某个工作区目录} 的文章设计配图 // 进行生命周期无关的原子任务
```

Codex 会读取 [`WORKFLOW.md`](WORKFLOW.md), 从已有文档判断任务主线, 再读取 [`指导/使用说明.md`](指导/使用说明.md) 和对应专项指导, 关键输入缺失, 方向改变或到达确认点时, 它会先与你对齐, 确认后逐步写入真实产物

## 工作方式

仓库区分两种工作:

- **生命周期阶段**表示任务主线发生了什么变化, 推荐路线是学习规划, 知识地图, Idea 立项, Story, 实验, 写作和投稿准备
- **按需能力**像可重复调用的函数, 包括文献与全文定位, 论文归档与处理, Idea 自动发现, 知识更新, 绘图, 翻译, 修订, 审稿, 投稿规则, 引用核验, Tex 验收和匿名检查

阶段之间的箭头是推荐路线, 不是门禁, 用户可以直接进入任意可执行阶段, 也可以明确跳过阶段, 跳过会记录原因和证据影响, 但不会伪装成完成, 翻译, 审稿, 引用核验等能力也可以作为一次性任务独立使用

`workspace/` 顶层保持扁平, 直接放中文领域目录和中文研究项目目录:

```text
workspace/
├─ <中文领域名>/
│  ├─ 学习目标.md
│  ├─ 最终论文列表.md
│  ├─ 领域全景地图.md
│  ├─ 资源获取记录.md
│  └─ <论文中文短标题>/
│     ├─ 论文信息.md
│     ├─ 原文.pdf
│     ├─ 中文译文.pdf
│     └─ 双语对照.pdf
│
└─ <中文研究项目名>/
   ├─ 研究想法.md
   ├─ 叙事定稿.md
   ├─ 核心主张.md
   ├─ 文献证据库.md
   ├─ 实验记录/
   └─ 稿件与其他研究产物
```

论文目录中的文件按实际需要逐步创建, 不要求一次生成全部内容, 一个论文档案集中保存原始 PDF, 中文 PDF, 双语对照 PDF, 研究项目的文献证据库通过路径引用领域论文档案, 不重复复制整套材料

如果用户只提供一篇尚未归类的 PDF, Codex 会先检查已有领域, 提出可能归属, 并询问应加入哪个已有领域或创建哪个中文领域目录, 领域归属不会由 Agent 静默决定

任务状态不保存在隐藏字段或额外状态文件中, 而由真实文档, 用户确认, 跳过记录和未决问题共同推导

论文处理时, PDF2zh 在独立进程中读取全文并生成中文与双语 PDF, 正文默认不进入 Agent 上下文, Agent 只接收状态, 大小, 哈希和错误摘要, 只有你要求摘要, 审稿或内容检查时, 它才按需读取局部内容

## 实验仓库

实验代码不放入中文研究项目目录, 需要实验工程时, 由用户把独立代码仓库手动准备到根目录的 `labs/<实际仓库目录>/`, 任务文档记录该路径, 上游仓库, 分支和基线 commit

`labs/` 被本仓库忽略, Codex 不会替用户创建, 初始化或拉取实验仓库, 也不会规定独立仓库内部的代码, 日志和结果布局, 它只在研究文档中维护实验目标, 证据和可追溯交接

## PDF 翻译工具

仓库通过外部开源项目 [`PDFMathTranslate-next`](https://github.com/PDFMathTranslate/PDFMathTranslate-next) 处理 PDF, 固定使用 `pdf2zh-next==2.9.0` 和 DeepSeek V4 Flash

**PDF2zh 的安装, 环境变量和首次资源下载建议由 Agent 引导完成**, 不要跳过 Agent 手工配置:

```text
@AGENTS.md
请安装 PDF2zh 环境
```

Agent 会处理 `uv`, Python, 固定版本安装, `PDF2ZH_DEEPSEEK_API_KEY` 环境变量的设置与验证, 首次翻译还可能下载模型和字体, Agent 会提前说明

配置完成后, 尝试翻译并归档一篇论文的 PDF (Agent 也可代为执行) :

```text
@AGENTS.md
请翻译并归档 .../原文.pdf
```

详细参数, 密钥解析顺序和行为边界见 [`指导/学习/论文处理工具配置.md`](指导/学习/论文处理工具配置.md), 完整处理流程见 [`指导/学习/论文处理工具使用.md`](指导/学习/论文处理工具使用.md), 翻译会把正文发送到配置的 API, 执行前必须得到用户许可

仓库维护者可阅读 [`开发指南.md`](开发指南.md), 了解文档分层, 路由设计和未来修改某类规则时应更新哪些文件
