# hxaxd-research

一个面向计算机科学研究的本地工作区。你可以把一个领域、一篇论文或一个研究 Idea 交给 Agent, 让它在真实文档和实验产物上继续检索、审查、设计、写作与投稿准备; 研究方向和关键判断始终由你确认。

## 快速开始

1. 克隆仓库, 用支持项目上下文的 Agent 打开仓库根目录
2. 如果当前工具不会自动读取`AGENTS.md`, 在第一次请求中引用`@AGENTS.md`
3. 直接描述目标, 或指定`workspace/`中的已有目录
4. 查看 Agent 对当前理解、证据缺口和关键选择的说明, 与它迭代到你认可

例如:

```text
我想系统学习 LLM Agent, 先帮我建立论文列表。
```

```text
我有一个研究 idea: ... 请先帮我整理并审查。
```

```text
继续 workspace/某个研究项目, 现在开始设计实验证明。
```

```text
根据现有叙事和实验结果开始写论文。
```

## 能做什么

### 领域学习

| 目标 | 主要产物 |
| --- | --- |
| 检索并收录论文 | `论文列表.md` |
| 建立领域知识结构 | `领域概览.md` |
| 获取合法完整原文 | `原文.pdf` |
| 翻译已有原文 | `中文译文.pdf`、`双语对照.pdf` |

### 研究推进

| 阶段 | 解决的问题 | 主要产物 |
| --- | --- | --- |
| R1 | 把原始 Idea 说准, 并审查是否撞车、是否值得做 | `研究想法.md` |
| R2 | 找到论文真正的中心叙事, 澄清全文共用事实与承重主张 | `叙事定稿.md` |
| R3 | 把方法、基线和各类实验组织成完整的证明方案 | `实验证明.md` |
| R4 | 在独立实验仓库中实现方案、运行实验并交付真实结果 | 实验产物与更新后的`实验证明.md` |
| R5 | 围绕已确认的事实和证据维护一份权威 TeX 稿件 | `论文稿件.tex`及其依赖 |
| R6 | 根据目标场所的当前规则准备投稿、返修和终稿 | 实际投稿材料 |

阶段之间表示常见依赖关系。已有可靠输入时可以直接进入相应工作; 前置事实不足时, Agent 会说明继续写作或实验会承担什么风险。

### 独立能力

- 科研配图: 把方法关系或真实结果转化为可用于论文的图件
- 学术审阅与反向论证: 对 Idea、叙事、实验或稿件给出证据稳定的反方判断
- 文献检索与收录、论文资源获取、PDF 翻译和领域概览

## 工作区

`workspace/`顶层直接放领域目录和研究项目目录:

```text
workspace/
├─ <中文领域名>/
│  ├─ 论文列表.md
│  ├─ 领域概览.md
│  └─ <论文中文短标题>/
│     ├─ 原文.pdf
│     ├─ 中文译文.pdf
│     └─ 双语对照.pdf
│
└─ <中文研究项目名>/
   ├─ 研究想法.md
   ├─ 叙事定稿.md
   ├─ 实验证明.md
   └─ 论文稿件.tex 及其真实依赖和投稿材料
```

研究状态由这些文档和真实产物本身体现, Agent 会直接在已有内容上继续工作。

## 实验代码

实验工程放在根目录的`labs/<仓库目录>/`, 与研究文档分开。`labs/`由用户手动准备并被本仓库忽略; Agent 会沿用实验仓库自己的工程规范, 把代码、配置、日志和原始结果留在那里, 再把论文需要的证明关系和结果摘要写回研究项目。

## PDF 翻译

仓库使用外部开源项目[`PDFMathTranslate-next`](https://github.com/PDFMathTranslate/PDFMathTranslate-next)处理 PDF。首次使用可以直接请求:

```text
安装 PDF2zh 环境
```

配置完成后, 对已取得的论文请求:

```text
翻译 .../原文.pdf
```

翻译由独立进程处理, Agent 只接收文件状态与必要错误摘要。配置和使用细节见[`PDF 翻译脚本配置`](指导/学习/PDF%20翻译脚本配置.md)与[`PDF 翻译脚本使用`](指导/学习/PDF%20翻译脚本使用.md)。
