# hxaxd-research

一个可以直接在 Codex 以及任何你喜欢的 Harness+Model 中使用的本地计算机科学研究工作区, 覆盖领域学习, Idea 审查, 实验, 论文写作, 审稿和投稿准备

本项目不是 Skill, 仓库中的 Markdown 文档就是公开, 可修改的执行规范

## 快速开始

1. 克隆仓库, 并用 Codex 以及任何你喜欢的 Harness+Model 打开仓库根目录
2. 在请求中引用 `@AGENTS.md`
3. 描述一个新任务, 或指定已有的领域, 论文目录或研究项目目录
4. 根据 Codex 给出的影响说明, 确认方向, 范围和是否落盘

例如:

```text
@AGENTS.md
我想系统学习 Coding Agent // 领域学习
```

```text
@AGENTS.md
我有一个 idea ... // 新建研究
```

```text
@AGENTS.md
请继续 {某个工作区目录} 的研究工作 // 继续研究
```

```text
@AGENTS.md
请帮我给 {某个工作区目录} 的文章设计配图 // 进行阶段无关的原子任务
```

引用 `@AGENTS.md` 后, Codex 会依次读取 AGENTS → ROUTER → 判断你的意图 → 检查工作区 → 路由到对应阶段或能力 → 读取使用说明和专项指导 → 与你对齐关键决定 → 逐步写入真实产物

## 工作方式

仓库围绕两件事组织: **领域学习**和**研究写作**, 每件事有自己的推荐路线, 路线上的阶段不是门禁 —— 你可以直接进入任意阶段

### 领域学习

系统掌握一个领域, 并归档你可能反复引用的论文:

| 顺序 | 做什么 | 产物 |
| ---- | ---- | ---- |
| 1 | 制定学习目标, 范围与阅读方案 | `学习目标.md` |
| 2 | 生成论文列表, 建立领域知识地图 | `论文列表.md`, `领域全景地图.md` |
| 3 | 获取全文, 归档并翻译重点论文 | `原文.pdf`, `中文译文.pdf`, `双语对照.pdf` |

### 研究写作

从 Idea 到投稿的完整主线:

| 顺序 | 做什么 | 产物 |
| ---- | ---- | ---- |
| 1 | 审查 Idea 是否成立, 是否撞车, 是否可验证 | `研究想法.md` |
| 2 | 构造论文叙事, 明确核心主张 | `叙事定稿.md`, `核心主张.md` |
| 3 | 设计实验, 定义每项 Claim 的证伪条件 | `实验证明设计.md` |
| 4 | 将实验映射到代码仓库与可执行计划 | `实验编程计划.md` |
| 5 | 执行实验, 保留原始结果与失败记录 | `实验记录/` |
| 6 | 判断证据强度, 确定论文口径 | `结果分析.md` |
| 7 | 写稿, 以任意章节顺序持续形成与修订 | 中文或英文 Tex 草稿 |
| 8 | 投稿准备, 终稿检查, 匿名化与合规核验 | 投稿材料 |

### 通用原子能力

以下能力不绑定任何阶段, 领域学习和研究写作均可按需调用:

- 文献检索与全文定位
- 论文归档与 PDF 翻译
- Idea 自动发现
- 领域知识地图更新
- 文本修订（润色、缩写、去 AI 痕迹）
- 中文稿英译
- 写作辅助（英文润色/扩写/缩写、逻辑检查、标题生成、图表标题、数据分析段落）
- 图表规划与生成
- 综合审稿
- 投稿规则与规范检查
- 引用核验
- Tex 成稿验收
- 稿件匿名化

### 工作区结构

`workspace/` 顶层扁平——领域目录和研究项目目录直接并列:

```text
workspace/
├─ <中文领域名>/
│  ├─ 学习目标.md
│  ├─ 论文列表.md
│  ├─ 领域全景地图.md
│  └─ <论文中文短标题>/
│     ├─ 论文信息.md
│     ├─ 原文.pdf
│     ├─ 中文译文.pdf
│     └─ 双语对照.pdf
│
└─ <中文研究项目名>/
   ├─ 研究想法.md
   ├─ 叙事定稿.md
   ├─ 核心主张.md
   ├─ 文献证据库.md
   ├─ 实验记录/
   └─ 稿件与其他研究产物
```

### 你会关心的几件事

**论文目录按需创建, 不预建空文件,** 一个论文档案集中保存原始 PDF 和译文, 研究项目通过路径引用领域论文档案, 不重复复制

**领域归属由你决定,** 如果你只提供一篇尚未归类的 PDF, Agent 会先检查已有领域并提出候选, 但最终归属由你确认, Agent 不会静默替你分类

**任务状态由真实文档推导,** 不依赖隐藏字段或额外状态文件

**PDF 翻译是机械过程, 正文不进入 Agent 上下文,** PDF2zh 在独立进程中处理全文, Agent 只接收文件状态, 大小, 哈希和错误摘要, 只有你明确要求摘要, 审稿或内容检查时, 它才读取完成任务所需的局部内容

## 实验仓库

实验代码不放入中文研究项目目录, 需要实验工程时, 由用户把独立代码仓库手动准备到根目录的 `labs/<实际仓库目录>/`, 任务文档记录该路径, 上游仓库, 分支和基线 commit

`labs/` 被本仓库忽略, Codex 不会替用户创建, 初始化或拉取实验仓库, 也不会规定独立仓库内部的代码, 日志和结果布局, 它只在研究文档中维护实验目标, 证据和可追溯交接

## PDF 翻译工具

> 仓库通过外部开源项目 [`PDFMathTranslate-next`](https://github.com/PDFMathTranslate/PDFMathTranslate-next) 处理 PDF, 固定使用 `pdf2zh-next==2.9.0` 和 DeepSeek V4 Flash

### 安装

**PDF2zh 的安装, 环境变量和首次资源下载建议由 Agent 引导完成**, 不要跳过 Agent 手工配置:

```text
@AGENTS.md
请安装 PDF2zh 环境
```

- Agent 会处理 `uv`, Python, 固定版本安装, `PDF2ZH_DEEPSEEK_API_KEY` 环境变量的设置与验证, 首次翻译还可能下载模型和字体, Agent 会提前说明

### 校验

配置完成后, 尝试翻译并归档一篇论文的 PDF (Agent 也可代为执行) :

```text
@AGENTS.md
请翻译并归档 .../原文.pdf
```

- 详细参数, 密钥解析顺序和行为边界见 [`指导/学习/论文处理工具配置.md`](指导/学习/论文处理工具配置.md)
- 完整处理流程见 [`指导/学习/论文处理工具使用.md`](指导/学习/论文处理工具使用.md)
