# 阶段与能力路由

任何阶段或能力都由当前公开输入触发。若需要的产物还不存在, 说明缺什么和继续风险, 由用户决定补前置还是接受风险继续。

## 1. 为什么这样组织

### 学习: 能力组合, 而非阶段

领域学习由一组可反复调用的能力组成: 文献检索与收录发现论文并写入`论文列表.md`, 论文资源获取取得合法完整 PDF 并保存为`原文.pdf`, PDF 翻译基于已有原文生成中文和双语译文, 领域概览从论文列表提炼知识结构并写入`领域概览.md`。新论文加入后可随时重新调用这些能力更新已有产物。

### 研究: R1 到 R6

研究主线只保留当前确认的六个阶段:

1. R1 Idea 预审与立项: 把用户随口描述的 Idea 对齐并写入`研究想法.md`; 撞车分析和硬度审查在对话中形成结论。
2. R2 论文叙事与分点核心主张: 基于`研究想法.md`先寻找并确认中心叙事, 再固定下游共用的论文事实并抽取承重主张, 产出`叙事定稿.md`。
3. R3 实验证明: 定义完整方法与公平比较边界, 以主实验、分点深化、消融、迁移、效率和诊断等实验家族组织证据, 再闭合经验性分点主张, 产出`实验证明.md`。
4. R4 实验执行与结果交付: 在用户手动准备的`labs/<仓库目录>/`中按冻结的方法身份、实验组合和共同协议实现与运行, 保存正式产物并回填`实验证明.md`。
5. R5 论文写作: 把确认后的 Story、论文事实、核心主张和真实证据写入唯一权威 TeX 稿件。
6. R6 投稿准备与返修: 按目标投稿场所当前官方规则生成真实投稿包, 并处理需要的审稿回复、修订稿和终稿。

### 能力: 独立触发, 做完即结束

能力由用户请求独立触发, 完成各自公开契约中的任务和产物。

### 条目格式

下面每个阶段和能力用统一字段描述:

- **做什么** — 一句话说明目标
- **触发** — 用户说什么时进入
- **不触发** — 什么情况下不该进入
- **需要** — 必须就绪的输入
- **产出** — 会生成或更新什么
- **指导** — 读取哪个专项指导; 若无稳定指导则明确写明

## 2. 阶段

### R1 Idea 预审与立项

- **做什么**: 对齐用户原始 Idea, 写成正式自然语言研究想法和四段式核心主张; 随后推荐并在对话中执行撞车分析和硬度审查。
- **触发**: 用户提出新的研究 Idea, 或要求重新审查尚未定稿的 Idea。
- **不触发**: 用户已有确认后的`叙事定稿.md`并只是继续实验、写作或投稿。
- **需要**: 原始 Idea 描述; 若要创建新研究目录, 需要可识别的中文项目名或用户确认的目录名。
- **产出**: `研究想法.md`; 撞车分析和硬度审查在对话中形成结论。
- **指导**: `指导/研究/研究想法审查与落地.md`, 参考: `指导/参考/研究想法.md`。

### R2 论文叙事与分点核心主张

- **做什么**: 根据`研究想法.md`先寻找并确认中心叙事, 再固定任务、失败、原因、认识、设计关系和边界等下游共用事实, 并抽取可用于后续证明的承重主张。
- **触发**: R1 对齐完成且用户确认继续构造叙事, 或用户已有可靠 Idea 并明确接受跳过 R1 对话审查的风险。
- **不触发**: Idea 尚未写成`研究想法.md`, 或用户仍在讨论 Idea 本身。
- **需要**: `研究想法.md`; 用户已明确的任务与方法信息; 必要的领域事实、相近工作和文献来源。
- **产出**: `叙事定稿.md`。
- **指导**: `指导/研究/构造论文叙事.md`, 参考: `指导/参考/叙事定稿.md`。

### R3 实验证明

- **做什么**: 固定完整方法、方法变体与比较基线, 设计有主次的实验组合和共同协议, 再建立实验到经验性分点主张的证据闭环。
- **触发**: 分点主张中存在需要经验支持的内容。
- **不触发**: 纯理论工作, 或用户明确跳过实验。
- **需要**: `叙事定稿.md`中的论文事实、贡献性设计关系和分点核心主张; 完整方法的当前定义; 已知 Benchmark、Baseline、指标、资源边界和会影响结论的候选实验变量。
- **产出**: `实验证明.md`。
- **指导**: `指导/实验/实验证明.md`, 参考: `指导/参考/实验证明.md`。

### R4 实验执行与结果交付

- **做什么**: 让 coding agent 在用户手动准备的独立实验仓库中, 在不改变 R3 方法身份、实验组合和共同协议的前提下自由实现, 区分冒烟测试、预跑和正式运行, 保存可追溯产物并回填结果。
- **触发**: `实验证明.md`已确认, 且实验仓库已由用户准备在`labs/<仓库目录>/`。
- **不触发**: `labs/`中仓库不存在, 或 R3 仍缺少会影响研究结论的方法身份、核心实验、实验条件、判据或结果责任。
- **需要**: `研究想法.md`, `叙事定稿.md`, `实验证明.md`, `labs/<仓库目录>/`。
- **产出**: 独立实验仓库中的代码、配置、机器可读运行清单、日志、原始结果和结构化汇总; 按 R3 实验编号回填可用于论文的结果事实与真实路径, 再更新`实验证明.md`中的分点主张判定。
- **指导**: `指导/实验/实验执行.md`。

### R5 论文写作

- **做什么**: 把用户确认的研究 Story、论文事实、分点核心主张、原始文献和真实实验结果写成唯一权威的 TeX 论文稿件。
- **触发**: 用户要求开始或继续写论文, 且`叙事定稿.md`已存在; 需要写入的经验性结论已有真实结果, 或用户明确接受只写当前有证据的部分。
- **不触发**: 用户仍在构造中心叙事, 只需要设计或执行实验, 或要求的是一次性文字润色而非维护项目稿件。
- **需要**: `叙事定稿.md`中的中心叙事、论文事实和分点核心主张; 与当前稿件主张有关的`实验证明.md`和正式实验产物, 若适用; 被使用的原始文献; 已有稿件内容, 若存在。
- **产出**: TeX 主入口文件, 默认`论文稿件.tex`; 已有官方模板时沿用其主入口。另包括稿件实际依赖的图表、`references.bib`、附录 TeX 源文件和编译产物。
- **指导**: 入口`指导/写作/论文写作.md`, 由入口按内容读取`指导/写作/论证与表达.md`、`指导/写作/去模型化表达.md`和`指导/写作/章节/`; TeX 环境`指导/写作/TeX 环境配置.md`。

### R6 投稿准备与返修

- **做什么**: 按明确投稿场所和轮次的当前官方规则, 从权威稿件生成真实可提交材料, 并在需要时完成审稿回复、修订稿和终稿准备。
- **触发**: 用户指定目标投稿场所并要求准备投稿, 或提供真实审稿意见、编辑决定、返修或终稿要求。
- **不触发**: 只是在 R5 中写论文内容, 尚未确定目标投稿场所却要求假定具体格式, 或用户只想讨论投稿策略而不创建材料。
- **需要**: R5 权威稿件及其依赖; 明确的投稿场所、track、年份或轮次; 当前官方作者指南; 作者与声明信息; 返修时还需要真实审稿材料, 以及争议主张对应的`叙事定稿.md`和`实验证明.md`内容。
- **产出**: 当前投稿场所实际要求的主稿、源文件包、补充材料、声明、表单内容或 checklist; 需要时的审稿回复、修订稿和终稿。
- **指导**: `指导/投稿/投稿准备.md`。

阶段编号反映典型依赖关系。用户可以直接进入任意阶段, 缺失输入时说明风险并由用户决定。

## 3. 能力

### 科研配图

- **做什么**: 从论文主张和真实证据出发, 设计并生成实际可使用的结果图、概念图、方法图和定性图, 同时完成必要来源、图注与稿件接入。
- **触发**: 用户要求为研究或论文新建、修改、重构或检查科研图件, 或 R2/R4/R5/R6 中出现必须通过图来解释或验证的关系。
- **不触发**: 用户只需要数据分析而不需要图件; 只需要装饰性图片、通用海报或交互式 Dashboard; 只要求文字描述一张可能的图。
- **需要**: 图件要支持或解释的主张; 已确认的任务、失败、核心认识与设计关系, 已进入 R2 时从`叙事定稿.md`读取; R3/R4 的实验责任和真实结果, 若为结果图; 真实数据、样本、图像或方法关系; 已有稿件和图件源文件, 若存在; 最终载体和投稿要求, 若已确定。
- **产出**: 实际最终图件; 必要的源数据、绘图脚本、可编辑图源或再生成来源; 实际稿件中的图注和引用。
- **指导**: `指导/能力/科研配图.md`。

### 学术审阅与反向论证

- **做什么**: 对 Idea、叙事、论文事实、主张、实验或完整稿件进行证据驱动的反向审阅, 固定审阅标准并在用户反驳时只依据新证据稳定更新判断。
- **触发**: 用户要求撞车审查、硬度审查、攻击研究想法、模拟审稿、寻找致命问题、检查实验是否证明主张, 或复审用户对原问题的反驳。
- **不触发**: 用户只要求摘要、精读、语言润色或直接修改稿件, 没有独立判断需求。
- **需要**: 明确的被审对象及其当前真实版本; 被审对象的论文事实、贡献性设计关系和核心主张, 若已经形成; 相关文献、实验产物或目标投稿场所, 视审阅问题而定; 复审时还需要原审阅与用户新材料。
- **产出**: 默认在对话中给出准确复述、最强成立理由、最强反方、决定性问题、严重程度、当前结论和改变结论所需证据; 只有用户需要真实对外交付时才创建正式评审文件。
- **指导**: `指导/能力/学术审阅与反向论证.md`。

### 文献检索与收录

- **做什么**: 为领域名、研究方向、论文标题或一篇 PDF 发现论文, 策划列表, 并将用户确认论文的完整记录收录到论文列表; 用户直接提供单篇 PDF 时, 收录完成后顺手规范保存为`原文.pdf`。
- **触发**: 用户要系统学习一个领域, 或提供一篇尚未收录的论文标题、稳定标识符或 PDF, 需要先建立领域记录。
- **不触发**: 用户已确认论文列表只需要获取全文, 应调用论文资源获取。
- **需要**: 领域名、研究方向、论文标题、稳定标识符或 PDF 中至少一种输入; 新领域需要用户确认中文领域名和范围。
- **产出**: 候选阶段维护临时`论文列表候选.md`; 用户确认后生成或更新`论文列表.md`, 并删除候选文件; 输入包含用户提供的单篇 PDF 时, 同时生成对应论文目录中的`原文.pdf`。
- **指导**: `指导/学习/文献检索与收录.md`。

### 论文资源获取

- **做什么**: 为`论文列表.md`中已收录、但用户尚未持有全文的指定论文寻找合法、完整、身份明确的 PDF, 验证身份和版本后规范保存为`原文.pdf`。
- **触发**: 论文已经收录, 用户手里没有 PDF, 需要寻找并获取全文。
- **不触发**: 用户已经提供本地 PDF, 此时应直接调用文献检索与收录; 论文尚未收录; 论文目录中已有已核验的`原文.pdf`且用户只需要翻译。
- **需要**: 已定稿`论文列表.md`中存在目标论文的完整记录; 明确的论文范围, 已核验的论文身份和版本, 用户确认的领域归属与论文中文短标题。
- **产出**: `workspace/<中文领域名>/<论文中文短标题>/原文.pdf`。
- **指导**: `指导/学习/论文资源获取.md`。

### PDF 翻译

- **做什么**: 对论文目录中已有的`原文.pdf`运行 PDF2zh, 成对生成中文译文和双语对照译文。
- **触发**: 用户需要翻译一篇或一批已获取的论文 PDF。
- **不触发**: `原文.pdf`尚不存在或身份未核验; 用户只是需要获取全文或阅读论文。
- **需要**: 一个或多个规范论文目录中的`原文.pdf`; 已按配置指导安装并验证可用的 PDF2zh 环境。
- **产出**: 每个论文目录中同时生成`中文译文.pdf`和`双语对照.pdf`; 失败时不提交任何单边或临时译文。
- **指导**: `指导/学习/PDF 翻译脚本使用.md`。

### 领域概览

- **做什么**: 从论文列表提取领域结构, 建立或更新可复用的知识索引。
- **触发**: 论文列表已定稿, 用户想建立或更新领域全景认知。
- **不触发**: 论文列表尚未建立。
- **需要**: 用户已确认并定稿的`论文列表.md`。
- **产出**: `领域概览.md`。
- **指导**: `指导/学习/领域概览.md`。

## 4. 初始化

### 4.1 新领域

确认中文领域名和范围后创建`workspace/<中文领域名>/`, 只建当前需要的文档。领域有论文收录后, 文件结构遵守`AGENTS.md`中"领域目录"的定义。

### 4.2 孤立论文

用户提供孤立论文标题、稳定标识符或 PDF 时, 先检查它是否已存在于某个领域的`论文列表.md`。由用户确认领域归属和论文中文短标题后调用文献检索与收录: 尚未收录则写入完整记录, 已收录则复用已有记录; 用户已经提供 PDF 时, 由该能力顺手将文件保存为`workspace/<领域>/<论文短标题>/原文.pdf`, 不再调用论文资源获取。只有用户未提供 PDF 且明确需要全文时, 才在收录后调用论文资源获取。领域归属由用户决定, Agent 可从标题和摘要提出候选但不得自行分类。同一论文只保留一份主档案, 其他位置不重复复制。

### 4.3 新研究 Idea

新 Idea 进入 R1。先对齐目录名和用户原意, 然后创建或更新`workspace/<中文研究项目名>/研究想法.md`。其他研究产物在相应阶段开始时创建。

### 4.4 独立能力

正式能力可以独立运行, 并只创建其公开契约声明的产物。未登记或未验证任务若由用户明确要求, 必须先说明缺少稳定专项指导; 低风险一次性任务可按用户原请求继续, 会产生长期产物或影响重大决策时暂停并由用户确认是否继续。

### 4.5 组合目标

"获取并翻译"等多动作目标按各能力的公开契约依次执行。已经完成且本身有效的独立产物可以保留; 后续动作失败时必须停止并说明整体目标尚未完成, 不删除先前有效产物, 也不得把部分完成报告为整体完成。

### 4.6 独立实验仓库

`labs/` 被本仓库忽略, 由用户手动准备。Agent 不执行 clone 或 git init。仓库存在时先读取其 `AGENTS.md` 和工程规范; 代码、配置、日志和原始结果留在独立仓库, 研究项目保存方法与比较边界、共同实验协议、实验组合、分点主张判定、结果摘要和可追溯路径。
