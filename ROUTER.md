# 阶段与能力路由

系统是无状态的——任何时候可以进入任何阶段, 调用任何能力, 唯一的约束是输入是否就绪: 如果某阶段需要的产物还不存在, 告诉用户缺什么, 由用户决定是先补前置还是直接继续

## 1. 为什么这样组织

### 学习: L1 → L2

领域学习从"我想学 X"开始, L1 把它变成一个可执行的方案——读哪些论文, 按什么顺序, 每篇担任什么角色, 产出 `学习目标.md` 和 `论文列表.md`, L2 把论文列表提炼成可复用的知识结构——领域怎么划分, 各方向代表什么论文, 哪些问题已经解决哪些还开着, 产出 `领域全景地图.md`, 两次落盘, 一次聚焦范围, 一次聚焦结构

### 研究写作: R1 → R8

研究从 Idea 到投稿有一条自然的依赖链: 先确认 Idea 成立(R1), 再构造叙事和 Claim(R2), 再设计实验验证(R3), 再把设计映射到代码(R4), 再执行(R5), 再判断结果能支撑到什么程度(R6), 再写稿(R7), 最后检查投稿(R8), 编号反映这个依赖关系——后面阶段需要前面阶段的产物作为输入, 但这不是门禁: 用户可以直接跳到 R7 开始写, 也可以跳过实验阶段

### 能力: 独立于阶段

审稿, 翻译, 绘图, 引用核验——这些事情不绑定任何阶段, 用户在写稿时可以审稿, 在 R1 读论文时也可以翻译, 能力由用户请求独立触发, 做完就结束

### 条目格式

下面每个阶段和能力用统一的 6 字段描述, 让 Agent 拿到用户请求后可以快速判断:

- **做什么** — 一句话说清这个阶段/能力是干什么的
- **触发** — 用户说什么时进入
- **不触发** — 什么情况下不该进(防止 Agent 误判)
- **需要** — 什么输入必须就绪(阶段),  能力无硬性前置则省略
- **产出** — 会生成什么(阶段),  能力则看各自指导
- **指导** — 看哪个专项指导

## 2. 阶段

### L1 学习规划与阅读方案

- **做什么**: 把"我想学 X"变成一个可执行的阅读方案, 生成论文候选列表并由用户确认为最终列表
- **触发**: 用户想系统学习一个领域
- **不触发**: 用户只是要归档一篇已有 PDF, 或审查一个已明确的 Idea
- **需要**: 领域名, 学习目标, 篇数范围
- **产出**: `学习目标.md`, `论文列表.md`
- **指导**: `指导/学习/领域学习论文列表生成.md`

### L2 领域知识地图

- **做什么**: 从论文列表中提取领域结构, 用 K 条目建立可复用的知识索引
- **触发**: 用户想建立领域全景认知, 或已有论文列表需要提炼知识结构
- **不触发**: 用户只需要论文列表本身, 不需要知识提炼
- **需要**: `论文列表.md`, 已核验的论文来源
- **产出**: `领域全景地图.md`
- **指导**: `指导/学习/领域全景地图.md`

### R1 Idea 审查与立项

- **做什么**: 审查原始 Idea 是否成立——检查本质假设, 相近工作, 撞车风险, 反例和可行性, 决定是否建立研究项目
- **触发**: 用户提出一个新的研究 Idea
- **不触发**: 用户已有一个确认立项的研究项目, 只是要继续推进
- **需要**: 原始 Idea 描述, 用户立项标准
- **产出**: `研究想法.md`, `相近工作与撞车分析.md`, `想法硬度审查.md`, `文献证据库.md`, `相关工作矩阵.md`
- **指导**: `指导/研究/研究想法审查与落地.md`，辅助: `指导/研究/本质假设识别.md`, `指导/研究/相近工作与撞车分析.md`, `指导/研究/想法硬度审查.md`

### R2 Story 与核心主张

- **做什么**: 把通过的 Idea 变成论文叙事——选择 Story 线, 拆解为可证伪的 Claim
- **触发**: R1 通过, 或用户已有可靠 Idea 并明确要构造叙事
- **不触发**: Idea 还在审查阶段,  用户只想讨论想法而不构造正式叙事
- **需要**: R1 产物或用户提供的可靠 Idea
- **产出**: `叙事定稿.md`, `核心主张.md`
- **指导**: `指导/研究/构造论文叙事.md`

### R3 实验证明设计

- **做什么**: 为每个需要经验支持的 Claim 设计证伪条件——选什么 Benchmark/Baseline/指标, 什么结果算支持, 什么算推翻
- **触发**: Claim 中有需要实验验证的部分
- **不触发**: 纯理论论文, 或用户明确跳过实验
- **需要**: `叙事定稿.md`, `核心主张.md`
- **产出**: `实验证明设计.md`
- **指导**: `指导/实验/实验证明设计.md`

### R4 实验实施计划

- **做什么**: 把证明设计映射到具体代码仓库——哪段代码对应哪个 Claim, 命令是什么, 接口在哪
- **触发**: 实验设计已确认, 实验仓库已由用户准备在 `labs/`
- **不触发**: `labs/` 中仓库不存在,  实验设计未确认
- **需要**: `实验证明设计.md`, `labs/<仓库目录>/`
- **产出**: `实验编程计划.md`
- **指导**: `指导/实验/实验编程计划.md`

### R5 实验执行与记录

- **做什么**: 按计划运行实验, 保留原始结果, 配置, 命令和失败记录
- **触发**: 实验计划可执行
- **不触发**: 计划不可执行, 或仓库不存在
- **需要**: `实验编程计划.md`, 可运行的实验仓库
- **产出**: `实验记录/`, `失败记录.md`
- **指导**: `指导/实验/创建并执行实验代码.md`

### R6 结果判断与论文口径

- **做什么**: 逐 Claim 判断实验结果——哪些被支持, 哪些被削弱, 哪些不确定, 确定论文允许采用的表述强度
- **触发**: 实验已运行, 有真实结果
- **不触发**: 实验还在运行中,  结果尚未整理
- **需要**: `实验记录/`, `失败记录.md`, `核心主张.md`
- **产出**: `结果分析.md`, 更新后的 `核心主张.md`
- **指导**: `指导/实验/实验结果分析.md`

### R7 论文写作

- **做什么**: 写稿——可以按任意章节顺序, 可以反复修订, 可以随时调用翻译, 绘图, 审稿等能力
- **触发**: 有足够的 Story, Claim 和证据支撑写作, 用户也可以在任何时候直接说"开始写"
- **不触发**: 没有最低要求——用户可以从空白开始写引言, 也可以等所有实验完成再写
- **需要**: 无硬性前置, 已有 Story/Claim/证据时写得更快, 没有时也可以写框架
- **产出**: 中文或英文 Tex 稿件
- **指导**: `指导/写作/论文写作编排.md`

### R8 投稿准备

- **做什么**: 终稿检查——核验投稿规则, 引用完整性, 匿名合规, Tex 编译, 汇总投稿材料
- **触发**: 稿件稳定, 用户明确了目标 venue
- **不触发**: 稿件还在大幅修改中,  venue 未确定
- **需要**: 稳定稿件, 目标 venue/年份/track
- **产出**: 更新后的各项检查记录, 投稿材料
- **指导**: `指导/投稿/投稿准备.md`

阶段之间没有强制顺序, R3-R6 面向经验研究, 非实验论文可跳过, 编号只反映典型依赖关系（后面需要前面的产物），不是流程门禁

## 3. 能力

### 文献检索与合法全文定位

- **做什么**: 为指定论文寻找合法, 完整, 身份明确的 PDF/HTML/Tex, 记录来源和获取状态
- **触发**: 用户需要获取某篇论文的全文
- **不触发**: 用户已提供本地文件且不需要额外检索,  需要绕过付费墙或登录
- **指导**: `指导/学习/候选文献检索.md`, `指导/学习/论文资源获取.md`, `指导/学习/合法全文定位.md`

### 论文归档与 PDF 翻译

- **做什么**: 把用户确认的论文 PDF 归档到论文目录, 并用 PDF2zh 生成中文和双语译文
- **触发**: 用户有论文 PDF 需要归档和/或翻译
- **不触发**: 用户只是想阅读一篇论文而不归档,  论文身份或来源未核验
- **指导**: `指导/学习/论文处理工具使用.md`, `指导/学习/论文处理工具配置.md`

### 自动发现研究想法

- **做什么**: 从领域论文中自动发现研究缺口和候选 Idea
- **触发**: 用户想在某个领域自动扫描研究机会
- **不触发**: 用户已有明确的 Idea,  领域论文列表尚未建立
- **指导**: `指导/研究/自动发现研究想法.md`

### 领域地图知识更新

- **做什么**: 向已有 `领域全景地图.md` 追加或更新 K 条目
- **触发**: 有新论文或新证据需要纳入领域知识索引
- **不触发**: 地图尚未创建(应先完成 L2)
- **指导**: `指导/学习/领域全景地图.md`

### 图表规划与生成

- **做什么**: 规划论文需要哪些图表, 每张图对应哪个 Claim, 生成图表代码和图像
- **触发**: 用户需要为论文设计或生成图表
- **不触发**: 论文不需要图表,  没有可展示的数据或概念
- **指导**: `指导/绘图/论文图表规划.md`，辅助: `指导/绘图/论文架构图设计.md`, `指导/绘图/实验图表类型推荐.md`

### 中文稿英译

- **做什么**: 把中文 Tex 稿件翻译为英文, 保留技术事实和引用
- **触发**: 用户有已完成的中文稿件需要翻译
- **不触发**: 稿件还在写作中(应先完成再翻译),  用户只需要润色而非翻译
- **指导**: `指导/写作/中文稿翻译为英文Tex.md`

### 文本修订

- **做什么**: 对稿件执行局部修改——润色, 缩写, 去 AI 痕迹, 修正措辞
- **触发**: 用户指定了需要修订的文本段落和修订目标
- **不触发**: 修订范围不明确,  修订会改变技术事实或 Claim 强度
- **指导**: `指导/写作/论文文本修订.md`

### 写作辅助

- **做什么**: 中文润色与去 AI 痕迹、英文 Tex 润色/扩写/缩写、论文逻辑检查、标题生成、图表标题、实验数据分析与段落生成——这些是写作过程中的局部辅助操作
- **触发**: 用户指定了文本段落和处理目标（如"帮我把这段中文润色"、"检查这章的逻辑"）
- **不触发**: 整篇翻译(用中文稿英译)；审稿(用综合论文审稿)；修订范围不明确
- **指导**: `指导/写作/中文学术表达润色.md`, `指导/写作/去除中文AI写作痕迹.md`, `指导/写作/去除英文AI写作痕迹.md`, `指导/写作/英文Tex润色.md`, `指导/写作/英文Tex扩写.md`, `指导/写作/英文Tex缩写.md`, `指导/写作/论文逻辑检查.md`, `指导/写作/论文标题生成与修订.md`, `指导/写作/生成图表标题.md`, `指导/写作/实验数据分析与段落生成.md`

### 综合论文审稿

- **做什么**: 模拟审稿——从贡献, 方法, 实验, 写作等维度审查稿件质量
- **触发**: 用户需要审稿意见来改进稿件
- **不触发**: 用户只需要格式检查(用投稿规则核验),  稿件还在起草中
- **指导**: `指导/审稿/综合论文审稿.md`

### 投稿规则核验

- **做什么**: 按目标 venue 的官方要求检查模板, 篇幅, 匿名方式, 强制材料等
- **触发**: 目标 venue 已确定, 需要核验合规性
- **不触发**: venue 未确定,  稿件未成形
- **指导**: `指导/投稿/投稿规范检查.md`

### 参考文献与引用核验

- **做什么**: 逐条核验引用——DOI, 页码, 版本, 预印本/正式版差异
- **触发**: 稿件引用列表已基本稳定
- **不触发**: 引用还在频繁增删
- **指导**: `指导/写作/参考文献与引用核验.md`

### Tex 成稿工程验收

- **做什么**: 检查 Tex 是否能无错误编译, 产出最终 PDF
- **触发**: 稿件内容完成, 需要验证可编译性和最终输出
- **不触发**: 稿件还在写作中
- **指导**: `指导/写作/Tex成稿工程验收.md`

### 稿件匿名化检查

- **做什么**: 检查稿件 PDF, Tex 源码, 文件名, 补充材料中是否有作者身份泄露
- **触发**: 目标 venue 要求匿名投稿
- **不触发**: venue 不要求匿名,  稿件未成形
- **指导**: `指导/投稿/稿件匿名化检查.md`

能力之间没有调用栈——每次用户请求独立触发, 做完就结束, 同一个能力可以在写作过程中多次使用（例如反复审稿）

## 4. 初始化

### 4.1 新领域

确认中文领域名和范围后创建 `workspace/<中文领域名>/`, 只建当前需要的文档

### 4.2 孤立论文

确认领域归属和论文中文短标题后, 归档到 `workspace/<领域>/<论文短标题>/`, 领域归属由用户决定, Agent 可从标题和摘要提出候选但不得自行分类, 同一论文只保留一份主档案, 其他位置通过路径引用

### 4.3 新研究 Idea

Idea 先在对话中审查, 立项后才创建 `workspace/<中文研究项目名>/`, 否决且用户停止时任务终止

### 4.4 自动发现候选

候选保存在当前领域或发现目录中, 用户选中后进入 R1 预审, 立项前不创建独立目录

### 4.5 独立能力

审稿, 翻译, 修订等能力可以独立运行, 不补建无关阶段文件, 涉及论文归档时遵守 4.2

### 4.6 组合目标

"归档并翻译"等多动作目标视为整体: 前置条件全部满足后再落盘, 任一动作无法完成时等待用户指示

### 4.7 独立实验仓库

`labs/` 被本仓库忽略, 由用户手动准备, Agent 不执行 clone 或 git init, 仓库存在时先读取其 `AGENTS.md` 和工程规范, 代码和工程产物留在独立仓库, 研究项目只保存计划和分析, 并引用仓库路径和 commit
