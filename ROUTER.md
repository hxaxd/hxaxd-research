# 阶段与能力路由

> 暂停使用：这是保留的旧阶段路由草案。当前仓库只运行论文学习工作台，Agent 入口以 `AGENTS.md` 和 `API.md` 为准。

任务由用户当前输入触发。缺少关键前置时，说明缺口和继续风险；低风险任务可按明确假设继续，重大决定或长期产物等待用户确认。

## 0. 共同协议

### 证据与判断

- 论文、引用、实验、外部事实和投稿规则必须可追溯；优先核对原始论文、官方文档和真实产物，未确认内容标为未核验、推断或待验证。
- 研究文档在使用文献的位置直接给出 DOI、arXiv 或官方论文链接；没有可靠链接时写明可识别标题和 `链接待补`，相关事实同步标为待核验。
- 结论力度与证据一致，冲突、反例、失败和不确定性不得隐藏。论文创新按问题、机制、知识或能力边界的真实变化判断。
- 主要指标、随机种子和结果集合按预先确认的实验口径处理，不通过换名、修辞或选择性报告强化结论。
- Agent 负责检索、规整、反驳、执行和检查；研究方向、Story、实验范围、结果口径、阶段跳过和投稿决定由用户确认。

### 权威位置

- 学习项目、论文记录、筛选状态和 PDF 资源以工作台服务器为唯一权威。Agent 使用当前接口，不直接读写 SQLite、服务器目录或等价 Markdown。
- 临时 JSONL 只用于一次批量提交，放在 `tmp/` 并在提交后删除。
- 研究进度由 `workspace/` 中的真实文档体现，不维护影子进度文件。
- 文件复制、下载上传、翻译、编译和批量检查等机械任务尽量不向模型回显正文；只有内容理解任务才按最小范围读取正文。

### 执行与暂停

1. 读取目标对应的服务器记录或研究文档，再读取专项指导。
2. 检查输入、服务器状态和已有产物；能安全继续时说明假设，高风险缺口暂停。
3. 在已确认范围内执行、边做边检查，并在结束时回读真实记录或产物。
4. 用户确认后的研究判断发生实质变化时重新对齐；局部补充和纠错可以直接完成。

关键输入冲突或缺失、关键事实无法核验、用户要求超出现有证据，或未验证任务将影响重大决定时暂停并询问用户。

学习侧候选论文是例外：核验完成后立即以 `discovered` 状态提交服务器，用户在页面查看后再通过对话要求更新为 `included`、`excluded` 或 `archived`。

## 1. 路由概览

学习由两项可重复调用的能力组成：文献检索与收录、论文资源获取。PDF 翻译由用户在页面启动；领域概览尚未接入。

研究主线包含六个典型阶段：

1. R1 Idea 预审与立项
2. R2 论文叙事与分点核心主张
3. R3 实验证明
4. R4 实验执行与结果交付
5. R5 论文写作
6. R6 投稿准备与返修

阶段编号表示常见依赖，不限制用户直接进入后续阶段；缺少前置时按共同协议说明风险。

## 2. 研究阶段

### R1 Idea 预审与立项

- **进入**: 用户提出新 Idea，或要求重审未定稿 Idea。
- **不进入**: 已有确认的 `叙事定稿.md`，且用户只是继续实验、写作或投稿。
- **需要**: 原始描述；创建项目时还需确认中文目录名。
- **产出**: 将用户原意写成正式研究想法和四段式核心主张的 `workspace/<项目名>/研究想法.md`，以及对话中的撞车分析和硬度审查。
- **指导**: `指导/研究/研究想法审查与落地.md`；参考 `指导/参考/研究想法.md`。

### R2 论文叙事与分点核心主张

- **进入**: R1 已对齐并确认继续，或用户明确接受跳过 R1 的风险。
- **不进入**: Idea 尚未写入 `研究想法.md`，或用户仍在讨论 Idea 本身。
- **需要**: `研究想法.md`、已知任务与方法信息，以及必要的领域事实和相近工作。
- **产出**: 固定中心叙事、任务、失败、原因、认识、设计关系、边界和承重主张的 `叙事定稿.md`。
- **指导**: `指导/研究/构造论文叙事.md`；参考 `指导/参考/叙事定稿.md`。

### R3 实验证明

- **进入**: 分点主张中存在需要经验支持的内容。
- **不进入**: 纯理论工作，或用户明确决定不做实验。
- **需要**: `叙事定稿.md`中的论文事实、设计关系和主张，以及完整方法、基线、Benchmark、指标和资源边界。
- **产出**: 固定方法身份、比较边界、主实验、消融、迁移、效率、诊断等实验组合及共同协议的 `实验证明.md`。
- **指导**: `指导/实验/实验证明.md`；参考 `指导/参考/实验证明.md`。

### R4 实验执行与结果交付

- **进入**: `实验证明.md` 已确认，且用户已把实验仓库放入 `labs/<仓库目录>/`。
- **不进入**: 实验仓库不存在，或 R3 仍缺少会改变研究结论的方法、条件、判据或结果责任。
- **需要**: `研究想法.md`、`叙事定稿.md`、`实验证明.md` 和实验仓库。
- **产出**: 实验仓库中的代码、配置、运行清单、日志、原始与结构化结果，以及按实验编号回填后的 `实验证明.md`。
- **指导**: `指导/实验/实验执行.md`。

### R5 论文写作

- **进入**: 用户要求维护正式论文，且 `叙事定稿.md` 已存在；经验性结论必须已有真实结果，除非用户接受只写当前有证据的部分。
- **不进入**: 仍在构造叙事、只需设计或运行实验，或只是一次性文字润色。
- **需要**: 叙事、主张、相关实验结果、原始文献和已有稿件。
- **产出**: 唯一权威 TeX 稿件，默认主入口为 `论文稿件.tex`，以及实际依赖的图表、`references.bib`、附录和编译产物。
- **指导**: `指导/写作/论文写作.md` 及其按内容路由的章节指导；环境见 `指导/写作/TeX 环境配置.md`。

### R6 投稿准备与返修

- **进入**: 用户指定投稿场所与轮次并要求准备材料，或提供真实审稿意见、编辑决定或终稿要求。
- **不进入**: 只是在写论文内容，尚未确定场所却要求假定具体格式，或只讨论投稿策略。
- **需要**: R5 权威稿件、官方作者指南、作者与声明信息；返修时还需真实审稿材料和相关证据。
- **产出**: 目标场所要求的主稿、源文件包、补充材料、声明和清单，以及需要时的审稿回复、修订稿和终稿。
- **指导**: `指导/投稿/投稿准备.md`。

## 3. 独立能力

### 科研配图

- **目标**: 根据论文主张和真实数据生成可直接使用的概念图、方法图、结果图或定性图。
- **进入**: 用户要求新建、修改、重构或检查科研图件，或研究阶段中出现必须用图解释的关系。
- **不进入**: 只需数据分析、装饰性图片或文字构想，不需要真实图件。
- **需要**: 图件要支持的主张、真实数据或方法关系、已有稿件，以及目标载体要求。
- **产出**: 最终图件及必要的源数据、脚本、可编辑图源、图注和稿件引用。
- **指导**: `指导/能力/科研配图.md`。

### 学术审阅与反向论证

- **目标**: 对 Idea、叙事、主张、实验或稿件进行证据驱动的反向审阅。
- **进入**: 用户要求撞车或硬度审查、模拟审稿、攻击主张、检查实验闭环，或基于新证据复审。
- **不进入**: 只需要摘要、精读、语言润色或直接修改，没有独立判断需求。
- **需要**: 被审对象的真实版本、相关文献或实验；复审时还需原审阅和用户的新证据。
- **产出**: 默认在对话中给出准确复述、最强正反理由、决定性问题、严重程度和改变结论所需证据。
- **指导**: `指导/能力/学术审阅与反向论证.md`。

### 文献检索与收录

- **目标**: 发现并核验论文，补全当前服务器契约的全部字段后提交目标项目。
- **进入**: 用户要学习或补充一个领域、寻找论文，或给出尚未入库的标题、标识符或 PDF。
- **不进入**: 论文记录已经完整存在，用户只需取得或上传全文。
- **需要**: 可识别的研究范围或论文身份、可访问的后端，以及用户确认的目标项目。
- **产出**: 新候选默认以 `discovered` 状态进入服务器；用户筛选后按明确指令更新状态。不生成候选 Markdown。
- **指导**: `指导/学习/文献检索与收录.md`。

### 论文资源获取

- **目标**: 为服务器中已有记录的论文取得合法完整 PDF，核验后上传为 `original`。
- **进入**: 论文记录存在但没有原文，或用户明确要求替换版本；尚无记录时先调用文献检索与收录。
- **不进入**: 已有正确原文且未要求替换，或用户只想在页面启动翻译。
- **需要**: 目标记录、`paper_id`、版本信息、可访问的后端，以及可选的用户本地 PDF。
- **产出**: 服务器可读取的 `original` 资源，不在 `workspace/` 保存长期副本。
- **指导**: `指导/学习/论文资源获取.md`。

### 暂不进入 Agent 路由

- **PDF 翻译**: 用户在论文页面启动，Agent 不安装、调用或维护翻译脚本。
- **领域概览**: 尚未接入工作台；现有指导仅作为未来迁移草案。

## 4. 初始化与组合

### 新学习项目

不创建 `workspace/` 领域目录。先读取服务器项目列表并确认中文名称和范围，再按需创建项目。

### 孤立论文

先按稳定键和论文身份在目标项目查重；目标项目由用户决定，Agent 只能提出候选归类。无记录时先完成文献检索与收录，有记录时复用；需要全文时再调用论文资源获取。同一项目只保留一条论文记录和一组资源。

### 新研究 Idea

进入 R1，确认目录名和用户原意后创建或更新 `workspace/<项目名>/研究想法.md`。

### 组合目标

按依赖顺序执行。例如“检索并获取全文”先提交完整论文记录，再上传原文。翻译不是 Agent 动作。后续动作失败时保留已经有效的记录，但不得把部分完成报告为整体完成。

### 独立实验仓库

`labs/` 由用户手动准备，Agent 不执行 clone 或 git init。进入仓库后先读取其 `AGENTS.md`；代码和原始结果留在实验仓库，研究项目只保存研究判断、结果摘要和可追溯路径。
