# 阶段与能力路由

系统是无状态的——任何时候可以进入任何阶段或能力, 唯一约束是输入是否就绪。若需要的产物还不存在, 说明缺什么和继续风险, 由用户决定补前置还是接受风险继续。

## 1. 为什么这样组织

### 学习: 能力组合, 而非阶段

领域学习由一组可反复调用的能力组成: 文献检索与归档发现论文并写入`论文列表.md`, 领域概览从论文列表提炼知识结构并写入`领域概览.md`, 论文资源获取下载全文, 论文归档与 PDF 翻译生成译文。新论文加入后可随时重新调用这些能力更新已有产物, 没有固定阶段边界。

### 研究: 新版 R1 到 R6

研究主线只保留当前确认的六个阶段:

1. R1 Idea 预审与立项: 把用户随口描述的 Idea 对齐并落盘为`研究想法.md`; 撞车分析和硬度审查在对话中完成, 不单独落盘。
2. R2 论文叙事与分点核心主张: 基于`研究想法.md`展开叙事, 并拆出分点核心主张, 产出`叙事定稿.md`。
3. R3 实验证明: 为需要经验支持的分点主张写出分点证明方案和结果交付要求, 产出`实验证明.md`。
4. R4 实验执行与结果交付: 在用户手动准备的`labs/<仓库目录>/`中实现、运行和保存结果, 并按 R3 规定回填结果状态和产物路径。
5. R5 论文写作: 当前只是未验证占位, 没有稳定专项指导。
6. R6 投稿准备: 当前只是未验证占位, 没有稳定专项指导。

### 文献证据库: 文献身份索引

`文献证据库.md`是研究项目内的文献身份索引。它只负责说明每篇文献自己是谁: 文献信息、核心主张、主要方法和主要结论。

研究想法、叙事、实验证明等文档如果使用某篇文献, 在自己的句子后用`[文献:E1]`引用; 同时`文献证据库.md`中必须有对应的`## E1 文献标题`条目。多篇文献写作`[文献:E1; E2]`。引用关系由引用所在文档负责表达, 不写回文献证据库。

### 能力: 独立触发, 做完即结束

能力由用户请求独立触发, 不伪造成阶段完成。当前正式登记的能力只保留已相对稳定的学习与论文处理能力。

### 条目格式

下面每个阶段和能力用统一字段描述:

- **做什么** — 一句话说明目标
- **触发** — 用户说什么时进入
- **不触发** — 什么情况下不该进入
- **需要** — 必须就绪的输入
- **产出** — 会生成或更新什么
- **指导** — 读取哪个专项指导; 若无稳定指导则明确写明

## 2. 阶段

### R1 Idea 预审与立项

- **做什么**: 对齐用户原始 Idea, 写成正式自然语言研究想法和四段式核心主张; 随后推荐并在对话中执行撞车分析和硬度审查。
- **触发**: 用户提出新的研究 Idea, 或要求重新审查尚未定稿的 Idea。
- **不触发**: 用户已有确认后的`叙事定稿.md`并只是继续实验、写作或投稿。
- **需要**: 原始 Idea 描述; 若要创建新研究目录, 需要可识别的中文项目名或用户确认的目录名。
- **产出**: `研究想法.md`。撞车分析和硬度审查只在对话中形成结论, 不单独落盘。
- **指导**: `指导/研究/研究想法审查与落地.md`, 参考: `指导/参考/研究想法.md`。

### R2 论文叙事与分点核心主张

- **做什么**: 根据`研究想法.md`把核心主张展开成论文叙事, 并拆成可用于实验和写作的分点主张。
- **触发**: R1 对齐完成且用户确认继续构造叙事, 或用户已有可靠 Idea 并明确接受跳过 R1 对话审查的风险。
- **不触发**: Idea 尚未写成`研究想法.md`, 或用户仍在讨论 Idea 本身。
- **需要**: `研究想法.md`; 必要的领域事实、相近工作和文献来源。
- **产出**: `叙事定稿.md`; 按需更新全周期`文献证据库.md`。
- **指导**: `指导/研究/构造论文叙事.md`, 参考: `指导/参考/叙事定稿.md`, `指导/参考/文献证据库.md`。

### R3 实验证明

- **做什么**: 对`叙事定稿.md`中需要经验支持的分点主张声明如何证明或推翻, 并写清 R4 需要回填的结果交付要求。
- **触发**: 分点主张中存在需要经验支持的内容。
- **不触发**: 纯理论工作, 或用户明确跳过实验。
- **需要**: `叙事定稿.md`中的分点核心主张。
- **产出**: `实验证明.md`。
- **指导**: `指导/实验/实验证明.md`, 参考: `指导/参考/实验证明.md`。

### R4 实验执行与结果交付

- **做什么**: 让 coding agent 在用户手动准备的独立实验仓库中, 以 R3 的证明目标为最终目标自由实现、运行、保存结果并回填交付状态。
- **触发**: `实验证明.md`已确认, 且实验仓库已由用户准备在`labs/<仓库目录>/`。
- **不触发**: `labs/`中仓库不存在, 或 R3 仍未说明需要交付哪些结果、日志和结构化数据。
- **需要**: `研究想法.md`, `叙事定稿.md`, `实验证明.md`, `labs/<仓库目录>/`。
- **产出**: 独立实验仓库中的代码、配置、日志、原始结果和结构化汇总; 按 R3 既定交付项回填`实验证明.md`中的实际结果、状态和产物路径。
- **指导**: `指导/实验/实验执行.md`。

### R5 论文写作（未验证占位）

- **做什么**: 暂无稳定流程。可在用户明确要求时自由协助写作, 但必须说明当前没有经过验证的仓库级专项指导。
- **触发**: 用户明确要求写论文、写章节、润色或翻译稿件。
- **不触发**: 用户只是推进 Idea、叙事、实验证明或实验执行。
- **需要**: 用户指定写作目标和可用材料。
- **产出**: 由用户请求决定; 不作为稳定阶段产物承诺。
- **指导**: 无稳定专项指导。

### R6 投稿准备（未验证占位）

- **做什么**: 暂无稳定流程。可在用户明确要求时自由协助检查投稿材料, 但必须说明当前没有经过验证的仓库级专项指导。
- **触发**: 用户明确要求投稿准备、匿名检查、投稿规则核验或终稿检查。
- **不触发**: 稿件仍在研究或实验阶段, 或用户没有明确目标 venue。
- **需要**: 稳定稿件、目标 venue/年份/track 和用户指定的检查目标。
- **产出**: 由用户请求决定; 不作为稳定阶段产物承诺。
- **指导**: 无稳定专项指导。

阶段之间没有强制门禁。编号只反映典型依赖关系; 用户可以直接进入任意阶段, 但缺失输入时必须说明风险。

## 3. 能力

### 文献检索与归档

- **做什么**: 为领域名、研究方向、论文标题或一篇 PDF 发现论文, 策划列表, 并将完整记录归档到论文列表。
- **触发**: 用户要系统学习一个领域, 或提供一篇 PDF 需要了解背景并归档。
- **不触发**: 用户已确认论文列表只需要获取全文, 应调用论文资源获取。
- **指导**: `指导/学习/文献检索与归档.md`。

### 论文资源获取

- **做什么**: 为论文列表中的指定论文寻找合法、完整、身份明确的 PDF/HTML/Tex, 验证身份和版本, 记录来源和获取状态。
- **触发**: 用户需要获取某篇论文全文。
- **不触发**: 用户已提供本地文件且不需要额外检索; 需要绕过付费墙或登录。
- **指导**: `指导/学习/论文资源获取.md`。

### 论文归档与 PDF 翻译

- **做什么**: 把用户确认的论文 PDF 归档到论文目录, 并用 PDF2zh 生成中文和双语译文。
- **触发**: 用户有论文 PDF 需要归档和/或翻译。
- **不触发**: 用户只是想阅读一篇论文而不归档; 论文身份或来源未核验。
- **指导**: `指导/学习/论文处理工具使用.md`, `指导/学习/论文处理工具配置.md`。

### 领域概览

- **做什么**: 从论文列表提取领域结构, 建立或更新可复用的知识索引。
- **触发**: 论文列表已定稿, 用户想建立或更新领域全景认知。
- **不触发**: 论文列表尚未建立。
- **指导**: `指导/学习/领域概览.md`。

## 4. 初始化

### 4.1 新领域

确认中文领域名和范围后创建`workspace/<中文领域名>/`, 只建当前需要的文档。领域有论文归档后, 索引文件结构遵守`AGENTS.md`中"领域目录"的定义。

### 4.2 孤立论文

确认领域归属和论文中文短标题后, 归档到`workspace/<领域>/<论文短标题>/`, 论文目录内的 PDF 命名遵守`AGENTS.md`中"论文目录"的规则。领域归属由用户决定, Agent 可从标题和摘要提出候选但不得自行分类。同一论文只保留一份主档案, 其他位置通过路径引用。

### 4.3 新研究 Idea

新 Idea 进入 R1。先对齐目录名和用户原意, 然后创建或更新`workspace/<中文研究项目名>/研究想法.md`; 不等待全部对话审查结束才落盘。R1 结束前不预建`叙事定稿.md`、`实验证明.md`或写作/投稿文件。

### 4.4 自动发现候选

自动发现研究想法暂不作为正式能力登记。用户明确要求时, 先说明缺少稳定指导, 再由用户决定是否继续; 候选被用户选中后进入 R1。

### 4.5 独立能力

正式能力可以独立运行, 不补建无关阶段文件。未登记能力若由用户明确要求, 先说明缺少稳定指导, 再由用户决定是否继续。

### 4.6 组合目标

"归档并翻译"等多动作目标视为整体: 前置条件全部满足后再落盘, 任一动作无法完成时等待用户指示。

### 4.7 独立实验仓库

`labs/`被本仓库忽略, 由用户手动准备。Agent 不执行 clone 或 git init。仓库存在时先读取其`AGENTS.md`和工程规范; 代码、配置、日志和原始结果留在独立仓库, 研究项目只保存研究目标、实验证明、结果交付摘要和可追溯路径。
