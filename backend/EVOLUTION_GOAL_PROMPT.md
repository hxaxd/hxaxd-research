# 长期实施目标提示词

完整实施 `backend/EVOLUTION_PLAN.md` 中定义的文献工作台前三阶段演化，直到计划中的全部功能、迁移、前端体验、失败恢复、测试和真实数据验收均完成。不要停留在调查、设计、脚手架、接口占位、部分后端实现或阶段性报告；只要仍有安全且属于计划范围的工作可以推进，就持续执行下一项。

开始时阅读仓库根目录 `AGENTS.md` 和 `backend/EVOLUTION_PLAN.md`，检查当前工作树并保留所有已有改动。当前未提交的智能体提示词集中化重构是已完成的前置工作，不得丢弃；先验证它，再作为独立提交落下。后续按可审阅的纵向功能切片工作，并在每个通过验证的稳定边界创建本地 Git 提交；不要推送远端。

必须完成以下整体结果：

1. 打通项目、智能体检索、候选审阅、用户筛选、元数据建议、资源建议、任务执行、附件打开和阅读的完整日常工作流。
2. 保留文献检索、元数据补全、资源获取和冲突分析四类智能体任务，并为每类任务实现真实、类型化、受作用域限制的领域工具。不得隐藏未完成能力来规避实现。
3. 建立独立 ChangeSet 领域。智能体只产生可审阅的结构化建议；用户逐项批准后，后端以版本校验、幂等、事务和审计保证执行。智能体不得修改用户筛选状态、批准自己的建议、直接操作数据库或直接访问附件目录。
4. 建立书目字段级修订历史，正确区分元数据修订与预印本、会议版、期刊版等真实文献版本。
5. 建立 Document、DocumentBlock、BlockTranslation、Annotation 和 ReadingState 等语义阅读模型，并通过公开领域服务访问。
6. 优先复用固定版本 PDF2zh/BabelDOC 的版面识别、阅读顺序、段落切分和实际可用的 OCR 中间结果。首先完成有测试证据的能力探针，明确区分 OCR、普通文本提取和版面分析。优先使用公开 Python API、翻译器回调或类型化子进程桥接，不得解析 TUI、日志或临时文件名来推断业务状态。
7. 将 PDF2zh/BabelDOC 的结构化段落映射为稳定块 ID；禁用其零散大模型翻译后，优先把本系统生成的整篇结构化译文按块回注，让它继续完成版面和 PDF 渲染。如果固定版本没有可靠回注能力，如实记录并采用计划中的降级方案，不以脆弱补丁伪造成功。
8. 建立可替换的 TranslationProvider，第一版实现 DeepSeek。普通论文默认一次请求提交全部带稳定 ID 的结构化段落，同时返回译文、统一术语和语义角色；超过安全预算、输出截断、缺块、重复块、公式或引用占位符损坏时，自动按章节降级并保持全局上下文。翻译是确定性持久任务，不通过通用智能体运行。
9. 以“原始 PDF + 结构化双语内容”为主要阅读体验，完成目录、块与页码联动、原译文切换或并排、全文搜索、书签、阅读进度、文本锚点、摘录、问题、主张、方法、结果、局限和自由标签。翻译 PDF 是可选导出，不阻塞主体交付。
10. 前端与后端同等重要。每个新后端能力必须在同一交付批次中具有现代、可发现、状态完整的前端交互，不接受“后端完成、前端以后再补”。
11. 所有主流程必须支持纯触屏和平板完整使用。实现桌面、平板横屏、平板竖屏响应式布局；不依赖悬停、右键或精细拖拽；重要触控目标不小于 44×44 CSS 像素；支持虚拟键盘、安全区域、横竖屏、触屏文本选择和所有拖拽操作的点按替代方案。
12. 提供丰富且结构清晰的设置，包括阅读、双语布局、PDF、翻译提供者与风格、智能体、任务、工具、Zotero、快照、完整性、局域网访问和设备管理。
13. 默认由电脑运行后端，平板通过受保护的局域网浏览器或 PWA 完整使用。局域网监听必须显式启用；首次设备访问使用短期配对码建立可撤销会话；凭据不得进入 URL、日志或任务事件；限制允许来源并防止跨站请求伪造；不实现不必要的公网账户系统。
14. 为关键前端页面和领域组件补齐交互测试、浏览器端到端测试、平板触屏测试和关键视口视觉回归。必须覆盖长列表、长论文、虚拟键盘、网络短暂断开、失败、取消和重试。
15. 按计划实施影子数据库迁移、快照、完整性验证和原子切换。不得原地破坏活动数据库，不得手工读取或修改 `backend/data/`。必须保留现有文献、项目关系、附件、内容哈希和筛选决定。

始终遵守这些架构约束：

- 产品代码只位于 `backend/` 和 `frontend/`。
- 数据库和文件系统只由后端领域服务访问。
- 前端、智能体、翻译提供者和集成适配器只调用类型化命令。
- 权限、审批、状态约束、幂等和审计由代码保证，不依赖提示词。
- 多输出操作先在任务暂存区完成并校验，再原子登记。
- OpenAPI 是前端字段契约的唯一权威。
- 不为旧接口或旧前端保留兼容层，但必须保护现有数据。
- 不提前扩展研究写作层、OCR 自研或跨论文知识图谱，除非它们成为完成当前计划不可避免的依赖。

每个功能都必须覆盖成功、失败、取消、重复请求、陈旧版本、服务重启和审计证据。优先解决根因，不通过新增并行文档、临时脚本、直接数据库访问、脆弱日志解析或前端假数据绕过领域模型。

在执行过程中自行做安全且不改变产品方向的实现判断。只有遇到计划无法推导、会显著改变产品语义、需要新的外部凭据或需要不可逆外部操作时才请求用户决策；不要因为工作量大、测试失败、上下文长度或一个里程碑完成而停止。

完成前必须：

- 运行并通过后端 `uv run pytest` 和 `uv run ruff check .`。
- 运行并通过前端 `npm test`、`npm run typecheck`、`npm run lint` 和 `npm run build`。
- 通过浏览器端到端测试完成全部主流程和触屏平板流程。
- 通过公开快照与完整性接口证明真实数据迁移无丢失。
- 回读并验证任务、变更集、附件、翻译、注释、设备会话和审计结果。
- 确认工作树只包含有意变更、运行数据和构建产物未被提交。
- 创建最后一个本地提交，并保持工作树干净。

只有当 `backend/EVOLUTION_PLAN.md` 的全局完成标准全部被可复现证据满足，且不存在计划范围内的未完成工作时，才报告目标完成。最终报告应列出交付结果、关键架构、数据迁移证明、测试结果、提交记录和仍然存在但明确不属于本目标的限制。
