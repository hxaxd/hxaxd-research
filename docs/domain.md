# 学习工作台领域语义

## Project 与 ProjectPaper

Project 只描述学习范围。ProjectPaper 表示一篇论文在某个项目中的筛选状态和项目判断；同一 Paper 可以进入多个项目，并拥有不同状态、角色、相关性和阅读重点。

状态含义：

- `discovered`：已识别候选，等待用户判断。
- `included`：用户明确收录；必须有可靠身份和项目相关性。
- `excluded`：用户明确不留，但继续用于去重。
- `archived`：历史上有价值，当前不在主流程。

角色是多选标签，不是论文的全局事实。

## Paper

Paper 保存可核验的客观事实：官方标题、完整作者、摘要、发表信息、公开标识符和链接。论文身份在整个工作区唯一，由后端根据规范化 DOI、arXiv、OpenReview 等标识符派生；调用方不生成稳定键。

发现候选时只需足以识别论文的最小事实。无法核验的字段保持为空，不用猜测值补齐。

## Resource

Resource 用三个独立维度描述论文文件：

- `format`：PDF 或 TeX。
- `representation`：原文、中文或双语。
- `origin`：出版社、预印本、作者、用户、生成或旧数据。

新文件总是创建新记录。`preferred` 决定同类资源的默认版本，不覆盖历史。生成资源通过 `parent_resource_id` 和 `job_id` 追溯输入与转换任务。

TeX 资源是完整源码包；默认阅读资源仍是 PDF。官方 PDF 与 TeX 编译 PDF 可以并存，默认优先官方来源，用户可以切换首选。

## Job 与能力

Job 表示编译或翻译。任务记录输入资源、工具版本、状态、进度、错误和输出资源；失败任务不得改变已有首选资源。

`supported` 表示平台实现了能力，`ready` 表示所需工具和配置当前可用。调用方先看工作区能力，再决定提交任务还是报告缺口。
