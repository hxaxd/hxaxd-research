# 论文资源获取

## 目标

为已经登记的论文取得合法、完整且身份一致的 PDF，核验后作为原文交给学习工作台管理。服务器调用、上传和回读方式统一见根目录 `API.md`。

## 来源顺序

1. arXiv 等正式预印本平台
2. 出版方、会议、期刊或 OpenReview 的开放全文
3. 作者主页或机构仓储公开的作者版本

非官方镜像、搜索缓存和聚合站只能作为线索。来源合法性或版本可靠性存疑时说明风险，不把便利性当作证据。

最终资源必须是完整主论文 PDF。补充材料、TeX、HTML、摘要、幻灯片或介绍页不能代替主论文，也不能转换后冒充官方 PDF。

## 核验

### 文件

- 文件以 `%PDF` 开头，结构、页数和大小合理，可被 PDF 工具正常读取
- 不是错误页、登录页或其他格式改名
- 主文完整；附录或补充材料是否包含应与来源版本一致

### 身份

- 标题与记录一致，只有可解释的版本或标点差异
- 作者集合和顺序没有无法解释的冲突
- DOI、arXiv ID、OpenReview ID 或正文元数据与目标记录对应

### 版本

- 内容等价时优先来源稳定、公开清楚的版本
- 正式版本、作者接受稿和预印本存在实质差异时告诉用户
- 已有正确原文时默认复用，不静默替换

## 文件边界

- 网络下载使用操作系统临时目录，只为核验和上传存在。
- 用户本地文件可以直接读取，但不得移动、删除或覆盖。
- 上传结束或失败后清理 Agent 创建的临时文件。
- 不在仓库、学习项目或后端数据目录中建立中间副本。
- 不自动获取用户未指定的其他论文，不在获取后自动启动翻译。

## 完成条件

- 请求的每篇论文已有可读取的原文，或准确报告无法获取的原因。
- 论文身份、来源和版本已经核验。
- 上传结果和读取结果已经回读确认。
- 临时下载已清理，用户源文件未被改动。

找不到合法完整 PDF 时，请用户提供本地文件；身份冲突时停止该篇上传并说明缺少的证据。服务器不可用时不能把 Agent 的临时下载报告为正式资源。
