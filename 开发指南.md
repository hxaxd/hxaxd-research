# 仓库开发与维护指南

本文面向仓库维护者。Agent 的日常执行入口仍是`AGENTS.md`。

## 1. 核心约束

- 同一规则只有一个主要归属, 其他文件只引用或补充。
- 研究阶段固定为 R1 Idea、R2 叙事、R3 实验证明、R4 实验执行、R5 写作占位、R6 投稿占位。
- 能力可在公开输入就绪时独立触发, 不依赖隐藏状态或固定阶段门禁, 也不能伪造成阶段完成。
- ROUTER 中的阶段和能力统一使用“做什么、触发、不触发、需要、产出、指导”六个字段。
- 专项指导只能细化 ROUTER 契约, 不能改写或放宽它。
- 未验证任务先披露缺少稳定指导; 低风险一次性任务可继续, 长期产物或重大决策必须暂停确认。
- 大文件由机械工具处理, 不自动进入 Agent 上下文。

## 2. 文档分工

| 位置 | 主要责任 |
| --- | --- |
| `README.md` | 面向首次使用者说明仓库定位、入口、路线和产物形态 |
| `AGENTS.md` | 定义目录模型、启动顺序、最高级边界和冲突优先级 |
| `ROUTER.md` | 定义阶段与能力的六字段公开契约及初始化规则 |
| `指导/使用说明.md` | 定义所有正式任务共同的确认、停止、证据和落盘协议 |
| 专项指导 | 在公开契约内定义执行方法、检查、确认点、完成条件和失败处理 |
| `指导/参考/` | 定义正式流程读写的共享文档格式 |
| `scripts/` | 执行 PDF 翻译、安装、校验等机械处理 |
| `tmp/` | 保存被忽略的旧资料或迁移中间件, 不得被正式规范引用 |

参考模板不得作为 Agent 状态文件。长期模板只有在现有文档无法承载独立责任时才新增; 临时模板必须由`AGENTS.md`或 ROUTER 明确允许并写清删除条件。

参考模板统一使用: 一级标题、简单介绍、`## 固定节`、严格骨架说明和一个完整代码块; 规则写入代码块, 不在其后追加说明。

工具版本、服务和环境变量由配置指导维护; 执行指导只保留必要调用方法, README 不复制配置细节。

## 3. 改哪里

1. 仓库对象、目录模型、启动或最高边界变化: 改`AGENTS.md`
2. 阶段、能力、路由、公开输入或产出变化: 改`ROUTER.md`
3. 所有任务共同的确认、停止、证据或落盘规则变化: 改`指导/使用说明.md`
4. 单个阶段或能力的执行方法变化: 改对应专项指导
5. 共享文档字段或格式变化: 改参考模板并同步所有读写指导
6. 首次使用说明或示例变化: 改`README.md`
7. 机械行为变化: 改`scripts/`并同步调用指导

跨层修改先改语义的主要归属, 再同步其他层。能力契约以 ROUTER 为主; 文档字段以参考模板为主。

## 4. 新增对象

### 阶段

- 确认它是研究主线依赖节点, 不是独立能力或内部步骤。
- 在 ROUTER 登记六字段, 创建专项指导。
- 新增长期产物时创建参考模板。
- 同步 README, 搜索旧编号、名称和路径。

### 能力

- 确认它可在公开输入就绪时直接触发且已经稳定。
- 在 ROUTER 登记六字段, 创建专项指导。
- 检查它读写的参考模板及输入特定产物。
- 完成后不得宣称研究阶段已经完成。

### 共享文档

新增前必须说明独立责任、创建者、更新者、读取者、稳定字段、历史修正规则, 以及为什么不能写入现有文档。不得新增仅供 Agent 跟踪进度的元文档。

文献身份默认进入`文献证据库.md`; Idea 与核心主张进入`研究想法.md`; 叙事和分点主张进入`叙事定稿.md`; 实验判据、状态和路径进入`实验证明.md`。

## 5. 修改核对

1. 搜索旧术语、编号和路径。
2. 检查 README、AGENTS、ROUTER、通用协议、专项指导和模板是否冲突。
3. 检查 ROUTER 六字段完整, 专项指导没有改写契约。
4. 检查所有具体指导路径真实存在。
5. 走查直接入口、跳过、返工、组合请求及失败情况。
6. 对资源能力区分“用户已经提供”和“用户需要寻找”。
7. 检查输入特定产物及部分失败的汇报口径。
8. 运行`git diff --check`, 确认没有空工作区模板或意外迁移。

目录模型变化时还要检查旧目录残留、重复材料、对象创建权和迁移确认。

## 6. 常见失败

- 还没读 ROUTER 就承诺工具、流程或目录。
- 同一规则在多处成为权威。
- 把专项步骤塞进 AGENTS 或 README。
- 两个能力同时处理同一种输入。
- 用户已有资源却再次进入资源获取。
- 关键前置或产物只藏在专项指导。
- 把推荐顺序变成不可跳过的门禁。
- Agent 替用户决定领域归属或研究方向。
- 组合任务只完成一部分却报告整体完成。
- 为整洁复制材料、创建空文件或新增状态文档。
- 把机械工具读取误当成 Agent 应阅读全文。

维护时优先修错误, 再补缺失, 最后处理轻微不严谨。新增确认点、文档或目录都必须具有不可替代的真实责任。
