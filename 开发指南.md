# 仓库开发与维护指南

本文面向仓库维护者。Agent 的日常执行入口仍是`AGENTS.md`。

## 1. 核心约束

- 同一规则只有一个主要归属, 其他文件只引用或补充。
- 同一含义只完整表达一次, 后文只写新增的判断责任。
- 活跃文档只描述当前有效设计: 对象是什么、何时触发、读取什么、产出什么、如何完成。
- 方案被放弃后, 从活跃文档中完整删除其名称、反向禁令、兼容说明和迁移叙述。只有仓库中仍有真实存量需要读取或迁移时, 才保留临时兼容规则; 存量处理完成后同步删除。
- 优先用正向契约限定行为。当前设计已经明确时, 不再列举仓库中不存在的替代产物、旧流程或外部项目做法。
- 研究阶段固定为 R1 Idea、R2 叙事、R3 实验证明、R4 实验执行、R5 论文写作、R6 投稿准备与返修。
- 能力在公开输入就绪时独立触发, 完成自己的公开产出契约。
- ROUTER 中的阶段和能力统一使用“做什么、触发、不触发、需要、产出、指导”六个字段。
- 专项指导只能细化 ROUTER 契约, 不能改写或放宽它。
- 未验证任务先披露缺少稳定指导; 低风险一次性任务可继续, 长期产物或重大决策必须暂停确认。
- 大文件由机械工具处理, 不自动进入 Agent 上下文。
- 高能力模型直接作用于真实研究产物, 用户负责方向、Story、实验范围、结果口径和投稿决定。
- 正式能力的登记依据是可复现的模型弱点, 或本地工具与真实产物之间需要稳定协作。当前跨阶段能力中的科研配图负责视觉论证与实际图件生成, 学术审阅与反向论证负责形成证据稳定的反方判断。
- 专项指导先解释任务中最难替代的判断和为什么这样判断, 再给可供模型选择的方法。无法从主题自然推出、违反后会破坏事实或产生不可逆影响的边界写成明确约束。

## 2. 文档分工

| 位置 | 主要责任 |
| --- | --- |
| `README.md` | 面向首次使用者说明仓库定位、入口、路线和产物形态 |
| `AGENTS.md` | 定义目录模型、启动顺序、最高级边界和冲突优先级 |
| `ROUTER.md` | 定义阶段与能力的六字段公开契约及初始化规则 |
| `指导/使用说明.md` | 定义所有正式任务共同的确认、停止、证据和落盘协议 |
| 专项指导入口 | 在公开契约内解释阶段的核心判断、仓库协作和推进方法 |
| 专项子指导与示例资产 | 承担可独立引用的详细方法、章节写法、示例或图示, 由对应入口按任务读取 |
| `指导/参考/` | 定义正式流程读写的共享文档格式 |
| `scripts/` | 执行 PDF 翻译、安装、校验等机械处理 |

参考模板服务于跨项目重复出现且字段稳定的真实产物。长期模板在现有文档无法承载独立责任、且确实存在跨项目固定格式时新增; 临时模板由`AGENTS.md`或 ROUTER 明确创建条件和结束条件。论文稿件以 TeX 为权威源格式, 投稿材料采用目标投稿场所的当前官方模板。

参考模板统一使用一级标题、简单介绍、稳定章节责任和一个完整代码块。跨阶段定位所需的标题保持稳定, 节内结构根据产物责任决定; 规则写入代码块, 不在其后追加说明。

工具版本、服务和环境变量由配置指导维护; 执行指导只保留必要调用方法, README 不复制配置细节。

当一个阶段入口同时混入多项可独立复用的详细方法时, 保留入口的阶段协作责任, 将详细方法拆到该阶段目录下有明确名称的子指导。子指导不改变 ROUTER 契约, 不产生新的研究阶段或长期研究产物; 示例和图示必须被某项真实指导直接引用。

`ROUTER.md`统一声明触发、输入和产出。专项指导围绕一个核心主题解释判断依据与推进方法; 确认写在真实的人类选择处, 终止判断写产物本身无法直接体现的条件, 高概率误判只在正文尚未覆盖时单独说明。

示例、骨架和问题清单展示可选思考路径及其适用条件。具体结构由研究对象、读者认知与证据关系决定。

## 3. 改哪里

1. 仓库对象、目录模型、启动或最高边界变化: 改`AGENTS.md`
2. 阶段、能力、路由、公开输入或产出变化: 改`ROUTER.md`
3. 所有任务共同的确认、停止、证据或落盘规则变化: 改`指导/使用说明.md`
4. 单个阶段或能力的执行方法变化: 改对应专项指导
5. 共享文档字段或格式变化: 改参考模板并同步所有读写指导
6. 首次使用说明或示例变化: 改`README.md`
7. 机械行为变化: 改`scripts/`并同步调用指导

跨层修改先改语义的主要归属, 再同步其他层。能力契约以 ROUTER 为主; 存在固定格式的共享文档以参考模板为主。

## 4. 新增对象

新增阶段或能力时先在 ROUTER 登记六字段公开契约, 再创建对应专项指导。

### 阶段

- 确认它承担研究主线中的独立依赖责任。
- 新增长期产物且存在跨项目固定字段时创建参考模板; 实际稿件、投稿包等格式由外部目标决定的产物不强制创建模板。
- 同步 README, 搜索旧编号、名称和路径。

### 能力

- 用可复现的模型弱点, 或本地工具、文件和外部系统之间的稳定协作需求说明新增理由。
- 确认它可在公开输入就绪时直接触发并承担独立产出责任。
- 检查它读写的参考模板及输入特定产物。
- 完成后不得宣称研究阶段已经完成。

### 共享文档

新增前必须说明独立责任、创建者、更新者、读取者、稳定字段和修正规则。

文献直接在使用它的研究句子附近引用; Idea 与核心主张进入`研究想法.md`; 中心叙事、下游共用的论文事实和分点主张进入`叙事定稿.md`; 方法与比较边界、共同实验协议、实验组合、证明判据、实际交付和分点判定进入`实验证明.md`; R4 的运行清单、日志和原始结果留在独立实验仓库; R5 维护 TeX 主入口及其真实依赖, 默认主文件名为`论文稿件.tex`; R6 创建目标投稿实际要求的材料。

## 5. 修改核对

1. 搜索旧术语、编号和路径。
2. 检查 README、AGENTS、ROUTER、通用协议、专项指导和模板是否冲突。
3. 检查 ROUTER 六字段完整, 专项指导没有改写契约。
4. 检查所有具体指导路径真实存在。
5. 走查直接入口、跳过、返工、组合请求及失败情况。
6. 对资源能力区分“用户已经提供”和“用户需要寻找”。
7. 检查输入特定产物及部分失败的汇报口径。
8. 搜索并删除已失效术语、旧路径、反向禁令和已经结束的兼容规则。
9. 检查每个新增文件是否承担公开、真实且不可替代的责任。
10. 运行`git diff --check`, 确认没有空工作区模板或意外迁移。
11. 逐篇检查语义是否只有一个主要归属, 每项约束是否具有独立且不可替代的理由。

目录模型变化时还要检查旧目录残留、重复材料、对象创建权和迁移确认。

维护时优先修错误, 再补缺失, 最后处理轻微不严谨。新增确认点、文档、栏目或目录都必须具有不可替代的真实责任; 如果删掉一段后任务含义和风险都不变, 这段通常不应存在。
