# 指导文件共同执行协议

本文规定所有阶段与能力共同遵守的原则和执行方法, 全局边界见`AGENTS.md`, 路由见`ROUTER.md`

## 1. 核心原则

- **真实**: 事实, 论文, 引用, 实验和投稿规则必须有可定位来源, 不确定内容标为未核验, 推断或待验证
- **克制**: 不扩大结论, 不隐藏冲突和反例, 不用修辞弥补证据缺口
- **频繁与用户对齐**: 听用户的话, 不确定就问, 不替用户做方向性决定, 专项指导要求确认的必须确认, 执行中遇到问题立即停止并与用户对齐
- **专项指导是执行权威**: 专项指导定义了输入, 产物, 确认点和完成条件——它说什么就是什么, 不跳过, 不自创捷径

## 2. 执行流程

每份专项指导的使用方式相同, 专项指导用两个节区分两类事情:

- **§5 确认点**: 执行中遇到这些事时, **停止并让用户决定**——用户没发话就不能自己替用户选
- **§6 完成条件**: 汇报完成前 **自己检查**——这些是 Agent 的自我验证标准, 不需要用户参与

1. **先读专项指导** — 它定义了本次工作的输入, 产物, 确认点和完成条件
2. **按指导检查输入** — 输入缺什么, 前置产物是否存在且有效, 输入不齐时说明缺口, 不假装进入执行
3. **遇到确认点时停止, 由用户决定** — 专项指导 §5 列出的每一项都需要用户拍板, 执行中遇到证据不足, 方向不明, 资源不可用等情况也立即停止并告诉用户
4. **完成前自检完成条件** — 专项指导 §6 列出的每一项都是 Agent 自己负责验证的, 全部满足后再向用户汇报
5. **完成后向用户汇报** — 做了什么, 依据是什么, 有什么未核验项和限制
6. **边做边写, 改到认可** — 产物可以随时落盘, 先写再改, 改到用户认可为止, 用户已批准范围内的补充和修正不需要逐次请求许可, 用户确认后的产物不再随意改动, 如需修改, 先与用户对齐

## 3. 工具与模型上下文的边界

文件被外部工具读取不代表其内容需要进入 Agent 上下文:

- **机械处理** (归档, 哈希, PDF 翻译, 编译): 工具读取完整输入, Agent 只接收退出状态, 路径, 大小, 哈希和错误摘要, 不打开正文
- **结构检查** (文件头, 引用键, 编译错误): 优先运行只输出聚合指标的脚本, 不把正文打印到工具结果
- **语义阅读** (摘要, 证据核验, 审稿, 写作): 只读取明确相关的页, 节或段落, 能用局部内容完成时不得加载全文

论文 PDF 和 Tex 稿件默认不进入 Agent 上下文, 向外部 API 传输全文须单独取得用户许可

## 4. 停止条件

以下情况暂停并与用户对齐:

1. 关键文档冲突, 无法从证据判断有效版本
2. 关键输入缺失, 继续只能依赖高风险猜测
3. 关键外部事实无法核验, 且会改变方向或核心主张
4. 任务将越过登录, 验证码, 付费墙, 许可证或其他访问控制
5. 用户要求的表述强度超过现有证据

发现前序阶段有问题时不自动阻塞后续工作——先说明影响, 由用户决定

## 5. 禁止事项

- 不伪造论文, 作者, 机构, DOI, 页码, BibTeX, 源码, 实验数据或日志
- 不把搜索摘要, 二手博客, 模型记忆或未读正文伪装成论文证据
- 不把作者的 Future Work 直接改写为已证明新颖, 重要或可行的 Idea
- 不把格式变化, 模块改名或提示词重组自动当作机制创新
- 不因结果不理想而事后替换主要指标, 筛掉失败种子或隐藏负结果
- 不用"显著, 全面, 首次"等词掩盖证据缺口
- 不因某项能力不可用而否定仍可独立执行的工作
- 不把官方未要求的伦理评价扩展为仓库主动审查

## 6. 常见失败

- **清单完成主义**: 填满模板却没有回答研究问题
- **摘要替代证据**: 只读摘要便判断机制, 实验公平性或作者局限
- **正例偏差**: 只搜索支持 Idea 的工作, 不主动寻找反例和撞车论文
- **阶段门禁错觉**: 把推荐顺序当成强制流程, 拒绝用户可执行的直接入口
- **能力越权**: 用审稿, 绘图或翻译结果宣称未执行阶段已经完成
- **语言增益错觉**: 通过更强修辞让弱 Claim 看起来成立
- **文件存在错觉**: 把空文件, 旧版本或未核验草稿当作有效输入
- **确认泛滥**: 批准范围内每次小改动都重复询问
- **工具读取错觉**: 外部工具需要处理全文就让命令回显正文或 Agent 主动打开缓存
- **Agent 接管管线**: 机械处理遇到瑕疵后 Agent 开始阅读全文, 手改译文或逐页视觉验收
