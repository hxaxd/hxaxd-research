# 指导文件共同执行协议

本文规定所有正式阶段与能力共同遵守的证据、协作和执行方法。

## 1. 证据与判断

- 事实、论文、引用、实验和投稿规则使用可定位来源; 未确认内容标为未核验、推断或待验证。
- 外部事实优先核对原始论文、官方文档和真实产物。搜索摘要、二手文章与模型记忆只提供线索。
- 研究文档在使用文献的位置直接引用。已有可靠网页来源时链接完整 DOI、arXiv 或官方论文网址; 暂无网页来源时写可识别标题并标记`链接待补`, 依赖事实同步标为待核验。本地原文使用相对仓库路径。
- 结论力度与证据一致, 冲突、反例、失败和不确定性进入真实判断。主要指标、种子与结果集合按预先确认的实验口径处理。
- 论文创新落在问题、机制、知识或能力边界的真实变化上; 格式、名称、提示词排列和 Future Work 只按其实际研究含义判断。
- Agent 直接在真实产物上完成事实、措辞、术语、引用和跨章节检查, 用户确认方向、Story、实验范围、结果口径和投稿决定。

## 2. 文档与指导

- `ROUTER.md`定义阶段和能力的公开契约, 专项指导在契约内解释核心判断与推进方法。
- 产物范围由公开契约声明, 每个文件承担清楚的人类可读责任, 任务进展由真实内容自然体现。
- 缺少稳定专项指导的任务先说明这一点。低风险一次性工作可以按用户授权继续; 长期产物和重大决定等待用户确认。
- 文件复制、PDF 翻译、编译与批量检查等机械工作在模型上下文外处理, 向 Agent 返回状态和必要错误摘要。正文只在用户目标确实要求理解内容时按需读取。

## 3. 协作节奏

1. 读取当前任务对应的专项指导, 理解真正要解决的判断与风险。
2. 检查真实输入和已有产物; 能安全继续时说明假设, 高风险缺口暂停。
3. 在方向、范围、核心主张、阶段跳过和重大投稿决定处与用户对齐。
4. 在批准范围内直接推进真实产物, 边做边写并根据内容迭代。
5. 按本阶段责任检查产物, 再向用户说明完成内容、依据、限制和待决定事项。

用户确认后的研究判断发生实质变化时重新对齐; 局部补充、纠错和表达修订可以直接进行。

## 4. 暂停条件

以下情况需要用户决定后才能继续:

1. 关键文档冲突, 证据无法确定有效版本。
2. 关键输入缺失, 继续需要高风险猜测。
3. 关键外部事实无法核验, 且会改变方向或核心主张。
4. 用户要求的结论超过现有证据。
5. 未登记或未验证任务会产生长期产物或影响重大决策。

前序阶段出现问题时说明其对当前工作的具体影响, 由用户选择修正前序内容或承担风险继续。
