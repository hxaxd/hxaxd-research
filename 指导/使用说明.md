# 指导文件共同执行协议

本文规定所有正式阶段与能力共同遵守的原则和执行方法。

## 1. 核心原则

- **真实**: 事实、论文、引用、实验和投稿规则必须有可定位来源, 不确定内容标为未核验、推断或待验证。
- **克制**: 不扩大结论, 不隐藏冲突和反例, 不用修辞弥补证据缺口。
- **频繁与用户对齐**: 听用户的话, 不确定就问, 不替用户做方向性决定。专项指导要求确认的必须确认。
- **专项指导是具体执行权威**: `ROUTER.md`定义阶段与能力的公开输入和产出契约, 专项指导只能在上层契约内细化执行方法、检查、确认点和完成条件, 不得改变公开契约。没有稳定专项指导的任务, 必须先说明这一点, 再按风险和用户授权决定是否继续。
- **只落盘契约产物**: 只有`ROUTER.md`和专项指导明确声明的产物才落盘, 不为对话判断、过程跟踪或格式整洁创建额外文档。

## 2. 执行流程

每份专项指导的使用方式相同:

1. **先读专项指导** — 它定义本次工作的具体方法、确认点和完成条件。
2. **按公开契约和指导检查输入** — 输入缺什么, 前置产物是否存在且有效, 输入不齐时说明缺口, 不假装进入执行。
3. **遇到确认点时停止, 由用户决定** — 方向、范围、核心主张、阶段跳过和重大投稿决定都需要用户确认。
4. **完成前自检完成条件** — 专项指导列出的完成条件由 Agent 自己负责验证。
5. **完成后向用户汇报** — 说明做了什么、依据是什么、有什么未核验项和限制。
6. **边做边写, 改到认可** — 产物可以随时落盘, 先写再改。用户已批准范围内的补充和修正不需要逐次请求许可; 用户确认后的产物不再随意改动, 如需修改先与用户对齐。

## 3. 停止条件

以下情况暂停并与用户对齐:

1. 关键文档冲突, 无法从证据判断有效版本。
2. 关键输入缺失, 继续只能依赖高风险猜测。
3. 关键外部事实无法核验, 且会改变方向或核心主张。
4. 用户要求的结论超过现有证据。
5. 用户请求不属于`ROUTER.md`已登记的阶段或能力, 或对应条目明确标为未验证, 且执行会产生长期产物或影响重大决策。低风险一次性任务在披露缺少稳定专项指导后可按用户原请求继续。

发现前序阶段有问题时不自动阻塞后续工作——先说明影响, 由用户决定。

## 4. 禁止事项

- 不伪造论文、作者、机构、DOI、页码、BibTeX、源码、实验数据或日志。
- 不把搜索摘要、二手博客、模型记忆或未读正文伪装成论文证据。
- 不把作者的 Future Work 直接改写为已证明新颖、重要或可行的 Idea。
- 不把格式变化、模块改名或提示词重组自动当作机制创新。
- 不因结果不理想而事后替换主要指标、筛掉失败种子或隐藏负结果。
- 不用“显著、全面、首次”等词掩盖证据缺口。
- 不因某项未登记能力缺少专项指导而否定仍可独立执行的工作。
- 不把官方未要求的伦理评价扩展为仓库主动审查。

## 5. 常见失败

- **清单完成主义**: 填满模板却没有回答研究问题。
- **摘要替代证据**: 只读摘要便判断机制、实验公平性或作者局限。
- **正例偏差**: 只搜索支持 Idea 的工作, 不主动寻找反例和撞车论文。
- **引用堆砌**: 用大量论文名制造判断权威, 却没有说明最关键论文与当前核心主张的对应关系。
- **阶段门禁错觉**: 把推荐顺序当成强制流程, 拒绝用户可执行的直接入口。
- **能力越权**: 用未登记能力的结果宣称正式阶段已经完成。
- **语言增益错觉**: 通过更强修辞让弱主张看起来成立。
- **文件存在错觉**: 把空文件、旧版本或未核验草稿当作有效输入。
- **确认泛滥**: 批准范围内每次小改动都重复询问。
- **工具读取错觉**: 外部工具需要处理全文就让命令回显正文或 Agent 主动打开缓存。
- **Agent 接管管线**: 机械处理遇到瑕疵后 Agent 开始阅读全文、手改译文或逐页视觉验收。
