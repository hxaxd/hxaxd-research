# Tex 成稿工程验收

## 1. 做什么

使用论文项目自己的工具链编译并验收终稿工程, 确认源文件, 引用, 图表, 版面, 元数据和最终 PDF 可以稳定重建, 该能力不替代论文内容审稿

## 2. 输入

- 论文项目路径, 主 Tex 入口和项目自身构建说明
- 目标模板, 页数与提交格式要求
- 当前 BibTeX, 图表, 附录和补充材料
- 允许清理或修改的本地生成文件范围

## 3. 执行

### 验收流程

1. 先读取项目自己的`AGENTS.md`, README, 构建脚本和忽略规则
2. 记录当前 commit, 工具链版本, 主入口和实际构建命令
3. 从可解释的工程状态执行完整编译, 不用旧 PDF 冒充本次成功
4. 检查编译错误, 未定义引用, 未定义 citation, 重复 label 和缺失资源
5. 检查图表路径, 字体嵌入, 页面尺寸, 页数, 溢出, 空白页和明显排版异常
6. 检查标题, 作者/匿名信息, PDF 元数据, 附件和最终文件名
7. 对最终 PDF 抽查目录, 公式, 图表, 参考文献和附录是否与源文件一致
8. 修复工程问题后重新全量编译, 并保存最终命令和输出摘要

### 边界

- 使用项目现有工具链, 不擅自迁移构建系统
- 不通过删除正文, 缩小到不可读字体或破坏模板来消除页数问题
- 不静默清理用户文件, 只处理确认范围内的安全生成物
- 内容事实, Claim 和引用证据问题转交对应能力, 不用编译通过替代内容核验

## 4. 产出

生成`Tex成稿验收记录.md`, 至少记录:

- 项目路径, commit, 工具链和命令
- 最终 PDF 路径和文件哈希
- 错误, 警告, 引用, 图表, 页数, 溢出, 字体和元数据检查
- 已修复项, 未解决项和是否阻断投稿

编译错误, 未定义引用, 缺失图表和无法重建最终 PDF 均为 R8 硬性阻断项, 能力完成后结束
