# 中文学术表达润色

## 任务目标

只修复中文论文中的口语化, 语病, 逻辑断层, 指代不明和严重欧化长句, 保留作者原有风格, 不为了显示修改而替换同义词

本指导处理表达, 不负责补论文逻辑, 证据或贡献, 若原文命题本身不成立, 应先指出,
而不是把它润色成更有说服力的错误表述

## 修改重点

- 使用现代, 平实, 准确的中文学术书面语, 避免陈旧公文腔
- 将主观表达改为有依据的客观陈述
- 仅在逻辑断裂时显化连接词, 避免机械堆砌
- 修复主语缺失, 指代不明, 搭配错误和过长定语
- 必要时调整信息顺序, 使条件, 动作和结果关系清楚

## 保护项

- 保留通用英文技术术语和必要公式
- 输出适合进入 Markdown 或 Tex 正文, 不引入 Markdown 强调符号
- 原文无需修改时完整保留并明确说明
- 不改变数字, 限定, 比较对象, 因果和 Claim 强度
- 不把“可能”删成确定结论, 也不凭空加入“显著”“有效”

## 执行步骤

1. 识别句子技术命题和必须保留的信息
2. 先修歧义和语病, 再处理学术语域
3. 对拆句, 合句或顺序变化说明理由
4. 与上下文术语和主语保持一致

## 常见陷阱

- 为追求简洁删除关键条件
- 把自然中文改成公文式表达
- 过度增加“此外, 因此, 值得注意的是”
- 将通用英文术语生硬翻译成领域不使用的中文

## 输出

1. 润色后的中文文本或原文
2. 必要修改说明, 区分语病修复和表达优化；未修改时说明原文已符合要求
