# 去模型化表达

## 1. 责任

本文负责论文内容与结构稳定后的语言诊断和修改, 目标是得到准确、具体且体现真实作者判断的学术语言。它不改变研究事实、中心叙事或证据边界, 也不以自动检测分数为目标。

## 2. 证据线索

[Science Advances 对超过一千五百万篇生物医学摘要的研究](https://doi.org/10.1126/sciadv.adt3813)观察到大模型普及后部分风格性动词和形容词频率突增; [ICML 2024 对同行评审文本的研究](https://arxiv.org/abs/2403.07183)也发现特定评价词的集中使用与文本同质化。[ACL 2025 的人类识别研究](https://aclanthology.org/2025.acl-long.267/)表明, 熟悉模型写作的人还会依据可预测句式、公式化结构、内容原创性和清晰度判断文本。

这些研究描述群体层面的语言规律, 不能用于判断某一篇论文的作者身份。ACL 研究面向英语非虚构文章而非学术论文, 因此这里只把其发现作为修改线索。

模型化表达通常同时发生在三个层面:

- 内容抽象, 评价多于信息
- 结构过度均匀, 论证被固定套路压平
- 词句集中使用华丽评价、公式化连接和安全总结

只替换几个高频词不能解决前两个层面。

## 3. 内容层修改

- 每个背景句提供当前论证需要的对象、事实或边界
- 把“重要、关键、全面、稳健、有效”等评价落实到机制、条件、指标或比较
- 每项贡献说明设计、原因、运行变化和证据
- 用真实失败、取舍、中间现象和边界体现作者判断
- 保留不对称结果和仍未排除的解释
- 结尾交付本文真正得到的认识

可以把一句话放进任何同领域论文而含义几乎不变时, 它通常缺少本文信息。修改时优先补入具体对象和关系; 没有独立信息时直接删除。

## 4. 结构层修改

- 段落和小节长度由证据关系决定
- 平行结构只表达真正平行的内容
- 相邻段落不重复总结同一信息
- 关键结果获得充分解释, 次要实现可以只占一句
- 贡献数量、并列项数量和句式不预设为三项
- 每节结束方式服从它实际交付的判断

检查是否每节都采用“泛泛背景—三项列表—完整总结”, 是否连续段落拥有相同长度和句法, 是否每段首尾重复同一判断。真实平行关系仍应保持清楚, 不为制造随机感破坏论证。

## 5. 词句层修改

以下英语表达容易在模型文本中集中出现, 出现时检查其是否提供可验证含义, 而不是机械视为禁词:

- 评价词: `pivotal`, `crucial`, `groundbreaking`, `remarkable`, `robust`, `significant`, `comprehensive`, `seamless`
- 空泛动作: `delve into`, `underscore`, `showcase`, `navigate`, `foster`, `unlock`, `pave the way`
- 公式化元话语: `it is important to note`, `in the landscape of`, `in conclusion`, `not only ... but also ...`, `a testament to`

逐处询问:

- 这个词提供了什么可以验证的含义
- 删除后信息是否完全不变
- 是否能改成具体主语、动作、条件或结果
- `significant`是否确实指向正式统计检验
- 连接词是否标记相邻句真实存在的关系

## 6. 修改示例

```text
空泛:
Our comprehensive framework seamlessly integrates project memory and significantly improves robust performance.

具体:
The compiler binds each declared knowledge requirement to a verified project fact before execution; Section [X] measures the resulting change in [metric] under [condition].
```

```text
空泛:
These remarkable results pave the way for a wide range of future applications.

具体:
The gain persists when [frozen artifact] is transferred to [target condition], but disappears under [observed boundary].
```

```text
空泛:
In the rapidly evolving landscape of [field], it is crucial to develop robust and efficient solutions.

具体:
[Method class] fails when [condition], increasing [measured consequence] because [supported reason].
```

方括号只表示待替换的真实论文事实。修改不能补造数据、机制、应用或适用范围。

## 7. 修改轮次

1. 删除不承载独立信息的句子
2. 把剩余抽象判断改成具体对象、机制、条件、数据或来源
3. 按真实论证关系调整段落和句式
4. 恢复作者的真实选择、取舍、反例和限制
5. 检查集中出现的评价词、空泛动作和公式化元话语
6. 重新核对事实、引用、术语和主张边界

修改后用`指导/写作/论证与表达.md`的反向提纲检查内容关系仍然完整。语言自然度来自准确内容与真实作者判断, 不通过语法错误、口语碎片、虚构经历、伪造细节或随机扰动模拟人工写作。

## 8. 完成检查

- 文字中的力度来自事实、机制与证据
- 章节和段落结构反映真实论证关系
- 术语稳定, 连接词有实际逻辑依据
- 不存在可以无差别放入其他论文的背景和前景句
- 真实取舍、失败与边界没有在润色中被抹平
- 修改没有引入新事实、数字或更强主张
