# 实验数据分析与段落生成

## 任务目标

从真实实验数据中提取趋势, 比较, 敏感性, 模块贡献和效率权衡, 并生成可核对的中文或英文 Tex 分析段落

本指导只把已经完成的研究分析转化为写作素材, 不代替 R6 对 Claim, 统计, 公平性和失败的判断

## 输入检查

- 原始或已验证的结构化结果, 而非手工截图中的模糊数字
- 实验编号, 配置, 指标方向, 单位和聚合方式
- 对应 Claim 及 `结果分析.md` 的允许解释
- 需要生成中文还是英文 Tex, 以及目标段落位置

## 分析流程

1. 核对数据来源和实验编号, 确认缺失值, 排除运行和样本量
2. 计算或复核绝对差, 相对差, 均值, 离散程度和所需统计量
3. 先列出可直接观察的现象, 再列可能解释
4. 围绕一个 Claim 选择最有说明力的数据, 不逐格报数
5. 比较总体趋势与异常子组, 避免平均值掩盖失败条件
6. 效率分析同时报告性能和成本边界
7. 没有明显优势或趋势时如实描述, 不强行总结

## 写作结构

推荐一段包含:

1. 结论句；
2. 关键数据；
3. 与 Baseline 或消融的比较；
4. 受证据约束的解释；
5. 必要的局限

## 禁止事项

- 不补造方差, 显著性, 样本量或不存在的数据点
- 不将相对提升与百分点混用
- 不把相关性写成因果
- 不用 `significant` 描述普通幅度, 除非有明确统计含义
- 不忽略方向相反的关键数据

## 常见陷阱

- 从格式化到两位小数的表格反推过度精确结论
- 把最好设置与 Baseline 平均设置比较
- 一段同时承担性能, 效率, 鲁棒和机制四个结论
- 解释比数据更强, 却没有实验区分替代原因

## 输出

- 结论提纲
- 每个结论对应的数据和 Claim
- 需要进入论文时, 使用 `\\paragraph{核心结论}` 加连续分析文本
- 附中文核对版本
- 附所有计算口径和无法从数据推出的内容
