# 局限性写作

## 能力目标

准确说明研究适用范围, 证据缺口, 资源约束和不能外推的结论, 本指导按稿件和 venue
需要调用, 不要求独立章节

## 输入

- `核心主张.md` 的适用范围和不支持项
- `结果分析.md`, `失败记录.md` 和实验覆盖范围
- 数据, 模型, Benchmark, 预算和实现限制
- venue 对局限性声明的官方要求 (若有)

## 写作流程

1. 从真实证据边界提取局限, 不生成通用免责声明
2. 区分研究范围选择, 当前实现限制和证据不足
3. 说明局限影响哪个 Claim, 在哪些条件下出现
4. 对可验证的后续问题给出克制表述, 不承诺结果
5. 若局限实际上推翻核心结论, 先回写研究文档, 不能只放在结尾

## 禁止事项

- 不用“未来将扩大数据集”掩盖当前结论无法成立
- 不列与论文无关的万能局限
- 不把致命缺陷淡化为轻微 Future Work
- 不写未核验的社会或伦理评价, 官方要求的声明按投稿规则处理

## 输出检查

局限具体, 可定位并与 Claim 相连, 没有削弱必要限定词, 读者能判断哪些结论可用,
哪些不能外推, 完成后结束
