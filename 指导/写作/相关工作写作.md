# 相关工作写作

## 任务目标

按问题和机制组织相关工作, 准确说明研究位置和本质区别, 而不是列论文摘要或刻意贬低已有方法

Related Work 的任务是建立可审查的研究坐标: 读者应看出主要路线如何解决问题,
最相近工作在哪里, 以及本文差异是否真实, 它不是“证明别人都不行”的章节

## 组织流程

1. 从 `相关工作矩阵.md` 选择两到四个真正影响本文定位的主题轴
2. 检查每个主题轴是按问题, 机制, 监督, 系统阶段或评价方式划分, 而非随意按论文名
3. 每个小节先定义主题与边界, 再概括路线演化和代表工作
4. 对每篇论文只写与当前主题和本文差异有关的内容, 不复制摘要
5. 对最高撞车风险工作给予足够篇幅, 不能藏在引用串中
6. 关键事实从 `文献证据库.md` 取用, 作者局限与 Agent 推断分开
7. 小节结尾总结尚未覆盖空间和本文位置, 但不提前重复全部方法贡献

## 比较语言

优先使用中性, 可核验表达:

- “A 在设定 X 下采用机制 M, 并在数据 D 上验证, ”
- “本文关注的 Y 未包含在其评价范围内, ”
- “两者都依赖假设 H, 但本文试图区分……”

谨慎使用 “fails to”“cannot”“ignores”, 若论文只是没有报告某项实验, 应写
“未评估”而不是“无法处理”

## 引用原则

- 不生成未核验 BibTeX
- 一句话涉及多个工作时, 确认它们真的共享该结论
- 优先引用原始方法和正式版本, 不只引用综述
- 版本, 年份或发表状态不确定时标记待核验
- 本文对已有工作的推断应明确是比较判断, 不冒充作者自述

## 不应做什么

- 不按年份逐篇摘要
- 不用宽泛引用串支撑具体技术结论
- 不遗漏最相近工作或负面复现
- 不让所有已有路线都恰好缺少本文模块
- 不把 Related Work 写成第二个引言或贡献宣传页

## 常见陷阱

- 分类主题彼此重叠, 同一段反复出现相同论文
- 只比较输入输出和架构, 不比较本质假设
- 小节末尾突然声称本文解决所有局限
- 引用很多, 但没有一句说明本文与最相近工作本质差异

## 输出要求

- 先输出主题结构与每段中心句
- 正文使用中文 Tex, 并保留准确引用占位
- 无法核验的引用标为待核验, 不生成虚假 BibTeX
- 完成后检查是否遗漏最高撞车风险工作
- 附主题轴到 `相关工作矩阵.md` 行和证据编号的映射
