# 引言版本 3: 一般任务 -> 具体设定

`版本 3: 先介绍一般任务的应用, 再介绍具体任务设定,  (如果这个设定相对新颖, 我个人推荐这种写法, )`

## 适用场景

- 一般任务为社区熟知, 但本文研究的是较新或较窄的输入, 输出, 数据或约束
- 具体设定的变化会产生本文要解决的技术挑战

```latex
% 介绍一般任务的应用
%% 示例: 准确的姿态估计对于增强现实, 自动驾驶和机器人操作等多种应用都至关重要
[xxx task] 有多种应用, 例如 [xxx], [xxx] 和 [xxx]

% 介绍具体任务设定
%% 示例: 本文聚焦于从一张 RGB 图像中恢复物体的 6DoF 姿态, 即其在三维空间中的旋转和平移
本文聚焦于从 [xxx input] 中恢复/重建/估计 [xxx output] 的具体设定
```

## 使用提醒

- 必须说明具体设定与一般任务相比改变了什么
- 下一段应解释该变化为何带来新挑战, 而不是直接宣布新方法
- 不把普通的数据集切换包装为新任务设定
- 若具体设定已有大量工作, 应在此后准确介绍现状, 不能称其 "新颖"
