# 引言版本 1: 先介绍任务, 再介绍应用

`版本 1: 如果任务相对小众, 先介绍任务, 再介绍应用,`

## 适用场景

读者可能不知道任务的输入输出, 且理解任务定义是后续挑战成立的前提

```latex
% 介绍任务 (如果该任务非常熟悉, 这一部分可以跳过) 
%% 示例: 目标姿态估计旨在从单张图像中估计物体相对于规范坐标系的朝向和位移
[xxx task] 旨在从 [xxx input] 中恢复/重建/估计 [xxx output]

% 介绍应用
%% 示例: 准确的姿态估计对于增强现实, 自动驾驶和机器人操作等多种应用都至关重要
[xxx task] 有多种应用, 例如 [xxx], [xxx] 和 [xxx]
```

## 使用步骤

1. 用一句话写清输入, 输出和必要条件
2. 只列与论文问题直接相关的应用
3. 下一段进入现有方法或目标挑战

## 常见误用

- 任务定义包含方法细节, 导致读者误以为任务只属于某类方法
- 应用列表过长, 却没有说明技术重要性
- 对熟悉任务重复教科书定义, 浪费引言篇幅
