# 引言版本 4: 先介绍应用, 再引出挑战

`版本 4: 如果任务比较熟悉, 可以直接介绍应用, 并在开头段通过已有方法暴露目标技术挑战,`

适用判断:

1. 如果开头段就已经说明我们想解决什么问题, 通常是好的
2. 但这种写法需要合适条件, 也更少见
3. 通常在目标挑战变得清晰之前, 还是需要若干段已有工作来铺垫

```latex
% 介绍应用
%% 示例 1: 从多视图图像中重建三维场景是增强现实, 机器人和自动驾驶等许多应用的基础
%% 示例 2: 实例分割是许多计算机视觉任务的基础, 例如视频分析, 自动驾驶和机器人抓取, 这些任务既需要准确性, 也需要效率

% 利用已有方法暴露目标技术挑战
%% 示例 1: 给定输入图像, 传统方法 [43, 44, 59] 通常先基于多视图立体 (MVS) 算法估计每张图像的深度图, 然后把这些估计的深度图融合成三维模型, 尽管这些方法在大多数情况下都能取得成功的重建效果, 但它们难以处理低纹理区域, 例如室内场景中的地板和墙面, 因为这些区域里的立体匹配不够可靠
%% 示例 2: 当前最先进的大多数实例分割方法 [18, 27, 5, 19] 都是在由目标检测器 [36] 给出的边界框内进行逐像素分割, 这可能对不准确的边界框较为敏感, 此外, 将目标形状表示为稠密二值像素通常会导致代价高昂的后处理
```

## 使用提醒

- 开头挑战必须是本文真正解决的挑战, 而非领域通用困难
- 已有方法与挑战之间的因果需要证据
- 若一两句无法让读者理解任务, 应先补任务定义
- 不要为制造强开场把小范围失败写成普遍问题
