# 技术挑战版本 3 (新任务)

`版本 3: 对于没有直接方法对应的新任务, 直接定义挑战, 并按需求/挑战点拆解,`

```latex
% 为了实现 xx 目标, 必须满足若干需求/挑战
%% 示例: 在这项工作中, 我们的目标是构建一个能够从单张图像中捕捉这类物体内在属性的模型, 这个问题之所以具有挑战性, 是因为有三个原因

% 描述第 1 点
%% 示例: 首先, 我们只有一张图像, 这使得我们的工作在本质上不同于现有的三维感知图像生成模型 [8, 9, 27, 28], 后者通常需要包含成千上万实例的大型数据集来训练, 相比之下, 单张图像中最多只有几十个实例, 因此推理问题高度欠约束

% 描述第 2 点
%% 示例: 其次, 这些本就有限的实例在像素值上可能差异很大, 这是因为它们具有不同的姿态和光照条件, 但这些因素都没有标注或已知, 我们也不能求助于基于结构光恢复姿态的现有工具, 例如 COLMAP [35], 因为外观变化违反了极几何的假设

% 描述第 3 点
%% 示例: 最后, 我们希望推断的物体内在属性是概率性的, 而不是确定性的: 自然界中没有两朵玫瑰是完全相同的, 因此我们希望捕捉它们在几何, 纹理和材质上的分布, 以利用底层的多视图信息
```

## 使用提醒

- 先确认任务确实缺少直接方法脉络, 避免把已有任务误包装成新任务
- 挑战数量由任务需求决定, 不要求恰好三个
- 每项挑战应能对应方法或实验责任
- 任务定义, 数据约束和科学难点应分层, 不混成并列列表
- 挑战来自可核验设定, 不从预设模块反向编造
