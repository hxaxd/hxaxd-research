# Neural Body 标注图（文本转写）

本文件把 Neural Body 的标注图转换成可复用的写作笔记。

## 目的

使用这份映射，理解一个 Method 小节如何显式地区分：

1. 模块动机
2. 模块设计（数据结构）
3. 模块设计（前向过程）
4. 技术优势

## 按块映射

### 第 3.1 节：结构化潜变量代码

1. **模块设计（数据结构）**
- 该段定义了锚定在可形变人体模型（SMPL）上的结构化潜变量代码。
- 它解释了构建了什么（潜变量代码 + 它们的锚点位置 + 由姿态驱动的帧相关变换）。

2. **技术优势**
- 该段解释了为什么这种设计更有效：它有利于动态人体表示和跨帧观测整合。
- 它强调了将代码锚定到可形变几何上为何有益。

### 第 3.2 节：代码扩散

1. **该模块的动机**
- 该段指出了剩余问题：直接插值稀疏的结构化代码会导致许多三维点附近出现接近零的向量。
- 这说明了为什么需要把表面代码扩散到附近三维空间。

2. **模块设计（前向过程）**
- 该段解释了执行流程：构建稀疏潜变量体，运行稀疏卷积，在查询点处插值潜变量代码，并将这些代码送入预测网络。
- 这是一个典型的输入 -> 步骤 -> 输出的模块描述。

### 第 3.3 节：密度与颜色回归

1. **密度模型的模块设计（前向过程）**
- 密度段定义了如何根据潜变量代码和帧条件回归密度。

2. **颜色模型的模块设计（数据结构）**
- 颜色段引入了所需的输入/嵌入（潜变量代码、视线方向、空间位置、时间嵌入）。

3. **颜色模型的模块设计（前向过程）**
- 紧接着的段落描述了如何对这些输入进行编码，并将其送入颜色 MLP 以最终预测颜色。

### 第 3.4 节：体渲染

1. **模块设计（前向过程）**
- 该段描述了光线采样和体积分，从而把预测的密度/颜色场渲染成图像输出。

## 由该图总结出的可复用写作模式

对于每个模块小节，按以下顺序写：

1. `动机`：说明未解决的挑战和技术原因。
2. `设计-1`：定义结构/表示/网络。
3. `设计-2`：按执行顺序描述前向过程。
4. `优势`：解释为什么这个模块优于替代方案。

## 建议的段落开头

1. 动机：`一个仍然存在的挑战是 ...`
2. 数据结构设计：`我们用 ... 来表示 ...`
3. 前向过程：`给定 [input]，我们先 ... 然后 ... 最后 ...`
4. 技术优势：`与既有方法相比，这种设计 ...，因为 ...`

## 使用限制

- 段落开头只是功能提示，不应机械照抄。
- 图注通常省略动机、条件和实现细节，不能直接扩成正文而不核验。
- “优势”必须区分结构性质与实验结论。
- 若图与正文不一致，应先修正方法定义或图，而不是强行让文字匹配。
