# 方法章节骨架

本骨架只表示常见信息顺序，章节数量应由方法依赖决定。

```latex
\section{Method}

% 1. 问题设定与符号（若未在前文定义）
% 输入、输出、目标和必要假设

% 2. 方法概述
% 整体机制、数据流、训练/推理差异和图示导航

% 3. 核心模块一
% 动机、设计、前向过程、作用和必要公式

% 4. 核心模块二
% 与模块一的接口及新增机制

% 5. 训练目标或优化
% 损失、采样和优化流程

% 6. 推理过程与复杂度
% 只写影响理解和复现的内容
```

## 使用原则

- 模块小节按依赖顺序，而不是按贡献重要性排序。
- 没有独立机制的工程组件可以放在实现细节。
- 训练和推理差异较大时分开描述。
- 公式密集不等于结构完整，仍需解释输入输出和作用。
- 骨架中的任何小节都可以删除或合并，不能为填满模板增加内容。
