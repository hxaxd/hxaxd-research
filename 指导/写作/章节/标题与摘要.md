# 标题与摘要

## 它们压缩的是论文差异

标题和摘要通常在方法、结果与结论稳定后收束, 承担整篇论文最强的一次信息压缩。读取`叙事定稿.md`、`实验证明.md`和正文最终术语, 保留足以让目标读者判断“研究什么、改变了什么、凭什么相信”的内容。

## 标题

标题可以突出任务、精确问题、关键机制、核心认识或它们的必要组合。选择取决于哪项信息最能把本文与相近工作区分开:

- 任务尚不熟悉时, 让读者先知道研究对象
- 同一任务下路线众多时, 突出机制或设置差异
- 核心认识本身就是贡献时, 标题可以直接表达认识
- 方法名只有在已经传达技术含义或需要建立稳定身份时才占据中心

效果、泛化和最佳水平等结果性判断只有在正式证据支持时进入标题。删去一个形容词后含义完全不变, 通常说明它没有贡献信息。

## 摘要

摘要要让离开正文的读者理解论文的“研究增量”。根据目标读者已经知道什么, 选择必要关系:

```text
读者需要的任务或设置
-> 限制现有能力的精确问题
-> 本文带来的认识或方法变化
-> 最有辨识力的运行机制
-> 足以校准结论的结果和边界
```

熟悉任务可以很快进入失败条件; 新任务需要先定义输入输出。方法由一个关键观察推出时, 观察值得成为转折; 贡献相对独立时, 可以分别交付其关系。没有独立信息的背景和前景句直接省略。

下面两个节奏展示常见的内容选择。

### 问题直接推出方法

```tex
\begin{abstract}
% 用目标读者所需的最少信息定位任务与设置
% 说明现有路线在目标条件下的具体失败
% 提出本文方法及最能区分它的运行机制
% 报告支持总体判断的结果, 并在必要时给出成本或范围
\end{abstract}
```

### 核心认识连接问题与方法

```text
This work studies [task] under [setting].
Existing methods rely on [assumption], which causes [failure] when [condition].
We find / observe that [supported observation or insight].
Based on this observation, [method] changes [mechanism or information flow].
Across [evaluated conditions], this yields [supported result], while [boundary if central].
```

多项贡献分别出现时, 每项都说明它改变了什么:

```text
We introduce [design], which changes [mechanism] to address [specific problem].
```

## 结果怎样进入摘要

摘要优先报告能决定论文判断的证据: 与强基线在公平协议下的主要差异、核心机制对应的现象、以及会改变读者评价的成本或边界。数字、比较对象和统计口径与主表逐项一致。

完成后通读一次: 读者能否准确复述任务条件、研究变化和证据; 技术问题是否正是方法解决的问题; 标题、摘要和结论是否在强度与范围上相互一致。缺失信息应补到真正承担它的位置, 不靠一句塞入多个职责。
