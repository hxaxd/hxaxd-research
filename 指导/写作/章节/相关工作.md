# 相关工作

## 相关工作是一项定位论证

相关工作不负责证明作者读过多少论文, 而是确定本文在已有知识中的真实位置。读者需要知道最相近路线已经解决什么、依赖什么假设或机制、在哪个与本文有关的条件下停止, 以及本文究竟改变了问题、信息边界、方法机制、优化目标还是适用范围。

写作以原始论文为依据。检索摘要可以提供线索, 但关键定位必须读到足以还原对方方法和主张的原文位置。

## 先建立比较关系

从`叙事定稿.md`读取最相近工作与真实差异, 再结合实验中实际使用的强基线、与核心机制接近但任务不同的工作、任务相同但路线不同的工作, 以及为核心认识提供先例或反例的工作。

一篇文献进入正文前应有明确角色。外围背景可以简写, 最可能改变新颖性判断的工作获得最充分比较。文献数量和引用密度服从定位需要。

## 按读者需要解决的比较问题组织

主题可以按方法路线、共享假设、信息来源或目标条件组织。一个主题段落通常先让读者知道讨论范围, 再概括代表工作解决了什么, 随后收窄到本文关心的边界并说明真实关系。句段节奏根据比较的复杂度决定。

```tex
\section{Related Work}
% 主题一: 通过一组工作建立与本文有关的比较判断
% 主题二: 处理另一项不能由主题一覆盖的定位问题
% 最相近工作在最需要的位置作直接比较
```

按年份罗列只有在技术演化本身构成论证时才有意义。

## 最相近工作要正面处理

对决定性工作, 用自己的话说明它研究什么、实际怎样做、与本文共享什么, 差异发生在哪个条件或机制, 以及这项差异是否进入直接实验。本文继承、组合或延伸已有思想时明确承认, 再说明新增知识。

真实重合不能通过换名、分散到多个小节或只谈外围差异来弱化。相反, 精确说明“相同在哪里、变化在哪里、证据怎样区分”会让贡献边界更可信。

## 与全文保持同一比较口径

引言中的技术问题应在相关工作中得到公平背景, 实验强基线应获得相称定位, 相关工作声称的差异应能由方法或实验事实支持。不可比的公开数字不证明本文更优, 新颖性措辞也不能超过 R1/R2 已核验边界。

正式稿件使用核验后的引用键与`references.bib`。完成后, 读者应能清楚复述本文继承什么、改变什么、仍不覆盖什么; 如果只能复述若干论文名称, 这一节还没有完成定位责任。
