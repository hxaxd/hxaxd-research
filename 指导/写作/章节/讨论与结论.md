# 讨论与结论

## 1. 讨论

目标场所设置独立讨论节时, 围绕结果对论文认识的影响组织:

- 结果怎样回答引言提出的问题
- 哪项证据最直接支持核心认识
- 哪些机制解释仍存在替代解释
- 本文与最相近工作的关系因结果发生了什么变化
- 限制属于方法缺陷、证据覆盖不足还是适用范围
- 哪些后续问题由当前结果自然产生

讨论允许综合多个结果, 但不能把相关性提升为因果, 也不能补入正文尚未建立的新主张。

## 2. 结论的基本结构

```tex
\section{Conclusion}
% 本文解决的精确问题
% 核心认识与方法响应
% 最有代表性的实验证据
% 由证据得到的知识或实际价值
% 适用边界与自然后续问题
```

可用于检查信息关系的英文骨架:

```text
This work addresses [problem] by introducing [method].
The key idea is [insight], which enables [supported advantage].
Experiments across [datasets / settings] support [main conclusion].
The current evidence is limited to [scope boundary].
Extending [specific mechanism or evidence] to [future setting] remains open.
```

结论不逐节复述, 而是重新交付问题、认识、证据、价值和边界。

## 3. 区分方法缺陷与范围局限

### 方法缺陷

例如:

- 在论文核心指标上明显弱于公平强基线
- 改善一个指标却损害另一个同样重要的指标
- 依赖与目标场景冲突的假设
- 每个新场景都需要额外调节或高昂适配
- 新机制的成本超过其可验证收益

### 范围局限

例如:

- 当前任务设置限制了评价范围
- 证据只覆盖特定数据、模型、框架或规模
- 在已声明范围内结论成立, 扩展范围需要新的研究

名称不决定类别。应依据强基线、任务目标、实际权衡和证据判断。

## 4. 具体写局限

局限说明:

- 在什么条件下受限
- 技术、数据或协议原因是什么
- 它影响哪些结论或应用
- 与最相近工作的范围关系如何
- 后续工作需要改变机制还是补充证据

例如:

```text
The current evaluation focuses on [range] because [technical or data reason].
This setting is comparable to [current scope], but it does not establish [unsupported broader claim].
Evaluating or adapting [specific component] under [future setting] remains open.
```

## 5. 自检

- 是否重新回答了论文解决什么问题
- 是否总结核心认识而不是复制摘要
- 是否引用最有代表性的真实证据
- 是否把观察、解释和价值分开
- 局限是否具体且能够改变结论范围
- 方法缺陷是否被如实呈现
- 后续方向是否由当前限制自然推出
- 是否在结论中新增正文没有支持的主张
