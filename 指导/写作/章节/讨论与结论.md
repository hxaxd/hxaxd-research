# 讨论与结论

## 讨论负责解释结果改变了什么

目标场所设置独立讨论节时, 它综合多项证据回答更高一层的问题: 结果怎样改变引言提出的认识, 哪个机制解释最可信, 哪些替代解释仍然存在, 本文与最相近工作的关系是否因此更清楚, 以及结论在哪些条件下停止。

讨论的内容由结果中真正值得解释的张力决定。结果完全符合预期时可以简短; 出现不对称收益、成本取舍或意外失败时, 需要给这些现象足够空间。不能把相关性提升为因果, 也不能在讨论中补入正文尚未建立的新主张。

## 结论重新交付知识增量

结论让读者离开论文时记住问题、核心认识、最有代表性的证据和边界。可以参考以下信息关系, 并根据论文重心取舍:

```tex
\section{Conclusion}
% 本文解决的精确问题及核心认识
% 方法怎样响应这个认识
% 哪项证据最能支持最终判断
% 由此得到的知识或实际价值
% 会真正限制结论的边界与自然后续问题
```

```text
This work addresses [problem] by [method change].
The results support [main conclusion] under [evaluated conditions].
They also reveal [important mechanism, trade-off, or failure if established].
The current evidence does not establish [boundary].
```

示例中的信息按实际关系组织, 结论的力度来自正文已经建立的证据。

## 把限制写成能够校准结论的信息

局限用于校准论文结论, 说明受限条件、产生限制的技术/数据/协议原因, 以及它会改变哪些结论或应用。

区分两类情况有助于准确定位:

- **方法缺陷**: 在论文核心目标上输给公平强基线、依赖与目标场景冲突的假设, 或新增成本超过可验证收益
- **证据或范围边界**: 当前只评测了特定数据、模型、框架或规模, 更广结论尚未建立

名称本身不决定严重程度。同一现象在论文声称的核心范围内可能是方法缺陷, 在明确排除的范围外则只是尚未覆盖。

```text
The current evaluation covers [range] because [reason].
It therefore supports [bounded conclusion] but does not establish [broader claim].
Testing or changing [specific mechanism] under [future setting] remains open.
```

后续方向应由当前限制或新现象自然产生。泛泛的“未来扩展到更多场景”若不能说明为什么值得做, 可以省略。

可参考[关于局限和技术缺陷的讨论](https://github.com/pengsida/learning_research/issues/12)。完成后检查结论是否真的回答引言的问题, 是否把观察、解释与价值区分开, 以及是否出现正文没有支持的新主张。
