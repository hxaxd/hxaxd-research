# R5 论文写作

## 写作在整条研究链中的责任

R5 把已经确认的研究认识和证据组织成读者能够理解、质疑并最终接受的论证。它可以改变表达顺序、章节边界和详略; 任务、方法与结果沿用上游确认的真实内容。

上游各自产生不同内容:

- `叙事定稿.md`给出中心叙事、全文共用事实和承重主张
- `实验证明.md`给出方法比较边界、真实结果及证据允许的判断
- 原始文献支持外部事实和相关工作定位
- TeX 稿件决定这些内容最终怎样向读者展开

写作中发现缺口时, 按缺口本质定位责任:

| 缺少的内容 | 回到哪里 |
| --- | --- |
| 任务、失败、原因、核心认识、完整方法或相关工作边界 | R2 |
| 主张尚无足够的实验设计 | R3 |
| 结果尚未运行、无法复算或不可追溯 | R4 |
| 章节组织、论证节奏、措辞、引用或排版 | 留在 R5 |

用户可以要求先写局部稿件。当前证据不足的结论不进入编译后的正文, 可以用 TeX 注释标明真实缺口, 例如`% 待实验: C2 / E3`。

## 指导怎样配合

`指导/写作/论证与表达.md`提供跨章节的内容组织方法, 开始实质写作或结构修改时读取。具体章节只在当前任务涉及它时读取:

| 内容 | 指导 |
| --- | --- |
| 标题与摘要 | `指导/写作/章节/标题与摘要.md` |
| 引言 | `指导/写作/章节/引言.md` |
| 相关工作 | `指导/写作/章节/相关工作.md` |
| 方法 | `指导/写作/章节/方法.md` |
| 实验与结果 | `指导/写作/章节/实验与结果.md` |
| 讨论与结论 | `指导/写作/章节/讨论与结论.md` |
| 图表在稿件中的组织与排版 | `指导/写作/章节/图表.md` |
| 英文文字显得抽象、均匀或公式化 | `指导/写作/去模型化表达.md` |

需要新建、修改或审查真实科研图件时调用`指导/能力/科研配图.md`; 章节指导只处理图表怎样参与正文论证。

这些文档分别提供不同层次的判断角度和例子。根据当前稿件问题读取相应指导。

## 唯一权威稿件

已有官方模板或可用 TeX 稿件时沿用其主入口和目录结构; 尚无稿件时在研究项目目录创建`论文稿件.tex`。主入口及其直接包含的源文件共同构成唯一权威正文, 参考文献使用`references.bib`, 资源使用可移植相对路径, 编译产物进入项目的`build/`。

已有 Markdown、Typst 或 Word 内容时, 先识别其中仍然有效的研究内容, 再迁入 TeX。它们可以作为来源, 但不与 TeX 并行维护不同最新版。

TeX 环境按`指导/写作/TeX 环境配置.md`处理: 优先复用本机已能编译真实稿件的环境, 不可用时才在仓库内配置隔离环境。稿件尚未确定投稿场所时使用简单、完整的结构; 确定后服从官方模板。

## 推进一篇稿件

### 先确定读者最终要接受什么

在当前工作上下文中用一句话说清论文最希望读者记住的判断, 再判断已有章节在整篇论证中承担什么责任。由此检查各节是否共同推进同一个故事。

同时核对上游文档、实验配置、图表和已有正文对同一对象的称呼与边界。真实内容冲突时先解决来源, 不用统一措辞掩盖冲突。

### 根据证据依赖选择写作顺序

新稿通常适合先写最受事实约束的内容, 如问题设置、方法、实验设置与结果, 再写需要概括全文的相关工作、引言、结论、摘要和标题。这样可以减少在前言中预想一个后来没有被方法和结果支持的故事。

成熟稿件的局部修订、理论论文或证据结构不同的工作可以从最能暴露当前问题的章节开始。标题、摘要和引言最终根据完整正文与真实结果重新收束。

### 按论证责任组织章节

写到一个章节时, 先问该节结束后读者应该多理解或多相信什么, 再从上游事实中选择足够内容。目标模板规定版式, 论文类型和证据关系决定章节, 示例帮助寻找适合当前研究的节奏。

### 接入文献与图表

每个外部事实和相关工作判断回到原始来源, 正式稿件使用核验后的引用键。研究文档中的网页或本地路径只帮助追溯, 不直接成为对外论文的引用形式。

图表先有论证责任, 再决定形式。定量结果来自 R4 的正式结构化产物和确定性脚本; 图件完成后同步更新正文首次引用、图注、路径和结果解释, 使它直接参与论证。

### 从内容到语言逐层修改

先处理会改变结论的事实、主张和证据边界, 再处理全文结构、段落判断、句子与术语, 最后处理模型化表达、引用和版面。过早润色会让错误关系变得更顺, 也会增加后续重写成本。

完成一节或一版后, 用`指导/写作/论证与表达.md`的反向提纲检查成文是否仍能还原中心叙事与证据。若文字偏离就在 R5 重写; 若真实结果改变了研究认识, 由用户决定返回 R2 或 R3。

### 编译真实稿件

每轮实质修改后编译主入口, 处理真实源文件中的公式、引用、交叉引用、图件、脚注和附录错误, 并查看 PDF 是否存在溢出、遮挡、乱码或不可读图表。编译和机械检查可以在模型上下文之外完成, 但修复必须作用于权威源文件。

## 与用户共同定稿

用户需要决定的是作者真正承担的研究判断: 标题、摘要和引言是否表达其本意, 与最相近工作的差异是否准确, 核心结果、成本、失败和限制怎样解释, 以及当前版本是否面向某一投稿场所。局部语言与排版修正可以直接迭代。

当 TeX 权威稿件能够稳定编译, 全文能够还原同一中心叙事和证据链, 未支持内容没有混入对外正文, 并且用户认可当前研究表述时, R5 才适合进入 R6。投稿前需要独立攻击稿件时调用`指导/能力/学术审阅与反向论证.md`。
