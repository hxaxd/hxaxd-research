# R5 论文写作

## 1. 做什么

R5 把已经确认的研究叙事、分点核心主张和真实实验结果写入唯一权威的 TeX 论文稿件。默认主文件是研究项目目录中的`论文稿件.tex`; 已有官方 TeX 模板时沿用其主入口文件。

Agent 直接维护 TeX 源文件及其真实依赖, 通过完整稿件上下文和用户对研究的整体把握完成跨章节协调。

R5 可以调整论文表达和章节组织, 不能为了写得顺而改变已经确认的研究事实。若真实结果要求实质性修改 Story、核心主张或证明范围, 应返回 R2 或 R3 与用户确认, 而不是在稿件中悄悄改写。

## 2. 输入

- `研究想法.md`
- `叙事定稿.md`
- `实验证明.md`, 若研究包含实验
- 实验仓库中的正式结果、图表、运行清单和必要日志
- 被实际使用的原始论文、官方文档或其他一手来源
- 用户指定的论文语言、目标读者、论文类型和投稿场所, 若已确定
- 已有稿件内容或官方 TeX 模板, 若项目已经开始写作
- 可用的 TeX 编译环境; 按`指导/写作/TeX 环境配置.md`探测或配置

没有目标投稿场所时可以先写内容完整的通用稿; 已确定目标场所时, 只吸收会影响篇幅、结构和匿名性的稳定要求。动态投稿规则由 R6 在准备投稿时重新核验。

## 3. 稿件与编译环境

进入 R5 时先确定 TeX 主入口:

- 已有可用 TeX 稿件或官方模板时, 沿用其主入口和目录结构
- 尚无稿件时, 在研究项目目录创建`论文稿件.tex`
- 已有 Markdown、Typst 或 Word 内容时, 先确认迁移范围, 再把当前有效内容转入 TeX 稿件
- 参考文献使用`references.bib`; 图件、表格和拆分附录按 TeX 稿件的实际依赖组织

TeX 主入口及其直接包含的源文件共同构成唯一权威稿件。R1–R4 的 Markdown 文档作为研究依据, R5 的论文内容统一进入这套 TeX 源文件。

首次写作或编译前读取`指导/写作/TeX 环境配置.md`。本机已有可工作环境时直接使用; 当前环境无法编译时, 在仓库`.tools/tex/`中配置项目内环境。编译输出写入研究项目的`build/`目录。

稿件模板优先采用目标投稿场所的官方模板; 尚未确定投稿场所时, 论文结构由研究 Story 和证据决定。

## 4. 执行

### 1. 先确认结果是否仍支持原叙事

写作前同时阅读`叙事定稿.md`与 R4 的真实交付, 判断:

- 核心冲突是否仍然成立
- 方法真正解决了哪一部分问题
- 哪些分点主张获得支持、被削弱、被推翻或仍不能判定
- 最有解释力、最有吸引力的主结果是什么
- 原先强调的贡献是否应降级, 是否出现了更准确的主线

局部措辞和章节重排可在 R5 直接处理。若需要改变论文的中心 Story、删除或新增核心主张、改变实验判据, 先与用户确认并回写 R2/R3 的真实研究记录。

### 2. 围绕一条主线组织论文

论文不是“几个共识、几个问题、几个方法、几个原因”的固定拼装。先找到能让读者持续追问下去的一条因果或认识链:

1. 读者原本如何理解这个问题
2. 哪个关键现象或矛盾使这种理解不再充分
3. 最相近的方法为什么仍受一个精确边界限制
4. 本文抓住了什么机制或表示上的缺口
5. 方法如何直接回应这个缺口
6. 哪组结果最有力地改变读者的判断

这条链只用于检验章节功能, 不要求机械对应六段。若某一环不能用已有事实和证据说清, 优先修正 Story, 不用修辞掩盖。

章节提纲在对话中与用户确认, 随后直接落实到 TeX 稿件结构。

### 3. 选择合适的写作顺序

默认优先写最受事实约束的部分:

- 方法、设定和实验协议
- 结果、消融、分析和限制
- 相关工作
- 引言
- 摘要与标题

这样可以让引言与摘要围绕真实结果收束。若论文类型或现有稿件更适合其他顺序, Agent 可以调整。

### 4. 写引言

引言应完成必要的认识推进, 但不采用固定段落模板或固定数量的共识、问题、方法和贡献。

重点检查:

- 开头建立的是与本文核心冲突直接相关的背景, 不是宽泛领域常识
- 问题来自具体矛盾、失败模式或未满足条件, 不是为了引出方法而临时制造
- 最相近工作被准确而公平地描述, 撞车部分不隐藏
- 缺口足够精确, 能自然推出本文方法的关键设计
- 方法介绍说明“为什么这样做”, 不只罗列组件
- 贡献条目对应可辨认的知识增量或证据, 不把工程步骤拆成多个贡献
- 摘要性的实验结论只使用已经完成的真实结果和正确口径
- 引言结束后, 读者能够预测后文为什么需要这些方法与实验

如果最吸引人的 Story 与原有章节顺序冲突, 可以重组论文; 不能为了吸引力夸大问题、贬低相关工作或越过实验边界。

### 5. 写相关工作

相关工作承担的是定位和区分, 不是罗列引用。围绕与本文主张有关的技术路线、表示方式、优化目标或假设分组, 并明确:

- 哪些工作与本文解决同一问题、采用相近机制
- 哪些工作目标相同但机制不同
- 哪些工作机制相近但使用场景或假设不同
- 最相近工作的真实重合点与剩余差异
- 本文实验中的主要 Baseline 与文字定位是否一致

每组文献都应服务一个比较判断。只有背景关系而不能影响本文定位的工作可以简写或删除。不要通过换名制造差异, 也不要把同类工作分散到多个小节以弱化相似性。

研究文档中的事实直接引用原始来源。正式稿件使用经过核验的引用键和`references.bib`; 每个关键定位判断应能回到原始论文。对外稿件不得保留本地文件链接。

### 6. 写方法、实验与结果

方法部分从主张需要的机制出发组织, 不按代码模块顺序抄写。每个关键设计应能回答:

- 它解决哪一个已陈述的问题
- 如果去掉它, 预期会失去什么
- 它与最相近方法的实质差异是什么
- 实验中如何单独观察或控制它的作用

实验部分以 R3 的共同实验协议和 R4 正式运行产物为事实来源。共享运行可以在论文中服务多个分析, 但不得重复包装成多个独立实验贡献。

结果表述应区分:

- 直接观察到的结果
- 由对照或消融支持的机制解释
- 合理但尚未被实验排除的推断
- 已知限制和不适用范围

这些边界直接在正文中写清。

### 7. 构造科研图件

当问题、方法关系、核心结果或失败边界通过视觉表达会明显更容易理解或验证时, 调用`指导/能力/科研配图.md`。

不要先按论文结构机械规定 Figure 1、Figure 2 和 Figure 3。每张图先对应一条明确主张或解释责任, 再决定使用表格、确定性数据图、概念图或真实案例图。最终图件写入稿件实际图件目录, 图注和正文引用直接进入 TeX 源文件。

### 8. 迭代、编译与检查

每轮修改优先处理会改变读者理解的问题:

- 主线是否连续
- 每节是否承担独立且必要的功能
- 核心术语是否稳定
- 方法、实验和贡献是否一一对应
- 图表是否在第一次需要它的位置出现
- 相关工作是否公平且足以支撑差异判断
- 摘要、引言、结论中的数字和范围是否与正文一致

语言润色放在结构与论证稳定之后。高能力模型直接完成跨章节一致性检查, 用户负责确认 Story、贡献和结果口径。

每轮实质修改后使用当前 TeX 环境编译主入口, 将产物写入`build/`, 并检查:

- TeX 源文件能够完整解析
- 公式、图表、脚注、附录和交叉引用位置正确
- 引用键均能解析, 参考文献内容与原始来源一致
- 图件路径、字体和最终尺寸可用
- 编译日志中没有会影响正文完整性的错误或未解析引用
- 生成的 PDF 中没有溢出、遮挡、乱码或明显版面异常

检查结果直接用于修正 TeX 源文件, 直到当前稿件能够稳定重编译。

### 9. 与用户定稿

至少让用户确认:

- 论文最核心的一句话 Story
- 最终贡献范围和最相近工作的差异
- 负结果、限制和不确定结果如何呈现
- 稿件语言与目标投稿场所
- 对外版本是否可以进入 R6

若用户需要投稿前模拟审稿或主动攻击当前论文, 调用`指导/能力/学术审阅与反向论证.md`。审阅先保持只读并给出稳定判断, 用户决定采纳后再返回 R5 修改实际稿件。

## 5. 产出

R5 的正式产出是可编译的 TeX 稿件及其真实依赖:

- TeX 主入口文件, 默认`论文稿件.tex`
- 稿件实际使用的拆分 TeX 源文件、图、表、`references.bib`和附录源文件
- `build/`中的当前编译 PDF 和必要日志

## 6. 确认点

- Story 或分点核心主张需要实质修改时
- 真实结果不足以支持原定贡献或需要改变结果口径时
- 选择正式英文写作或目标投稿场所时
- 大幅删除、合并或重排论文核心内容时
- 稿件准备进入对外投稿版本时

## 7. 完成条件

- 项目中存在唯一权威的 TeX 主入口及其真实依赖
- 当前 TeX 环境已经按配置指导验证
- 稿件能够从 TeX 源文件稳定编译为 PDF
- 稿件 Story 与用户确认的研究判断一致
- 核心主张均由正文论述、原始文献或真实实验结果支撑
- 引言、相关工作、方法、实验和结论形成连续论证
- 最相近工作与本文差异被准确呈现
- 结果数字、范围、图表和引用在整篇稿件中一致
- 引用键和交叉引用已经解析
- 没有伪造结果、引用、原文位置或论文能力
- 用户确认可以进入 R6

## 8. 常见失败

- **模板化叙事**: 强行凑固定数量的共识、问题、方法和贡献
- **组件即贡献**: 把实现模块逐项列成知识贡献
- **结果倒灌**: 为了匹配好看的 Story, 改写 R2 主张或 R3 判据
- **相关工作堆砌**: 罗列大量文献, 却不说明最相近工作的重合与差异
- **代码顺序写方法**: 按类和函数介绍系统, 读者看不出设计理由
- **源格式漂移**: 研究内容分散在多个稿件格式中, TeX 主稿没有吸收最新修改
- **只写不编译**: 源文看似完整, 实际 PDF 存在引用、图件或版面错误
