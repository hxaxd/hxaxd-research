# R5 论文写作

## 1. 做什么

R5 把用户确认的中心叙事、论文事实、分点核心主张、原始文献和真实实验结果展开为唯一权威的 TeX 论文稿件。

本文只负责 R5 与整个仓库怎样配合、稿件怎样推进和最终交付什么。跨章节论证见`指导/写作/论证与表达.md`; 语言定稿见`指导/写作/去模型化表达.md`; 每个论文章节的具体写法和示例见`指导/写作/章节/`。

R5 可以调整章节组织和表达, 但不重新决定研究事实。需要改变任务、中心矛盾、核心认识、完整方法或分点主张时返回 R2; 需要改变经验结论、实验比较或成立范围时返回 R3/R4。

## 2. 输入

- `叙事定稿.md`, 包括中心叙事、论文事实和分点核心主张
- `实验证明.md`及其引用的正式实验产物, 若研究包含实验
- 被正文实际使用的原始论文、官方文档或其他一手来源
- 用户指定的论文语言、目标读者、论文类型和投稿场所, 若已确定
- 已有稿件、官方 TeX 模板和图表源文件, 若存在
- 按`指导/写作/TeX 环境配置.md`探测或配置的可用编译环境

开始写作前判断缺口归属:

- 任务、输入输出、关键失败、技术原因、核心认识、方法关系或相近工作边界不清楚: 返回 R2 澄清`叙事定稿.md`
- 主张缺少实验设计: 返回 R3
- 实验尚未执行或正式结果不可追溯: 返回 R4
- 事实已经稳定, 只是章节组织、文字、引用或图表问题: 留在 R5

用户明确要求先写局部稿件时, 只展开当前证据已经支持的内容。未完成的结果与结论只使用 TeX 注释标识, 例如`% 待实验: C2 / E3`, 不进入编译后的对外正文。

## 3. 写作指导路由

开始 R5 时先读`指导/写作/论证与表达.md`。随后只在处理相应内容时读取对应章节指导:

| 当前任务 | 指导 |
| --- | --- |
| 标题或摘要 | `指导/写作/章节/标题与摘要.md` |
| 引言 | `指导/写作/章节/引言.md` |
| 相关工作 | `指导/写作/章节/相关工作.md` |
| 方法 | `指导/写作/章节/方法.md` |
| 实验与结果 | `指导/写作/章节/实验与结果.md` |
| 讨论或结论 | `指导/写作/章节/讨论与结论.md` |
| 图表排版、图注和图文接入 | `指导/写作/章节/图表.md` |
| 英文定稿与去除模型化表达 | `指导/写作/去模型化表达.md` |

需要新建、修改或审查真实科研图件时调用`指导/能力/科研配图.md`; 章节指导只负责图表在稿件中的内容与呈现。

## 4. 稿件与格式

进入 R5 时先确定 TeX 主入口:

- 已有官方模板或可用稿件时沿用其主入口和目录结构
- 尚无稿件时在研究项目目录创建`论文稿件.tex`
- TeX 主入口及其直接包含的源文件共同构成唯一权威稿件
- 参考文献使用`references.bib`
- 图件、表格和拆分附录按稿件真实依赖组织
- 稿件资源使用可移植相对路径, 不写入本机绝对路径
- 编译产物写入研究项目的`build/`

已有 Markdown、Typst 或 Word 内容时, 先确认需要保留的当前有效内容, 再迁入 TeX 权威稿件。正式论文内容不在多个源格式中并行维护。

稿件采用目标投稿场所的官方模板。尚未确定投稿场所时, 使用能够完整表达当前论文的简单 TeX 结构; 摘要环境、章节命令、参考文献和补充材料最终服从官方模板。

## 5. 执行

### 1. 对齐上游事实

同时读取`叙事定稿.md`和`实验证明.md`, 建立当前写作所需的对应关系:

```text
中心叙事
-> 论文事实与贡献性设计
-> 分点核心主张
-> 文献或实验依据
-> 稿件章节与图表
```

核对同一对象在上游文档、实验配置、图表和已有正文中的名称。发现冲突时定位真实来源并先解决冲突。

### 2. 确定全文结构

用一句话确认全文中心判断, 再为每一节写一句论证责任。章节由当前论文类型、中心叙事和证据依赖决定; 目标模板只提供版式边界。

需要用户确认:

- 全文最希望读者记住的判断
- 核心认识与完整方法的关系
- 最相近工作的真实差异
- 负结果、限制和不确定结果的呈现范围
- 会改变论文定位的章节选择

### 3. 按证据依赖写作

默认先写最受事实约束的部分:

1. 问题设置、方法和实现事实
2. 实验设置、结果、消融、分析和限制
3. 相关工作
4. 引言
5. 讨论或结论
6. 摘要与标题

已有成熟稿件或论文类型适合其他顺序时可以调整。无论采用什么顺序, 标题、摘要和引言最后都要按完整正文与真实结果重新收束。

每到一个章节, 读取`指导/写作/章节/`中的对应文档, 选择符合当前论文事实的写法。示例骨架用于检查语义角色, 不要求固定段数、句数或贡献数量。

### 4. 接入文献和图表

正式稿件使用核验后的引用键。每个外部事实、相关工作定位和比较判断都能够回到原始来源; 对外稿件不保留研究工作区中的本地文献路径。

图表先确定支持的判断, 再制作和接入。定量结果来自 R4 的正式产物和确定性绘图脚本。完成图件后同步更新文件路径、正文首次引用、图注和结果解释。

### 5. 反向检查与修改

按`指导/写作/论证与表达.md`从成文反向提取全文、章节和段落判断, 检查它们是否仍能还原用户确认的中心叙事与证据链。

修改顺序是:

1. 事实、主张与证据边界
2. 全文结构与章节依赖
3. 段落判断与证据
4. 句子、术语和衔接
5. 按`指导/写作/去模型化表达.md`完成英文定稿
6. 图表、引用和版面

### 6. 编译与检查真实稿件

每轮实质修改后使用当前 TeX 环境编译主入口, 并检查:

- TeX 源文件、公式、图表、脚注和附录能够解析
- 引用键与交叉引用完整
- 图件路径、字体和最终尺寸可用
- 编译日志没有影响正文完整性的错误
- PDF 中没有溢出、遮挡、乱码或明显版面异常
- 标题、摘要、图表和正文传达同一中心判断

机械检查可以在模型上下文之外执行; 发现的问题直接修复真实 TeX 源文件。

### 7. 与用户定稿

至少请用户确认:

- 标题、摘要和引言是否表达其真正认可的研究判断
- 贡献范围和与最相近工作的差异
- 核心结果、成本、失败和限制的解释
- 稿件语言、目标投稿场所和对外版本
- 当前稿件是否可以进入 R6

用户要求投稿前模拟审稿或主动攻击论文时, 调用`指导/能力/学术审阅与反向论证.md`; 审阅完成后再根据用户选择修改稿件。

## 6. 产出

R5 的正式产出是可编译的 TeX 权威稿件及其真实依赖:

- TeX 主入口, 默认`论文稿件.tex`
- 稿件实际使用的拆分 TeX、图、表、`references.bib`和附录源文件
- `build/`中的当前 PDF 和必要编译日志

## 7. 完成条件

- 项目中存在唯一权威 TeX 主入口及其真实依赖
- 稿件可以在已验证环境中稳定编译
- 全文能够反向还原用户确认的中心叙事、论文事实和证据链
- 标题、摘要、引言、相关工作、方法、实验和结论职责闭合
- 方法、数字、图表、引用、术语和成立范围在全文一致
- 最相近工作与本文差异被准确呈现
- 未完成或未支持内容没有进入对外正文
- 用户确认可以进入 R6

## 8. 常见失败

- **上游未清便扩写**: 到摘要或引言才临时发明任务、原因和贡献关系
- **入口承担所有写法**: 阶段协作、章节方法、示例和语言规则挤在一个文档中
- **章节齐全但论证断裂**: 各节分别像论文, 合起来却不能还原同一中心判断
- **结果倒灌**: 为迁就结果静默改写 R2 的叙事或 R3 的判据
- **多源稿件**: TeX、Markdown 和 Word 各自保留不同最新版
- **只改文字不改证据关系**: 用润色掩盖事实、主张或实验缺口
- **只写不编译**: 源文件存在, 真实 PDF 的引用、图件或版面不可用
