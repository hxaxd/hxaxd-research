# 论证与表达

## 1. 责任

本文负责 R5 中跨章节重复使用的内容组织与语言修改方法。各章节具体写法见`指导/写作/章节/`; TeX 稿件、仓库协作与交付顺序见`指导/写作/论文写作.md`。

论文写作的基本单位不是章节名, 而是判断及其证据。写作时同时检查四个尺度:

- 全文围绕一个读者最终应当接受的中心判断
- 每节完成一项不可替代的论证责任
- 每段交付一个能够单独复述的判断
- 每句提供理解、支持、限定或连接该判断所需的信息

## 2. 从论文事实建立内容链

写作以`叙事定稿.md`中的论文事实、中心叙事和分点核心主张为上游基线, 以`实验证明.md`中的实际交付和当前判定为经验事实。

每项贡献性设计应能够还原为同一条关系:

```text
目标技术问题或原因
-> 设计选择
-> 实际运行机制
-> 可检验的技术优势
-> 直接证据
-> 成立边界
```

这条关系可以在不同章节采用不同详略, 但不能改变含义。引言解释为什么需要它, 方法说明它怎样工作, 实验验证它改变了什么, 结论只保留证据允许的认识。

先区分四类信息:

- **核心认识**: 本文改变了读者对问题的什么理解
- **贡献性设计**: 为实现该认识而提出的机制、表示、目标或推理方式
- **实验发现**: 真实数据揭示的效果、过程、失败和边界
- **实现细节**: 复现所需但不构成知识增量的工程选择

## 3. 组织全文、章节与段落

### 全文

先用一句话写出本文希望读者最终记住的判断, 再检查整篇论文是否形成以下认识变化:

1. 读者原本怎样理解目标问题
2. 哪个真实矛盾使这种理解不再充分
3. 最相近路线为什么仍受一个精确条件限制
4. 本文发现了什么缺口或关键关系
5. 方法怎样直接回应这个缺口
6. 哪组证据最有力地改变读者判断
7. 该判断在哪些条件下成立和停止

### 章节

为每节写一句功能陈述。功能陈述描述本节完成后读者多知道或多相信了什么, 不只是“本节介绍方法”或“本节展示实验”。章节顺序由论证依赖决定。

### 段落

一段只交付一个主要判断:

- 首句尽早说明本段对象、问题或判断
- 后续句承担解释、证据、细化、比较或限定
- 末句交付本段结论, 或建立下一段不可缺少的前提
- 段落长度服从内容, 不追求外观整齐

[PLOS Computational Biology 的论文结构指南](https://doi.org/10.1371/journal.pcbi.1005619)提出可在论文、章节和段落尺度检查“上下文—内容—结论”。这里把它作为信息功能检查: 先提供理解新信息所需的上下文, 再交付内容和证据, 最后说明得到什么判断。它不规定固定句数。

### 句子

- 让真正执行动作或承担判断的对象成为清楚的主语
- 用准确动词表达动作, 让评价落到具体对象、条件或结果
- 新术语出现前提供理解它所需的对象、范围或比较
- 已知信息靠前, 新信息与句子重心靠后
- 同一概念稳定使用同一名称和符号
- 因果、转折、并列和限定先有真实关系, 再使用连接方式
- 一个句子只承担读者能够一次处理的判断关系

## 4. 反向提纲

完成一节或一版稿件后, 从成文反向提取:

- 全文的一句话中心判断
- 每节的一句话论证责任
- 每段的一句话判断
- 支持该判断的事实、证据、推理或限定
- 相邻段落之间的必要关系

若反向提纲与`叙事定稿.md`不一致, 先判断是文字结构偏离, 还是真实结果已经改变研究认识。前者在 R5 重写; 后者返回 R2 或 R3 由用户确认。

## 5. 修改顺序

论文修改按影响范围从大到小进行:

1. 事实、主张与证据边界
2. 全文中心叙事和章节依赖
3. 段落判断与证据关系
4. 句子信息、术语和衔接
5. 模型化表达与语言自然度
6. 图表、引用和版面

语言自然度来自准确内容与真实作者判断。最终文字应能够反向还原到研究事实和证据, 同时保留作者真正愿意承担的判断。
