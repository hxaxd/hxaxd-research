# 论证与表达

## 论文写作是在安排认识变化

论文安排一条让读者从已有认识走向新判断的路径。章节、段落和句子的作用都从这条路径中产生:

- 全文最终改变读者对什么问题的判断
- 当前章节为这个变化补上哪一项必要认识
- 当前段落交付哪个局部判断
- 当前句子为理解、支持、限定或连接该判断提供什么

这些尺度帮助定位问题。篇幅和结构服从信息复杂度与证据重量。

## 从研究内容中建立论证

`叙事定稿.md`提供中心叙事、论文事实与承重主张, `实验证明.md`提供经验事实和成立范围。正文根据读者在每个位置已经知道什么, 选择此刻需要揭示的关系。

贡献性设计往往包含以下逻辑:

```text
问题或技术原因
-> 设计选择与运行变化
-> 可观察后果
-> 证据与成立边界
```

引言可能只需要其中的因果主线, 方法节展开运行机制, 实验节展示可观察后果, 结论提炼证据允许的新认识。不同章节的详略可以不同, 关系的含义不能漂移。

写作时尤其要区分:

- **核心认识**: 本文使问题理解发生了什么变化
- **贡献性设计**: 为实现这种认识提出了什么机制
- **实验发现**: 数据实际揭示了什么效果、过程或失败
- **实现细节**: 复现所需但不构成知识增量的选择

混淆这些层次会让实现步骤被包装成贡献, 或让实验观察被倒写成事先已知的机制事实。

## 设计全文和章节

可以先尝试写出一句全文中心判断, 然后问: 一个持合理怀疑态度的目标读者, 在接受它之前还缺哪些认识? 常见缺口包括任务条件、真实失败、现有路线的边界、核心认识、方法响应、直接证据和适用范围, 但每篇论文需要的组合与顺序不同。

为每节写一句临时功能说明有助于检查结构。例如“本节说明上下文晚绑定如何避免把项目知识固化进流程记忆”比“本节介绍方法”更能约束内容。这句话只在当前写作上下文中辅助判断。

章节顺序服从依赖关系: 读者理解后一项判断前必须知道什么。若两个小节能够互换而不影响理解, 它们可能只是分类并列; 若论文中心依赖二者的因果或递进关系, 正文应把这种关系写出来。

## 形成自然的段落节奏

一段通常围绕一个可复述的主要判断展开。读者先获得理解该判断所需的上下文, 再看到内容或证据, 最后知道这对当前论证意味着什么。[PLOS Computational Biology 的论文结构指南](https://doi.org/10.1371/journal.pcbi.1005619)把这种功能概括为“上下文—内容—结论”。具体句数和节奏由该段信息决定。

实际写作中:

- 首句尽早让读者知道对象、问题或判断, 除非前置条件确实需要先建立
- 中间句只保留解释、证据、比较和限定所需的信息
- 末句可以交付局部结论, 也可以自然建立下一段的必要前提
- 真实并列使用并列结构, 因果和转折则写清条件, 不用连接词制造关系

如果一段同时在定义任务、批评三类工作、提出方法并报告结果, 通常需要重新找出它真正承担的判断。反过来, 两段若只是换词重复同一结论, 应合并或删除其中一段。

## 让句子承载真实关系

句子清楚通常来自内容关系清楚:

- 让实际执行动作或承担判断的对象成为明确主语
- 用准确动词说明发生了什么, 把评价落到条件、机制或结果
- 已知信息靠前, 新信息与句子重心靠后
- 新术语出现前提供理解它所需的对象和范围
- 同一概念稳定使用同一名称、符号与数值口径
- 一个长句只有在关系必须一起读取时才保留为长句

句子长度和段尾方式反映信息结构与相邻关系。

## 用反向提纲诊断成文

完成一节或一版后, 从已有文字反向提取全文中心、各节责任和各段判断, 再为每个判断指出实际证据或限定。反向提纲用于暴露成文真正说了什么, 在当前修订上下文中完成即可。

若提取出的故事与`叙事定稿.md`不同, 判断差异来自两种情况:

- 写作偏离了确认后的研究认识: 在 R5 重组
- 新结果确实改变了研究认识: 返回 R2 或 R3 由用户决定

修改按影响范围从大到小进行: 先事实与证据边界, 再全文和章节依赖, 然后段落判断、句子与术语, 最后处理语言风格、图表、引用和版面。这样每次精修都建立在仍然有效的内容上。
