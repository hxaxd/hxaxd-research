# 叙事定稿

`叙事定稿.md`是 R2 的唯一正式产物, 用来保存论文叙事和分点核心主张。

## 固定节

产出文件必须严格遵循以下骨架, Agent 绝对不能自行增减一级节。

```markdown
# 叙事定稿

## 叙事

用连续中文段落说明:

- 当前领域或任务中已经成立的背景
- 在什么场景下存在什么问题
- 这个问题为什么重要
- 现有工作为什么没有充分解决
- 本文为什么选择这个切入点

使用文献时在句子后标注`[文献:E1]`; 对应条目必须存在于`文献证据库.md`。未核验内容直接标为待核验。

## 分点核心主张

每条分点核心主张使用三级标题, 不使用表格。

对应部分决定证据责任:

- 共识: 可以只用文献引用和论述证明
- 问题: 可以用文献引用和论述证明, 也可以用实验或现象观察证明
- 方法: 通常需要实验结果证明
- 原因: 通常需要可解释性分析、机制观察、消融、案例或中间过程证据

### C1 主张标题

- 分点主张:
- 对应部分: 共识 / 问题 / 方法 / 原因
- 需要的证据:
- 削弱或推翻条件:

### C2 主张标题

- 分点主张:
- 对应部分: 共识 / 问题 / 方法 / 原因
- 需要的证据:
- 削弱或推翻条件:
```
