# 叙事定稿

`叙事定稿.md`是 R2 的唯一正式产物, 用来保存用户确认的中心叙事、下游共享的论文事实和支撑叙事成立的分点核心主张。

## 固定节

产出文件必须严格遵循以下骨架, Agent 绝对不能自行增减一级节。

```markdown
# 叙事定稿

## 叙事

用连续中文段落写出一条已经由用户确认的中心叙事。

叙事应根据研究本身自然组织, 不采用固定段落数量或填槽顺序。它需要让读者理解:

- 研究面对的精确场景和真实矛盾
- 现有理解或解决方式在何处不足
- 这项研究带来的关键认识变化
- 方法为什么从这个认识中自然产生
- 如果叙事成立, 会改变什么问题理解、解决方式或能力边界

以上是内容检查, 不是五段式模板。不得按 R1 的共识、问题、方法、原因分别扩写, 也不得用“首次、重要、显著”代替真实冲突。

使用文献时在相关句子附近直接引用: 已有可靠网页来源时, 用论文标题或简称作为 Markdown 链接; 暂无网页来源时, 直接写论文标题并标记`链接待补`, 相关判断在核验前标为待核验。

## 论文事实

### 任务与目标

- 研究对象:
- 输入与输出:
- 真实场景:
- 最重要的评价目标:

只写当前论文实际需要的任务事实。目标读者已经熟悉的常识可以简写。

### 关键失败与技术原因

- 失败对象与发生条件:
- 可观察表现与后果:
- 技术原因:

明确哪些内容已有外部来源或观察支持, 哪些是本文准备验证的机制判断。关键外部事实在使用位置直接引用。

### 核心认识与完整方法

- 核心认识:
- 完整方法:

按实际数量继续写贡献性设计。每项写清:

#### 贡献性设计一

- 回应的问题或原因:
- 设计选择:
- 运行机制:
- 预期改变的可观察性质:
- 后续需要的证据:
- 当前边界:

### 相近工作与主张边界

- 最相近工作及真实重合:
- 本文改变的机制、信息边界、目标或适用条件:
- 当前明确主张:
- 当前明确不主张:

## 分点核心主张

这里只保留中心叙事成立所必需的承重主张。主张数量和类型由叙事决定, 不固定数量, 不要求覆盖共识、问题、方法、原因四类。

每条主张使用三级标题, 不使用表格。

### C1 主张标题

- 分点主张:
- 叙事作用:
- 需要的证据:
- 削弱或推翻条件:
```
