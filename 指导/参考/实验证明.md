# 实验证明

`实验证明.md`是 R3 与 R4 共用的正式研究记录。R3 定义受测方法、比较边界、共同协议、实验组合和分点判据; R4 在同一文件中回填实际交付与当前判定。

文档以实验家族形成论证节奏, 再用分点主张的证据映射检查闭环。实验编号是论证单元, 共享运行是执行单元; 两者不要求一一对应。

## 固定节

产出文件必须严格遵循以下一级节骨架, Agent 不得自行增减一级节。三级实验条目根据研究需要增减, 共同实验协议和方法矩阵可以使用表格。

```markdown
# 实验证明

## 方法与比较边界

### 完整方法

说明正式评测的完整方法由哪些部分组成, 各部分如何配合, 训练或进化阶段产生什么, 运行阶段读取什么, 哪些状态更新, 哪些制品冻结。

### 方法变体

列出正式实验使用的方法变体及稳定名称。说明每个变体相对完整方法只改变什么、用于回答什么问题。

### 比较基线

列出强公开方法、基本对照、资源匹配对照或必要上界。逐项说明纳入理由、来源、复现方式、信息访问、模型能力、预算和公平性。

### 总体比较目标

说明主实验准备证明的总体竞争力, 是主要效果领先、可比协议下的 SOTA、效果—成本优势, 还是其他预先定义的联合目标。

## 共同实验协议

固定任务与数据版本、划分和泄漏边界、评测单位、主要/次要/机制/资源指标、模型角色、执行框架、预算、冻结与迁移规则、冒烟测试/预跑/正式运行边界、重试与排除规则、统计单位、重复次数、聚合方法和成本口径。

可以使用方法 × 模型 × 执行框架 × 数据条件的实验矩阵。只固定会影响研究结论的条件, 工程实现由 R4 决定。

## 实验组合

### E1 主实验: 实验标题

- 证据等级: 核心
- 服务主张: C1, C2

#### 研究问题

说明完整方法与强基线的总体比较要回答什么。

#### 设计与对比

说明参与的方法、任务、条件、共享运行和公平控制。

#### 重点观察

按优先级说明主要结果、关键成本、必要子集结果和机制现象。主实验首先观察总体竞争力。

#### 预定交付

说明 R4 需要提供的结构化汇总、子集或种子级结果、成本、配置、运行引用和分析产物。结果责任应覆盖直接观察、成立条件、不确定性、允许的解释、仍未排除的替代解释、主张边界, 以及适合生成主表或主图的真实材料。

#### 实际交付

待执行。R4 回填直接观察、比较与统计条件、允许的解释、不能推出的结论、主张影响和真实产物路径。

### E2 分点深化 / 消融 / 迁移 / 效率 / 诊断: 实验标题

- 证据等级: 核心 / 强化 / 探索
- 服务主张: C2

#### 研究问题

说明该实验单独回答的精确问题。

#### 设计与对比

说明相对共同协议改变的唯一因素、对照、干预、共享运行和替代解释控制。

#### 重点观察

按优先级说明与研究问题直接对应的指标、现象、成本或失败类型。

#### 预定交付

说明 R4 需要提供的真实结果和可追溯产物, 包括图表或案例需要的结构化材料。

#### 实际交付

待执行。R4 回填直接观察、比较与统计条件、允许的解释、不能推出的结论、主张影响和真实产物路径。

按实际实验数量继续增加 E3、E4 等三级标题。除主实验外, 实验类型和数量由分点主张决定。
```
