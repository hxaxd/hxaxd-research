# 实验证明

`实验证明.md`是 R3 的正式产物, 用来保存分点证明方案和 R4 需要回填的结果交付要求。

## 固定节

产出文件必须严格遵循以下骨架, Agent 绝对不能自行增减一级节。

```markdown
# 实验证明

## 分点证明方案

每条需要经验支持的分点主张沿用`叙事定稿.md`中的三级标题, 不使用表格。主张标题本身就是实验项标题。

### C1 主张标题

#### 实验设计

自由描述这条主张准备怎么做实验。可以包括 Benchmark 或数据、指标或中间现象、Baseline 或对照等; 具体写成段落、分点还是更细的小标题, 由 Agent 根据实验本身决定。

#### 预期证据

自由描述什么样的结果会支持、削弱或推翻这条主张。具体写法由 Agent 根据主张和实验决定。

### C2 主张标题

#### 实验设计

自由描述这条主张准备怎么做实验。可以包括 Benchmark 或数据、指标或中间现象、Baseline 或对照等; 具体写成段落、分点还是更细的小标题, 由 Agent 根据实验本身决定。

#### 预期证据

自由描述什么样的结果会支持、削弱或推翻这条主张。具体写法由 Agent 根据主张和实验决定。

## 结果交付要求

结果交付要求沿用分点证明方案中的三级标题, 不另起实验编号, 不使用表格。R3 先写需要交付什么, R4 只在对应条目中回填真实状态、结果摘要和产物路径。

### C1 主张标题

自由描述 R4 需要交付什么, 以及 R4 执行后应该回填什么。可以包括主要指标、原始结果、日志和配置、结构化汇总、种子级/样本级/子集级结果、资源消耗、实际状态、结果摘要、产物路径、与判据的对应、当前不能得出的结论等; 具体结构由 Agent 根据实验本身决定。

### C2 主张标题

自由描述 R4 需要交付什么, 以及 R4 执行后应该回填什么。可以包括主要指标、原始结果、日志和配置、结构化汇总、种子级/样本级/子集级结果、资源消耗、实际状态、结果摘要、产物路径、与判据的对应、当前不能得出的结论等; 具体结构由 Agent 根据实验本身决定。
```
