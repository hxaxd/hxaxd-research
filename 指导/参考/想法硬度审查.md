# 想法硬度审查

`想法硬度审查.md` 是 R2 对候选 Story 和核心主张的红队记录, 用来决定哪些主张能进入`叙事定稿.md`。

## 推荐结构

```markdown
# 想法硬度审查

## 审查对象

- 候选 Story:
- 候选核心主张:
- 相关证据编号:

## 六类审查

| 维度 | 结论 | 证据/反例 | 必须修改 |
|---|---|---|---|
| 知识正确性 | | | |
| 问题重要性 | | | |
| 机制硬度 | | | |
| 可实现性 | | | |
| 可证伪性 | | | |
| 论文贡献性 | | | |

## 最强反对意见

| 攻击 | 影响的主张 | 当前回答 | 状态 |
|---|---|---|---|
| | | | 待验证 |

## 判定

- 结论: 通过 / 需要修改 / 暂缓 / 不建议继续
- 进入定稿的主张:
- 删除或收缩的主张:
- 用户确认:
```

## 规则

- 审查对象必须是候选主张, 不是方法名或模块图
- “需要修改”必须说明改哪项主张或机制
- 不通过的主张不得进入`叙事定稿.md`
