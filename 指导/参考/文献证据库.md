# 文献证据库

## 核心作用

`文献证据库.md` 是全周期唯一的文献, 证据, 引用和相关工作比较入口。任何阶段只要发现可复用文献, 领域共识, 原文位置, 反证或相近工作比较, 都应写入这里。

证据库不是论文摘要库, 也不是 BibTeX 清单。每条记录只服务于一个可判断真假的论断或一个相关工作比较判断, 并说明原文究竟能支持到什么强度。

## 何时更新

- R1 检索相近工作和撞车风险时创建或更新
- R2 构造 Story, 审查硬度和定稿核心主张时补充证据
- R3 设计实验时补充 Benchmark, Baseline 和评价协议来源
- R6 解释结果时补充反证, 复现或对比来源
- R7 写作时为实际使用的外部事实补充精确位置
- 审稿和引用核验能力检查引用是否被过度解释
- 新版本论文, 反证或勘误出现时保留变更记录并更新状态

## 核验状态

- **已核验**: 已查看指定版本正文, 位置可复现
- **部分核验**: 已确认相关段落, 但论断强度仍需收缩或补证据
- **待核验**: 目前仅有摘要, 二手来源或记忆线索
- **已冲突**: 存在同等级反证, 不能单独作为确定事实
- **已失效**: 版本变化, 勘误或新证据使原记录不再适用

## 规则

- 没有精确页码, 章节或可定位位置的内容不能标为已核验
- 原文只摘录核验所需的短片段, 其余使用中文释义
- 区分论文明确陈述, 实验结果直接支持和 Agent 推断
- 同一论断存在相反证据时同时记录, 不选择性隐藏
- 同一来源支持多个论断时拆成多条记录, 避免一条证据承担过宽结论
- 作者未报告某项实验只能写“未报告”, 不能直接推断为“无法做到”
- Story, 摘要, 引言和 Related Work 使用的外部事实优先从本文件取用
- 领域概览只能作为发现和索引入口, 不能代替项目证据库中的原始来源
- 删除记录时不复用旧编号; 新证据推翻旧记录时保留修改说明

## 可支持强度

每条证据明确允许的最大表述。例如:

- 允许: “论文 A 在数据集 D 上报告了较低的任务恢复率”
- 不允许: “现有方法普遍缺乏恢复能力”

写作阶段不得超出该范围; 若需要更强表述, 应补充独立来源或弱化 Claim。

## 完成检查

- 进入`叙事定稿.md`的关键事实是否都有编号
- 最相近工作的机制, 证明范围和局限是否有精确位置
- 是否区分作者声明和 Agent 推断
- 是否记录相反证据和允许表述上限
- 写作实际引用的事实是否仍对应当前论文版本
