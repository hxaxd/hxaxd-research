# 研究想法

`研究想法.md`是 R1 的唯一正式产物, 用来保存用户 Idea 的正式化描述和四段式核心主张。

## 固定节

产出文件必须严格遵循以下骨架, Agent 绝对不能自行增减一级节。

```markdown
# 研究想法

## idea

用正式、准确、自然的中文描述用户的 Idea。

写法要求:
- 保留用户真正想研究的问题和方法
- 去掉随口表达中的歧义、重复和宣传性包装
- 不补写用户没有确认的贡献、结果或适用范围
- 必要时注明当前明确不处理的边界

## 核心主张

### 共识

当前可以作为出发点的背景判断是什么。

### 问题

在什么场景或条件下出现什么问题, 导致什么效果不好。

### 方法

用户主张做什么改进、机制或系统变化。

### 原因

为什么该方法应当改善上述问题, 预期改变什么中间行为或可观察结果。
```
