# 领域全景地图.md 格式

定义 `领域全景地图.md` 的固定结构, 该文件由 L2 领域知识地图生成

## 固定节

1. **这个领域在研究什么** — 一句话定义 + 研究对象, 输入输出, 不包含的相邻问题
2. **为什么这个领域重要** — 它回答了哪些其他领域回答不了的问题
3. **主要子方向及边界** — 每个子方向的定义, 代表工作和边界
4. **技术路线演化** — 按"什么旧限制促成了什么新路线"组织, 时间线辅助
5. **核心论文关系图谱** — 文本关系表 + 可选图形, 使用明确关系词, 奠定/延续/反驳/扩展/提供评测/暴露边界
6. **每篇论文为什么必须读** — 逐篇说明入选必要性, 不重复摘要
7. **当前发展状态与已解决问题** — "已解决"须说明在哪些设定和证据范围内成立
8. **典型 Benchmark 与评价指标** — 每项说明覆盖什么, 不覆盖什么
9. **关键未解问题** — 区分作者局限, 多篇共同暴露的空白, Agent 推断和猜测
10. **可能的论文选题入口及风险** — 每个入口说明撞车风险, 验证难度和所需能力
11. **推荐阅读顺序** — 分阶段, 每阶段有阅读目标和应回答的问题
12. **K 条目** — 按主题组织

## K 条目字段表

| 字段 | 说明 |
|---|---|
| K 编号 | 稳定且不复用 |
| 知识问题或结论 | 当前可复用判断 |
| 适用范围 | 条件, 任务和不能外推的范围 |
| 证据位置 | 来源版本, 章节/页码, 链接或本地材料 |
| 核验状态 | 待核验 / 部分核验 / 已核验 / 冲突 / 失效 |
| 冲突与未知 | 反证, 争议和待回答问题 |
| 更新时间 | 最近核验日期 |
| 影响位置 | 地图章节, Story 或研究项目 |

## 维护规则

- 初始 L2 只形成领域结构和首批关键 K 条目
- 之后在回答问题, 构造 Story, 比较论文或核验事实时发现证据不足, 才定向获取证据并更新条目
- 研究项目采用某个 K 条目时, 必须把原始来源和精确位置同步到该项目的 `文献证据库.md`, 并记录来源 K 编号
