# PDF 翻译

## 1. 做什么

对规范论文目录中已有的`原文.pdf`运行外部 PDF2zh, 生成保留原版面的中文 PDF 和双语对照 PDF。本能力不获取、复制或规范落位原文。

## 2. 输入

- 一个或多个已经存在的`workspace/<中文领域名>/<论文中文短标题>/原文.pdf`
- 每篇原文身份和版本已经核验

用户只提供论文链接、本地 PDF 或尚未规范落位的文件时, 应调用论文资源获取, 不能直接进入翻译。

## 3. 执行

### 工具自检

```powershell
.\scripts\verify-pdf2zh.ps1
```

成功退出则工具就绪, 若提示 NOT FOUND 或 WRONG VERSION, 告知用户并指向 `指导/学习/PDF 翻译脚本配置.md`

### 翻译 (单篇)

```powershell
.\scripts\translate-pdf.ps1 -InputPdf "workspace\<中文领域名>\<论文中文短标题>\原文.pdf" -OutputDir "workspace\<中文领域名>\<论文中文短标题>"
```

单篇可选参数:

- `-Pages "1-5,8"`: 只翻译指定页
- `-Glossary "术语表.csv"`: 传入 PDF2zh 术语表
- `-Qps 4`: 请求速率
- `-Workers 4`: 翻译工作线程数

### 翻译 (批量)

```powershell
.\scripts\translate-pdf.ps1 -InputPdfList "workspace\<中文领域名>\<论文1>\原文.pdf","workspace\<中文领域名>\<论文2>\原文.pdf",...
```

显式指定每篇论文的`原文.pdf`路径, 输出目录自动取各 PDF 的父目录。可选参数:

- `-Parallel 4`: 并行篇数上限 (1–30, 默认 4)
- `-Qps 4`: 每篇请求速率
- `-Workers 4`: 每篇翻译工作线程数

### 执行方式

翻译是长任务, 由现有脚本以前台阻塞方式执行:

- **单篇**: 直接在前台阻塞调用脚本, 等脚本退出后检查 §6 完成条件
- **批量**: 默认按小批次阻塞执行, 每批显式传入若干个尚未完成的 `原文.pdf` 路径, 直接等待该批脚本退出, 然后检查本批与整体 §6 完成条件, 再进入下一批
- **批次大小**: 默认每批 3-4 篇; 可根据 PDF 大小、页数和机器负载调小或调大
- **增量执行**: 构造每批 `InputPdfList` 前, 跳过已经同时存在 `中文译文.pdf` 和 `双语对照.pdf` 且满足 §6 的论文目录

工具版本、翻译服务和环境变量以`指导/学习/PDF 翻译脚本配置.md`为唯一权威。脚本固定执行英文到简体中文翻译, 无水印, 单语与双语同时输出, 只返回输出路径和文件大小——Agent 不打开, 截图, 全文搜索或手工修正生成的 PDF。

脚本先在临时目录运行, 确认两个输出以 `%PDF` 开头, 末尾包含 `%%EOF`, 大小非异常小后才替换`中文译文.pdf`和`双语对照.pdf`。失败时保留已有`原文.pdf`, 不得提交任何单边或临时译文, 也不得转为 Agent 手工翻译。

脚本在临时目录处理运行数据并在结束时清理。论文目录最终符合 §4 的固定文件集合。

## 4. 产出

```text
workspace/<中文领域名>/<论文中文短标题>/
├─ 原文.pdf
├─ 中文译文.pdf
├─ 双语对照.pdf
```

## 5. 确认点

- 无

## 6. 完成条件

- 每个论文目录中已有的`原文.pdf`保持不变
- 固定版本 PDF2zh 成功退出
- 中文与双语 PDF 同时存在, 以 `%PDF` 开头, 末尾包含 `%%EOF`, 大小非异常小
- 论文目录内容与 §4 的固定文件集合一致

## 7. 不应做

- 不获取、复制、移动或规范命名原始 PDF; 原文缺失时返回论文资源获取能力
- 不复制, 修改或分发 PDF2zh 源码
- Agent 不打开, 截图, 全文搜索或手工修正生成的 PDF
- 脚本失败时不得提交临时结果, 也不得转为 Agent 手工翻译
- 不能在本仓库维护 PDF2zh 分支

## 8. 常见失败

- 脚本失败后遗留临时输出, 被误认为正式结果
- 密钥缺失或版本不符导致翻译中断
