# 候选文献检索

## 1. 做什么

为已经明确的学习方向或研究问题建立一个可核验, 可去重, 覆盖正反证据的候选池
本任务只负责发现和初步判断相关性, 不替用户决定最终阅读列表, 也不能用搜索摘要代替后续精读

## 2. 输入

- 明确的学习方向或待研究问题
- 时间范围, 目标 venue, 任务类型和允许的相邻领域
- 候选池的用途: 学习列表, 撞车分析, 相关工作写作, 或验证单一事实
- 用户要求的特殊侧重: “最新”, “高引用”, “奠基”, “系统/Benchmark” 等

## 3. 执行

### 开始前先明确

- 检索对象是宽领域, 具体子方向, 某项机制, 还是某个待验证假设
- 时间范围, 目标 venue, 任务类型和允许的相邻领域
- 候选池用于学习列表, 撞车分析, 相关工作写作, 还是验证单一事实
- 用户是否要求”最新””高引用””奠基””系统/Benchmark”等特殊侧重

这些侧重会改变排序, 但不会降低书目信息和来源核验标准

### 构造检索问题

不要只把用户原句放进搜索框, 先建立四组查询:

1. **问题组**: 目标问题, 失败现象, 常见同义表达和上位概念
2. **机制组**: 可能的方法名, 机制作用, 替代实现和历史术语
3. **任务组**: 数据集, Benchmark, 评价指标, 应用场景和系统名称
4. **反证组**: `limitation`, `failure`, `negative result`, `benchmark leakage`,
   `revisit`, `does not improve` 等可能揭示反例的表达

对于撞车检索, 还应把”本质假设”和”实现外观”分别查询, 防止只找到同名方法

### 来源与核验顺序

1. 会议, 期刊, 出版社, arXiv, OpenReview, PubMed 等稳定学术页面
2. Crossref, OpenAlex, Semantic Scholar 等聚合服务, 用于补全和交叉核验
3. 作者主页, 项目页和官方代码仓库, 用于确认版本关系和补充材料
4. 博客, 媒体和非官方列表只能提供线索, 不能作为最终书目信息来源

每个候选至少核验:

- 英文标题, 作者, 年份或首次公开日期
- 正式发表状态, venue, track 或预印本状态
- DOI, arXiv, OpenReview 等稳定标识符
- 官方或稳定链接
- 版本关系, 例如预印本是否对应正式会议版本
- 与当前问题相关的依据来自标题, 摘要还是已读正文

### 候选筛选

为每篇候选记录以下状态:

| 字段 | 说明 |
|---|---|
| 相关性 | 高 / 中 / 低, 并说明针对哪个问题 |
| 角色 | 综述 / 奠基 / 方法 / 系统 / Benchmark / 反例 / 相邻工作 |
| 核验层级 | 书目已核验 / 摘要已读 / 正文关键段已核验 |
| 纳入理由 | 它补足了哪段知识或哪项比较 |
| 排除理由 | 重复, 范围不符, 证据弱或版本过旧 |

先按 DOI, OpenReview forum ID 或 arXiv ID 去重, 再按规范化标题去重, 预印本和正式版本通常视为同一工作, 但应保留版本差异

### 停止规则

- 已覆盖核心综述, 奠基工作, 代表路线, 关键 Benchmark, 主要前沿和明显反例
- 新增结果连续若干轮只重复已有路线, 不再带来新的分类或证据
- 已达到当前学习指导的默认规模或用户明确规定的候选规模
- 某篇资源难以定位时记录缺口, 不为补齐列表无限递归搜索

停止检索不代表领域已经穷尽, 输出中应写明时间, 数据库, 查询式和覆盖边界

### 简短示例

若问题是”长程 Agent 是否因上下文失配而失败”, 仅搜索
`long-horizon agent context mismatch` 不够, 还应查询上下文遗忘, 状态漂移,
任务恢复, 轨迹压缩, 长期记忆, 失败分析及相关 Benchmark, 候选表中需区分
“作者明确归因于上下文失配”和”Agent 根据现象推断可能相关”

## 4. 产出

学习阶段写入 `论文列表候选.md`, Idea 阶段把关键工作及比较依据同步写入
`文献证据库.md` 和 `相关工作矩阵.md`, 输出必须附检索边界, 核验日期和未覆盖项

## 5. 确认点

- 每篇候选至少核验英文标题, 作者, 年份, 发表状态, 稳定标识符和版本关系
- 记录相关性, 角色, 核验层级, 纳入理由和排除理由
- 按标识符和规范化标题进行去重, 保留预印本与正式版本的差异
- 检索边界, 核验日期和未覆盖项必须在输出中写明

## 6. 完成条件

- 已覆盖核心综述, 奠基工作, 代表路线, 关键 Benchmark, 主要前沿和明显反例
- 新增结果连续若干轮只重复已有路线, 不再带来新的分类或证据
- 已达到指定或默认的候选规模
- 难以定位的资源已记录缺口, 未无限递归搜索

## 7. 不应做

- 不按引用量直接等同于质量或相关性
- 不把”最新上传”自动等同于前沿代表作
- 不因论文支持当前 Idea 就提高可信度, 也不隐藏否定性结果
- 不仅凭标题判断机制相同, 标题相似可能本质不同, 标题不同也可能高度撞车
- 不把未核验的作者单位, 发表状态或投稿传闻写成事实

## 8. 常见失败

- 搜索范围过于狭窄, 只使用用户原句而忽略同义表达, 上位概念和反证查询
- 未区分”本质假设”与”实现外观”, 导致撞车检索遗漏表面不同但本质相同的工作
- 将博客, 媒体和非官方列表作为最终书目信息来源
- 未核验版本关系, 预印本与正式版混用页码
- 引用量或上传时间被误作质量或相关性的直接指标
- 候选去重不充分, 同一工作因不同标识符被重复收录
