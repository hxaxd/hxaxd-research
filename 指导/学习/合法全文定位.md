# 合法全文定位

## 任务目标

在不绕过访问控制的前提下, 为已确认论文找到身份正确, 内容完整, 来源可解释的PDF, HTML 或 Tex, 该任务只负责定位和验证来源, 未经用户明确要求不阅读全文,
也不自动进入翻译或处理

## 来源优先级

1. 出版方或会议官方开放页面, arXiv, OpenReview
2. 机构仓储中的作者接受稿, 作者主页公开稿, 官方项目页
3. 用户明确提供的本地文件
4. 用户明确要求使用的 Zotero 附件, 严格只读

非官方镜像, 搜索缓存和内容聚合站只能作为线索, 若最终只能取得这类材料, 必须说明来源风险, 不能把它冒充正式版本

## 身份与完整性核验

下载或采用文件前至少检查:

- 标题, 作者和版本是否对应目标论文
- 页数是否合理, 是否只有摘要, 海报或补充材料
- 是否包含正文, 参考文献和必要附录
- 正式版, 作者稿和预印本之间是否存在标题或内容变化
- Tex/源码包是否确实来自作者或官方平台, 而不是第三方复刻
- 文件能否正常打开, 是否存在明显损坏或页面缺失

如果存在多个合法版本, 记录版本差异, 并让后续任务知道引用页码对应哪个版本

## 有限检索流程

1. 从已核验的论文标识符进入官方页面
2. 检查页面公开的 PDF, HTML, supplementary, code 和 source 链接
3. 再检查作者主页, 机构仓储和项目页
4. 到达登录, 验证码, 付费或地区限制时停止
5. 记录已尝试来源, 失败原因和用户需要自行提供的文件

不要通过更换镜像, 模拟登录, 抓取受限接口或持续递归搜索来绕过限制

## 状态定义

- **完整正文已取得**: 已验证身份和完整性, 可供用户批准后处理
- **公开版本已取得**: 版本合法, 但可能与正式版存在差异
- **仅摘要/介绍页**: 不能用于全文处理或精确页码引用
- **来源待核验**: 文件存在, 但身份, 完整性或许可不明确
- **未取得**: 记录检索边界和失败原因

## 常见陷阱

- 文件名相同不代表论文相同, 尤其是作者主页中的旧草稿
- supplement 不能代替主论文, 主论文也不能代替数据或代码许可
- 搜索结果中显示“PDF”不代表链接公开合法
- 找不到 Tex 时不得从 PDF 手工伪造“作者源码”

## 输出

记录选定 URL 或本地路径, 来源类型, 版本, 核验日期,
完整性, 访问状态和备注, 用户确认具体全文, 领域归属及处理范围后, 如需归档或机械处理, 再调用论文处理能力
