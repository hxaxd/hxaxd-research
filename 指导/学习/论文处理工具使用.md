# 论文归档与 PDF 翻译

## 1. 做什么

把用户批准的论文归档, 并使用外部 PDF2zh 生成保留原版面的中文 PDF 和双语对照 PDF

## 2. 输入

- 已选定合法 PDF 全文
- 已确认 `workspace/<中文领域名>/<论文中文短标题>/`

用户只提供孤立论文时, 先检查现有领域目录并询问领域归属, Agent 可以提出候选领域, 但不能自行创建或选择领域路径

## 3. 执行

### 工具自检

```powershell
.\scripts\verify-pdf2zh.ps1
```

成功退出则工具就绪, 若提示 NOT FOUND 或 WRONG VERSION, 告知用户并指向 `指导/学习/论文处理工具配置.md`

### 归档

1. 把用户指定的原始 PDF 复制到论文目录, 不移动, 删除或覆盖用户提供的源文件
2. 保存为 `原文.pdf`

论文目录不再生成 PDF 解析 Markdown, 独立图片目录或仓库自有机器状态文件

### 翻译

```powershell
.\scripts\translate-pdf.ps1 -InputPdf "workspace\<中文领域名>\<论文中文短标题>\原文.pdf" -OutputDir "workspace\<中文领域名>\<论文中文短标题>"
```

可选参数:

- `-Pages "1-5,8"`: 只翻译指定页
- `-Glossary "术语表.csv"`: 传入 PDF2zh 术语表
- `-Qps 4`: 请求速率
- `-Workers 4`: 翻译工作线程数

脚本固定行为: DeepSeek V4 Flash, 思考模式关闭, 英文到简体中文, 无水印, 单语与双语同时输出, 脚本只返回输出路径和文件大小——Agent 不打开, 截图, 全文搜索或手工修正生成的 PDF

脚本先在临时目录运行, 确认两个输出均非空且具有 PDF 文件头后才替换 `中文译文.pdf` 和 `双语对照.pdf`, 失败时不得提交临时结果, 也不得转为 Agent 手工翻译

### Agent 上下文边界

- 工具向 DeepSeek 发送正文不代表正文进入 Codex 模型上下文
- PDF2zh 的用户级缓存不进入论文目录, 也不由 Agent 逐项检查
- 只有用户另行请求摘要, 问答, 审稿, 修订或翻译质量评价时, 才读取完成该任务所需的最小页, 节或样本

## 4. 产出

```text
workspace/<中文领域名>/<论文中文短标题>/
├─ 原文.pdf
├─ 中文译文.pdf
├─ 双语对照.pdf
```

## 5. 确认点

- 确认领域归属, 论文目录和正文传输许可已获用户批准——未批准前不开始归档和翻译
- 翻译完成后告知用户输出路径, 由用户决定是否需要语义质量审校

## 6. 完成条件

- 原始 PDF 已复制到论文目录, 未改动用户源文件
- 固定版本 PDF2zh 成功退出
- 中文与双语 PDF 同时存在, 非空, 文件头有效
- 没有遗留临时输出

## 7. 不应做

- 不移动, 删除或覆盖用户提供的源文件
- 不复制, 修改或分发 PDF2zh 源码
- Agent 不打开, 截图, 全文搜索或手工修正生成的 PDF
- 脚本失败时不得提交临时结果, 也不得转为 Agent 手工翻译
- 不能在本仓库维护 PDF2zh 分支
- 不承诺 PDF→Markdown, 也不提供通用 Markdown, HTML 或 TeX 翻译

## 8. 常见失败

- 首次安装或运行下载资源较大, 用户可能误以为卡死
- 脚本失败后遗留临时输出, 被误认为正式结果
- 密钥缺失或版本不符导致翻译中断
- 用户未确认领域归属或正文传输许可, Agent 自行决定导致路径错误
