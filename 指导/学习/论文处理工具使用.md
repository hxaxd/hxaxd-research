# 论文归档与 PDF 翻译

## 能力目标

把用户批准的论文归档, 并使用外部 PDF2zh 生成保留原版面的中文 PDF 和双语对照 PDF
该能力不评价论文可信度, 研究价值或翻译语义质量

## 前置

- 已选定合法 PDF 全文
- 用户明确要求处理全文, 并允许正文发送到 DeepSeek API
- 用户已确认 `workspace/<中文领域名>/<论文中文短标题>/`
- PDF2zh 已按 `指导/学习/论文处理工具配置.md` 安装并验证

用户只提供孤立论文时, 先检查现有领域目录并询问领域归属, Agent 可以提出候选领域,
但不能自行创建或选择领域路径, 不得增加 `论文资料/` 中间层

## 归档定义

1. 把用户指定的原始 PDF 复制到论文目录, 不移动, 删除或覆盖用户提供的源文件
2. 当前处理版本保存为 `原文.pdf`, 其他版本使用可辨识名称并在 `论文信息.md` 标记
3. 记录来源, 版本, 稳定标识符, 归档日期和 SHA-256
4. 在 `论文处理记录.md` 记录工具版本, 命令, 服务, 输出, 哈希和失败

论文目录不再生成 PDF 解析 Markdown, 独立图片目录或仓库自有机器状态文件

## 工具边界

- 使用官方
  [`PDFMathTranslate-next`](https://github.com/PDFMathTranslate/PDFMathTranslate-next),
  固定 `pdf2zh-next==2.9.0`
- 该项目采用 AGPL-3.0, 本仓库只调用已安装工具, 不复制, 修改或分发其源码
- PDF2zh 只接受 PDF 输入, 标准输出为单语 PDF 与双语 PDF
- TeX 用于阅读翻译时, 先由论文项目自己的工具链编译成 PDF
- 可编辑 TeX 的中英翻译继续使用写作指导, 不属于本机械能力
- 不承诺 PDF→Markdown, 也不提供通用 Markdown, HTML 或 TeX 翻译

## Agent 上下文边界

- Agent 不打开, 截图, 全文搜索或手工修正原文, 中文 PDF 和双语 PDF
- `scripts/translate-pdf.ps1` 只返回版本, 路径, 大小, 哈希和错误
- PDF2zh 的用户级缓存不进入论文目录, 也不由 Agent 逐项检查
- 只有用户另行请求摘要, 问答, 审稿, 修订或翻译质量评价时, 才读取完成该任务所需的
  最小页, 节或样本

工具向 DeepSeek 发送正文不代表正文进入 Codex 模型上下文

## 命令路线

安装或校验固定版本:

```powershell
.\scripts\install-pdf2zh.ps1
```

翻译:

```powershell
.\scripts\translate-pdf.ps1 `
  -InputPdf "workspace\<中文领域名>\<论文中文短标题>\原文.pdf" `
  -OutputDir "workspace\<中文领域名>\<论文中文短标题>"
```

可选参数:

- `-Pages "1-5,8"`: 只翻译指定页
- `-Glossary "术语表.csv"`: 传入 PDF2zh 术语表
- `-Qps 4`: 请求速率
- `-Workers 4`: 翻译工作线程数

## 执行流程

1. 确认领域归属, 论文目录, 处理版本和正文传输许可
2. 告知用户首次安装和首次运行可能下载较大的 Python, 模型, 字体或布局资源
3. 运行安装脚本, 不通过实际付费翻译测试密钥
4. 运行翻译脚本, 脚本固定使用 DeepSeek V4 Flash, 关闭思考模式, 英文到简体中文,
   无水印, 单语与双语同时输出
5. 脚本先在论文目录内的临时目录运行, 确认两个输出均非空且具有 PDF 文件头后, 再直接
   替换 `中文译文.pdf` 和 `双语对照.pdf`
6. Agent 只记录脚本返回的机器结果, 不做视觉或语义抽查

脚本失败时不得提交临时结果, 也不得转为 Agent 手工翻译, 可以报告错误, 调整官方公开参数或重试, 不能在本仓库维护 PDF2zh 分支

## 输出

```text
workspace/<中文领域名>/<论文中文短标题>/
├─ 论文信息.md
├─ 原文.pdf
├─ 中文译文.pdf
├─ 双语对照.pdf
├─  (论文处理记录, 阅读笔记, 评议等按需创建) 
```

## 完成条件

- 原始 PDF, 版本, 来源和哈希可追溯
- 固定版本 PDF2zh 成功退出
- 中文与双语 PDF 同时存在, 非空, 文件头有效且记录 SHA-256
- 没有遗留临时输出
- 处理记录明确机器验收不等于语义质量审校
- 能力结束后返回原调用位置, 不改变生命周期阶段
