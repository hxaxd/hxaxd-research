# PDF2zh 工具配置

## 1. 做什么

安装并验证论文 PDF 翻译所需的最小外部环境, 不在仓库中维护 Python 项目, 虚拟环境或依赖锁

## 2. 输入

- 依赖管理工具: `uv`
- 运行环境: Python 3.12
- 工具版本: `pdf2zh-next==2.9.0`
- DeepSeek API Key: 通过 `PDF2ZH_DEEPSEEK_API_KEY` 环境变量提供

## 3. 执行

### 固定环境

- `uv`: 负责安装隔离的命令行工具
- Python 3.12: 由 `uv tool` 使用, 不创建仓库 `.venv/`
- `pdf2zh-next==2.9.0`: 固定版本, 不自动跟随最新版本
- DeepSeek V4 Flash: 默认翻译服务, 思考模式关闭

安装命令:

```powershell
.\scripts\install-pdf2zh.ps1
```

安装脚本只安装和核验工具, 不调用付费 API, 也不主动执行 PDF 翻译, 首次安装可能下载Python 和依赖, 首次实际翻译还可能下载 PDF2zh/BabelDOC 的模型, 字体与布局资源, 执行前应向用户说明

### API Key

运行脚本按以下顺序解析密钥:

1. 当前进程的 `PDF2ZH_DEEPSEEK_API_KEY`
2. Windows 用户级同名变量
3. Windows 机器级同名变量

密钥只在运行脚本进程内映射为 PDF2zh 所需变量, 不得显示, 记录或写入任何密钥值

### 行为边界

- 不用仓库 `pyproject.toml`, `config.toml`, `uv.lock` 或 `.venv/` 管理 PDF2zh
- 不复制, 修改或 vendoring PDF2zh/BabelDOC 源码
- 不静默升级固定版本, 升级必须先核对官方变更, CLI 参数和输出文件名, 再修改脚本与
  文档
- 工具未安装, 版本不符, 输入不是 PDF, 密钥缺失或上游失败时明确停止
- PDF2zh 的缓存位于用户环境, 不复制到论文目录, 不作为研究证据
- 不为安装验收发起付费请求

运行安装脚本:

```powershell
.\scripts\install-pdf2zh.ps1
```

待脚本成功退出后定位 `pdf2zh_next` 可执行文件, 确认版本严格为 2.9.0, 仍需首次资源下载或 API Key 时明确说明, 但不把它们伪装为已经验证

## 4. 产出

- 已安装并核验的 `pdf2zh_next` 可执行文件, 版本 2.9.0
- 安装脚本成功退出记录

## 5. 确认点

- 安装脚本成功退出
- 定位到 `pdf2zh_next` 可执行文件
- 确认版本严格为 2.9.0
- 仍需首次资源下载或 API Key 时明确说明, 不伪装为已验证

## 6. 完成条件

`scripts/install-pdf2zh.ps1` 成功退出, 定位到 `pdf2zh_next` 可执行文件, 并确认版本严格为 2.9.0, 仍需首次资源下载或 API Key 时明确说明, 但不把它们伪装为已经验证

## 7. 不应做

- 不使用仓库 `pyproject.toml`, `config.toml`, `uv.lock` 或 `.venv/` 管理 PDF2zh
- 不复制, 修改或 vendoring PDF2zh/BabelDOC 源码
- 不静默升级固定版本
- 不为安装验收发起付费请求
- 不得显示, 记录或写入 API Key 值

## 8. 常见失败

- 首次安装未下载 Python 或依赖, 用户误以为安装过程卡死
- API Key 未设置或设置错误, 密钥解析顺序被忽略
- Python 或 pip 版本不兼容导致依赖安装失败
- 工具未安装, 版本不符, 输入不是 PDF 或上游失败时未明确停止
- 误以为安装验收已完成就可以翻译, 忽略首次资源下载需求
