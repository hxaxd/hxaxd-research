# 论文资源获取

## 1. 做什么

为论文列表中的指定论文寻找合法, 完整, 身份明确的 PDF, HTML, Tex, 官方页面, 项目页和代码仓库, 验证身份和版本, 记录可复查的来源, 目标不是"把能下载的都下载下来", 而是为后续处理任务取得身份明确的材料

## 2. 输入

- 用户确认的 `论文列表.md`, 或独立任务中明确指定的论文清单
- 需要获取全文的具体范围
- 每篇论文已核验的稳定标识符和版本信息

允许的来源: 出版方或会议官方开放页面, arXiv, OpenReview, 机构仓储中的作者接受稿, 作者主页公开稿, 官方项目页, 用户明确提供的本地文件, 非官方镜像, 搜索缓存和内容聚合站只能作为线索, 若最终只能取得这类材料, 必须说明来源风险

## 3. 执行

### 获取与验证流程

1. 从 `论文列表.md` 或用户明确指定的论文建立待获取清单
2. 按以下顺序寻找: 官方出版页 → arXiv / OpenReview → 机构仓储 → 作者主页 → 官方项目页 / GitHub
3. 下载或采用文件前核验身份:
   - 标题, 作者和版本是否对应目标论文
   - 页数是否合理, 是否只有摘要, 海报或补充材料
   - 是否包含正文, 参考文献和必要附录
   - 正式版, 作者稿和预印本之间是否存在标题或内容变化
   - Tex/源码包是否确实来自作者或官方平台
   - 文件能否正常打开, 是否存在明显损坏或页面缺失
4. 对 PDF, HTML, Tex, supplement, 项目页和代码分别记录, 不用一个链接代表全部
5. 到达登录, 验证码, 付费或地区限制时停止, 记录原因
6. 记录已尝试来源, 失败原因和用户需要自行提供的文件
7. 可以采用用户明确提供的本地文件, 记录来源为"用户提供", 不猜测其公开许可
8. 找不到全文时记录已尝试路径和失败原因
9. 只有摘要或介绍页时标为"未获得完整正文"

### 版本选择

- 阅读优先选择内容完整, 页码稳定的版本
- 引用时明确使用的版本, 避免正式版与预印本页码混用
- Tex 源码适合结构化翻译, 但不得假设其一定与最终发表版一致
- 代码仓库和项目页是补充材料, 不能代替论文中的方法与实验陈述
- 多个公开版本均可用时, 说明差异并给出推荐, 不替用户静默选择

## 4. 产出

每篇论文记录以下字段, 格式详见 `指导/参考/论文信息格式.md`:

- PDF, Tex, HTML, 官方页面, 标识符, 项目页, 代码仓库
- 获取状态: 可处理 / 仅可阅读 / 仅元数据 / 待核验 / 未取得
- 版本, 完整性, 来源可信度, 核验日期和备注

## 5. 确认点

- 多版本可用时向用户说明差异并给出推荐, 由用户选择使用哪个版本
- 非官方来源或来源存疑时向用户说明风险, 由用户决定是否采用
- 需要用户自行提供的文件, 如付费墙后的论文, 明确告知用户需要提供什么

## 6. 完成条件

- 每个已请求论文都有成功来源或明确失败记录
- 每篇论文的获取状态, 版本, 完整性和来源可信度已记录
- 每个可处理文件已核对标题, 作者, 版本, 页数和完整性
- 非官方来源已标注风险, 摘要/介绍页已标为"未获得完整正文"
- 未绕过登录, 验证码, 付费墙, 许可证或访问控制
- 用户选择具体全文和领域归属后, 需要归档或生成中文与双语 PDF 时, 再调用论文归档与 PDF 翻译能力

## 7. 不应做

- 不批量抓取用户未选择的全部论文
- 不绕过登录, 验证码, 付费墙, 许可证或访问控制
- 不因某个镜像下载方便就省略官方来源核验
- 不把非官方镜像或内容聚合站的材料冒充正式版本
- 不把 supplement 代替主论文, 也不把主论文代替数据或代码许可
- 不把代码 README, 媒体介绍或幻灯片当成完整论文
- 不在尚未确认版本时开始全文处理
- 找不到 Tex 时不得从 PDF 手工伪造"作者源码"

## 8. 常见失败

- 仅凭镜像下载方便就省略官方来源核验, 使用非官方版本
- 文件名相同不代表论文相同, 尤其是作者主页中的旧草稿
- 版本尚未确认就开始全文处理, 导致引用页码错乱
- 把代码 README, 媒体介绍或幻灯片当成完整论文
- 搜索结果中显示"PDF"不代表链接公开合法
- 找不到全文时无限递归搜索, 未及时记录失败路径
- 找不到 Tex 时自行手写或逆向还原, 冒充作者源码
