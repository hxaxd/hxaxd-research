# 论文资源获取

## 能力目标

为论文列表寻找合法, 可信且尽量完整的 PDF, Tex, HTML, 官方页面, 项目页和代码仓库, 并留下可复查的来源记录, 目标不是“把能下载的都下载下来”, 而是为用户确认的论文和后续处理任务取得身份明确的材料

## 输入与前置

- 用户确认的 `论文列表.md`, 或独立任务中明确指定的论文清单
- 需要获取全文的具体范围
- 每篇论文已核验的稳定标识符和版本信息

## 执行流程

1. 从 `论文列表.md` 或用户明确指定的论文建立待获取清单, 确认处理范围
2. 按官方出版页, arXiv, OpenReview, 机构仓储, 作者主页, 官方项目页和官方
   GitHub 的顺序寻找
3. 对 PDF, HTML, Tex, supplement, 项目页和代码分别记录, 不用一个链接代表全部
4. 调用 `指导/学习/合法全文定位.md` 验证身份, 版本, 完整性和访问边界
5. 可以采用用户明确提供的本地文件, 记录来源为“用户提供”, 不猜测其公开许可
6. 只有用户明确要求时才读取 Zotero 附件路径, 并保持严格只读
7. 找不到全文时记录已尝试路径和失败原因, 不无限递归搜索
8. 只有摘要或介绍页时标为“未获得完整正文”
9. 找不到 Tex 时不得自行手写, 逆向还原或把 OCR 文本冒充作者源码

## 资源选择原则

- 阅读优先选择内容完整, 页码稳定的版本
- 引用时明确使用的版本, 避免正式版与预印本页码混用
- Tex 源码适合结构化翻译, 但不得假设其一定与最终发表版一致
- 代码仓库和项目页是补充材料, 不能代替论文中的方法与实验陈述
- 多个公开版本均可用时, 说明差异并给出推荐, 不替用户静默选择

## 输出

每篇论文记录 PDF, Tex, HTML, 官方页面, 标识符,
项目页, 代码仓库, 获取状态, 版本, 完整性, 来源可信度, 核验日期和备注

建议使用以下状态:

| 状态 | 含义 |
|---|---|
| 可处理 | 完整正文已验证, 等待用户确认处理 |
| 仅可阅读 | 有合法正文, 但格式不适合自动处理 |
| 仅元数据 | 只有摘要, 介绍页或书目信息 |
| 待核验 | 文件存在但身份, 完整性或许可不清 |
| 未取得 | 在限定检索范围内未找到 |

## 不应做什么

- 不批量抓取用户未选择的全部论文
- 不绕过登录, 验证码, 付费墙, 许可证或访问控制
- 不因某个镜像下载方便就省略官方来源核验
- 不把代码 README, 媒体介绍或幻灯片当成完整论文
- 不在尚未确认版本时开始全文处理

## 完成条件

- 每个已请求论文都有成功来源或明确失败记录
- 未绕过登录, 验证码, 付费墙, 许可证或访问控制
- 每个可处理文件已验证论文身份和完整性
- 用户选择具体全文和领域归属后, , 需要归档或生成中文与双语 PDF 时,
  再调用论文归档与 PDF 翻译能力
