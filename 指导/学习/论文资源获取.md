# 论文资源获取

## 1. 做什么

为论文列表中的指定论文获取全文——只获取 PDF, Tex 源码或 HTML 三种格式 (arXiv 恰好提供这三种) 优先获取 PDF; 无 PDF 时告知用户当前可用的格式, 由用户决定是否采用 Tex 或 HTML, 不静默替换官方页面, 项目页和代码仓库已在归档阶段记录于论文列表, 本能力不再重复获取

## 2. 输入

- 用户确认的 `论文列表.md`, 或独立任务中明确指定的论文清单
- 需要获取全文的具体范围
- 每篇论文已核验的稳定标识符和版本信息

## 3. 执行

### 获取顺序

按以下顺序寻找 PDF (优先), Tex 源码或 HTML:

1. arXiv — 同时提供 PDF, Tex 源码和 HTML
2. 出版方或会议官方开放页面
3. 作者主页公开稿

非官方镜像, 搜索缓存和内容聚合站只能作为线索, 若最终只能取得这类材料, 必须说明来源风险

### PDF 优先规则

- 优先获取 PDF——完整, 页码稳定, 适合翻译
- 不因 Tex 或 HTML 获取更方便就跳过 PDF

### 核验下载

下载后做轻量核验, 不依赖外部工具:

- 文件以 `%PDF` 开头
- 文件末尾包含 `%%EOF`
- 文件大小合理, 非异常小

### 失败处理

- 只有摘要或介绍页时告知用户"未获得完整正文"
- 可以采用用户明确提供的本地文件

## 4. 产出

论文目录下的 PDF (优先), Tex 源码或 HTML 文件

## 5. 确认点

- 无 PDF 时告知用户当前可用的格式, 由用户决定是否采用 Tex 或 HTML
- 多版本可用时向用户说明差异并给出推荐, 由用户选择使用哪个版本
- 非官方来源或来源存疑时向用户说明风险, 由用户决定是否采用
- 需要用户自行提供的文件, 明确告知用户需要提供什么

## 6. 完成条件

- 每篇请求论文的 PDF 已获取并放入论文目录, 或已告知用户无法获取及原因
- 全部下载完成后, 询问用户是否调用翻译能力对所有论文进行翻译

## 7. 不应做

- 不批量抓取用户未选择的全部论文
- 不把非官方来源的材料当作正式版本——即使下载方便
- 不把 supplement 代替主论文
- 不在尚未确认版本时开始全文处理
- 找不到 Tex 时不得从 PDF 手工伪造"作者源码"
- 不落盘任何 PDF 之外的文件

## 8. 常见失败

- 文件名相同不代表论文相同, 尤其是作者主页中的旧草稿
- 搜索结果中显示"PDF"不代表链接公开合法
- 找不到全文时无限递归搜索, 未及时停止并告知用户
