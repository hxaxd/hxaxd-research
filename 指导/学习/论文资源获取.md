# 论文资源获取

## 1. 做什么

为`论文列表.md`中已收录、但用户尚未持有全文的指定论文寻找合法完整 PDF。核验论文身份和版本后, 将文件规范保存为论文目录中的`原文.pdf`。用户已经提供本地 PDF 时说明路由错误, 改由文献检索与收录处理, 本能力不接收或复制该文件。目标论文尚未收录时也先返回文献检索与收录。没有可用 PDF 时报告失败, 不用 Tex、HTML、摘要或介绍页代替全文。项目页和代码仓库已在收录阶段记录于论文列表, 本能力不再重复获取。

## 2. 输入

- 用户确认并定稿的`论文列表.md`, 且其中已有目标论文的完整记录
- 需要获取全文的具体范围
- 每篇论文已核验的稳定标识符和版本信息
- 用户确认的领域归属和论文中文短标题

## 3. 执行

### 获取顺序

开始前先确认目标论文已经收录, 且用户尚未持有 PDF。`论文列表.md`中没有对应完整记录, 或用户已经提供本地 PDF 时停止, 改由文献检索与收录处理。

按以下顺序寻找 PDF:

1. arXiv 官方 PDF
2. 出版方或会议官方开放页面
3. 作者主页公开稿

非官方镜像, 搜索缓存和内容聚合站只能作为线索, 若最终只能取得这类材料, 必须说明来源风险

### PDF 限定

- 本能力只接受 PDF 作为最终全文资源
- 只能取得 Tex、HTML、摘要或介绍页时, 明确告知用户未获得可规范保存的完整 PDF 并停止
- 不把其他格式转换或伪装成作者提供的 PDF

### 核验下载

下载后做轻量核验, 不依赖外部工具:

- 文件以 `%PDF` 开头
- 文件末尾包含 `%%EOF`
- 文件大小合理, 非异常小

### 规范落位

核验完成后才创建`workspace/<中文领域名>/<论文中文短标题>/`, 将取得的 PDF 保存为`原文.pdf`。

目标位置已有`原文.pdf`时先核验是否为同一论文和版本; 内容一致则报告已经存在, 内容或身份冲突时停止并由用户决定, 不静默覆盖。

### 失败处理

- 只有摘要、介绍页、Tex 或 HTML 时告知用户"未获得合法完整 PDF"
- 网络来源均不可用时, 可以请用户改为向文献检索与收录提供本地 PDF
- 失败时不创建空论文目录, 不留下临时下载文件

## 4. 产出

`workspace/<中文领域名>/<论文中文短标题>/原文.pdf`

## 5. 确认点

- 多版本可用时向用户说明差异并给出推荐, 由用户选择使用哪个版本
- 非官方来源或来源存疑时向用户说明风险, 由用户决定是否采用
- 需要用户自行提供 PDF 时, 明确告知用户应改为调用文献检索与收录
- 目标位置已有不同内容或版本的`原文.pdf`时, 说明冲突并由用户决定如何处理

## 6. 完成条件

- 每篇请求论文的合法完整 PDF 已核验并保存为对应论文目录中的`原文.pdf`, 或已告知用户无法获取及原因
- 新建论文目录中至少存在`原文.pdf`, 没有空目录、临时下载或其他格式正文
- 全部下载完成后, 询问用户是否调用翻译能力对所有论文进行翻译

## 7. 不应做

- 不批量抓取用户未选择的全部论文
- 不把非官方来源的材料当作正式版本——即使下载方便
- 不把 supplement 代替主论文
- 不在尚未确认版本时开始全文处理
- 不把 Tex、HTML、摘要或介绍页作为本能力产物落盘
- 不接收、复制、移动或规范保存用户已经提供的本地 PDF
- 不静默覆盖论文目录中已有的`原文.pdf`

## 8. 常见失败

- 文件名相同不代表论文相同, 尤其是作者主页中的旧草稿
- 搜索结果中显示"PDF"不代表链接公开合法
- 找不到全文时无限递归搜索, 未及时停止并告知用户
