# L2 领域知识地图

## 阶段宗旨

解释领域如何组织, 为什么选择这些论文, 论文之间是什么关系, 以及当前真正值得关注的研究入口, 地图是持续增长的知识索引, 不是逐篇精读记录, 也不把人的阅读过程纳入生命周期状态

## 输入

- 可用的 `学习目标.md` 和 `最终论文列表.md`
- 已核验的摘要, 综述结构, 论文证据和处理材料
- 用户确认的学习重心

直接进入 L2 时, 用户也可以提供等价的领域范围, 知识问题和已知来源, 不要求为了进入本阶段补建 L1 文档

## 构建流程

### 1. 定义边界

给出领域一句话定义, 研究对象, 输入输出和不包含的相邻问题, 若术语存在多个含义,
说明本地图采用哪一种

### 2. 建立分类轴

归纳主要子方向及边界, 分类轴应来自问题, 假设, 机制, 系统形态或评测方式, 而不是仅按论文作者自行命名的方法分组, 同一论文可以出现在多个关系中, 但要说明原因

### 3. 梳理演化

按“什么旧限制促成了什么新路线”解释技术演化, 时间线只是辅助, 重点是机制和研究问题如何变化, 区分真正的路线转折与仅在规模, 数据或工程上改进的工作

### 4. 建立论文关系

对核心论文使用明确关系词:

- 奠定问题或评价方式
- 延续同一假设并改变实现
- 反驳或修正早期结论
- 将方法扩展到新场景
- 提供系统化 Benchmark
- 暴露失败边界或开放问题

逐篇说明入选必要性, 而不是重复摘要

### 5. 判断领域现状

总结当前前沿, 已解决问题, 仍有争议的问题, 典型 Benchmark, 评价指标和已知缺陷
“已解决”必须说明在哪些设定和证据范围内成立, 不能泛化到整个领域

### 6. 提炼研究入口

区分:

- 作者明确提出的局限或未来工作
- 多篇论文共同暴露的空白
- Agent 基于证据推断的研究机会
- 仍缺证据, 暂不应作为 Idea 的猜测

若用户以写论文为目标, 指出每个入口的撞车风险, 验证难度和所需能力, 不只列
“未来可研究方向”

### 7. 设计阅读路径

给出分阶段阅读顺序, 并为每阶段写明阅读目标, 应回答的问题和可以略读的部分

### 8. 维护 K 条目

在相关主题下维护稳定且不复用的 `Kxx` 条目:

| 字段 | 内容 |
|---|---|
| K 编号 | 稳定且不复用 |
| 知识问题或结论 | 当前可复用判断 |
| 适用范围 | 条件, 任务和不能外推的范围 |
| 证据位置 | 来源版本, 章节/页码, 链接或本地材料 |
| 核验状态 | 待核验 / 部分核验 / 已核验 / 冲突 / 失效 |
| 冲突与未知 | 反证, 争议和待回答问题 |
| 更新时间 | 最近核验日期 |
| 影响位置 | 地图章节, Story 或研究项目 |

初始 L2 只需形成领域结构和首批关键 K 条目, 之后在回答问题, 构造 Story, 比较论文或核验事实时发现证据不足, 才调用本指导定向获取证据并更新条目, 不要为了“保持地图活跃”而进行无目标扩充

研究项目采用某个 K 条目时, 必须把原始来源和精确位置同步到该项目的
`文献证据库.md`, 并记录来源 K 编号, 不能只引用地图中的二手总结

## 输出

生成 `领域全景地图.md`, 至少包含:

1. 这个领域在研究什么
2. 为什么这个领域重要
3. 主要子方向及边界
4. 技术路线演化
5. 核心论文关系图谱
6. 每篇论文为什么必须读
7. 当前发展状态与已解决问题
8. 典型 Benchmark 与评价指标
9. 关键未解问题
10. 可能的论文选题入口及风险
11. 推荐阅读顺序
12. 按主题组织的 K 条目及其冲突, 未知和影响位置

建议在“核心论文关系图谱”中同时提供文本关系表, 避免图形本身成为唯一信息载体

## 不应做什么

- 不把“近期论文更多”写成某方向更重要的证据
- 不把综述的分类原样复制而不检查其时效与偏见
- 不把作者自称 SOTA 当作领域共识
- 不把一个 Benchmark 上的领先结果推广为问题已经解决
- 不为生成选题而夸大开放问题

## 常见陷阱

- 每个子方向只有论文清单, 没有问题和机制边界
- 技术演化只按年份排列, 看不出为什么发生变化
- “研究空白”只是把作者局限换一种说法, 没有撞车与可验证性判断
- 推荐顺序和最终论文列表顺序完全相同, 却没有学习目标

## 完成条件

地图形成可解释的知识结构, 不是逐篇论文摘要拼接, 外部事实可追溯, 研究空白明确标注证据强度, 用户能据此说明主要路线, 代表论文, 评价方式和至少一个尚未解决的真实问题, 初始结构和首批 K 条目得到用户确认后, L2 可以完成, 后续更新作为按需能力执行, 不自动改变生命周期阶段
