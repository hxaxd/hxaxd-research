# R4 实验实施计划

## 1. 做什么

把已经确认的证明设计转化为可直接实施, 可复现和可恢复的工程计划

计划应让另一名工程人员只阅读研究文档和代码仓库就能复现实验, 不依赖聊天中的隐含决定, 此阶段不重新发明实验目标, 发现设计不可实现时应回写 R3

## 2. 输入

- `研究想法.md`
- `叙事定稿.md`
- `核心主张.md`
- `实验证明设计.md`
- 用户手动拉取到根目录 `labs/<仓库目录>/` 的目标代码仓库

## 3. 执行

### 1. 先理解现有工程

先确认目标仓库真实存在, 若不存在, 将 R4 标为阻塞并请用户手动创建 `labs/`, 准备目标仓库, Agent 不执行 clone, git init, 也不替用户决定 checkout 目录名（确认点）

进入仓库后先读取它自己的 `AGENTS.md` 和工程规范, 再检查仓库结构, 已有入口, 配置方式, 测试, 数据加载, 日志和结果格式, 优先复用现有模式, 不为了整洁重写无关模块
在计划中记录:

- 相对根研究仓库的路径 `labs/<实际目录>`
- 上游仓库 URL
- 当前分支和开始实验前的基线 commit
- 该代码仓库与中文研究任务的关系

将实验仓库视为独立 Git 工作区, 本研究任务不规定其内部代码, 日志, 结果和产物目录结构, 只要求后续记录能精确定位

### 2. 数据与环境

记录数据获取方式, 版本, 许可, 校验和目录结构, 规定环境锁定方式, 硬件要求, 外部服务和密钥环境变量, 敏感凭据不得写入仓库或运行记录

### 3. 模块与接口

把方法, Baseline, 评测, 配置和数据流程拆成可测试模块, 对每个模块说明输入, 输出,
依赖, 对应 Claim 和最低验证, 避免把整套实验写成一个无法单测的脚本

### 4. 实验入口与配置

为每类实验分配稳定编号和入口命令, 配置应显式记录模型, 数据, 预算, 随机种子,
超参数和输出路径, 相同实验只通过配置变化, 不复制多份难以同步的脚本

### 5. 日志与产物

代码, 配置, 日志, 原始结果和工程产物保存在独立实验仓库内, 具体布局沿用该仓库约定每次运行仍应保存:

- 代码版本或 commit
- 环境和依赖版本
- 完整配置与随机种子
- 实际执行命令
- 标准输出, 错误日志和运行状态
- 原始预测, 指标和中间产物
- 资源消耗和运行时间
- 可生成表格和图表的结构化数据

明确哪些文件是原始结果, 哪些是聚合结果, 防止手工表格脱离原始数据, 中文研究任务中的 `实验记录/` 不复制整套工程产物, 而是记录 commit, 命令, 配置和实验仓库内部实际路径

### 6. 最小可运行版本

先规划一条最短闭环: 少量数据, 一个 Baseline, 一个关键指标和一个目标 Claim它的目标是验证数据流, 指标, 日志和机制接口, 不用于正式论文结论

### 7. 完整实验与依赖顺序

按“基础设施验证 → 小规模机制实验 → 主实验 → 消融 → 鲁棒性/效率 → 图表导出”安排顺序, 后续实验依赖前序通过时写清门槛, 避免昂贵运行建立在错误指标上

### 8. 恢复与备选方案

设计断点续跑, 缓存校验, 失败重试, 部分结果隔离和资源不足方案, 备选方案不能悄悄改变 Claim, 需要改变数据, 模型或预算时返回用户确认（确认点）

## 4. 产出

生成 `实验编程计划.md`, 至少包含:

1. 代码仓库规划
2. 数据与 Benchmark
3. 工程模块
4. 实验脚本和入口
5. 配置系统
6. Baseline 接入
7. 指标计算
8. 日志与结果保存
9. 可复现设置
10. 最小可运行版本
11. 完整实验版本
12. 运行命令
13. 独立实验仓库映射, 内部预期输出位置与 Claim 映射
14. 风险, 失败恢复与备选方案

文档同时附实验清单, 依赖顺序, 资源估算和恢复策略, 开始创建或修改实验代码前必须取得用户确认

## 5. 确认点

- 确认目标仓库真实存在, 若不存在则请用户手动创建 `labs/` 并准备目标仓库
- 开始创建或修改实验代码前必须取得用户确认
- 需要改变数据, 模型或预算时返回用户确认

## 6. 完成条件

无

## 7. 不应做

- 不替用户创建, 初始化或拉取实验仓库
- 不在计划中使用无法直接执行的伪命令
- 不把“参考官方实现”作为 Baseline 接入的全部说明
- 不让日志和结果文件使用会被下一次运行覆盖的固定路径
- 不只保存最终平均分而丢失种子级结果
- 不把最小可运行版本的结果写进论文主结论
- 不在 R4 擅自扩大 R3 已确认的实验范围

## 8. 常见失败

- 计划很详细, 但没有 Claim 到实验编号的映射
- 代码模块按软件架构拆分, 却没有实验依赖顺序
- 数据版本, 模型快照和外部服务版本未记录
- 只考虑成功路径, 没有中断, 超时和部分完成恢复
