# R3 实验证明

## 1. 做什么

R3 把`叙事定稿.md`中的完整方法、贡献性设计关系和需要经验支持的分点核心主张, 组织成一套有主次、有先后、能够共同闭合论文结论的实验论证。

实验设计包含三个不同层次:

- **分点主张**是证明责任, 说明论文必须让读者相信什么。
- **实验家族**是论证与呈现单元, 例如主实验、分点深化、消融、迁移、效率或误差分析。
- **共享运行**是 R4 的实际执行单元, 一组运行可以服务多个实验家族和多条主张。

R3 以实验家族组织文档节奏, 再通过分点主张的证据映射检查证明闭环。整体顺序通常是: 先定义完整方法和公平比较边界, 再用主实验确认完整方法的总体竞争力, 随后用针对性实验解释独特收益、组件归因、适用范围和成本, 最后回到每条分点主张作综合判定。每个实验同时预定下游需要的结果事实和图表材料, 使 R4 的真实交付能够直接被 R5 使用。

R3 固定会影响研究结论的方法身份、实验变量、比较条件和判定规则。R4 在这些边界内自由决定代码结构、依赖、脚本、目录、调度和并行实现。

## 2. 输入

- 用户已经确认的`叙事定稿.md`
- 其中的论文事实、贡献性设计关系、需要经验支持的分点核心主张及其削弱或推翻条件
- 完整方法的当前定义、主要模块、更新阶段和运行阶段
- 已知的最相近工作、可用 Benchmark、Baseline 候选、指标和资源边界
- 会影响研究结论的模型、执行框架、方法变体、演化或训练预算、冻结策略、项目记忆条件等候选变量

若任务、技术原因、核心认识、贡献性设计关系或分点主张仍不清楚, 先回到 R2 修正。若方法还不能被明确描述为一个固定的受测对象, 先与用户确认完整方法由什么组成、哪些部分会学习、哪些部分会冻结、执行时允许改变什么。

R3 只处理需要经验支持的主张。文献边界、新颖性判断和形式推导由其真实证据支持; 机制、因果、有效性、效率和泛化主张则分别需要与其强度相称的实验。

## 3. 执行

### 1. 定义方法与比较边界

先把论文实际评测的对象固定下来。

从`叙事定稿.md`中的完整方法和贡献性设计关系开始, 检查每个贡献性设计在正式方法中对应什么机制、状态变化或约束。R3 可以补充实现身份和实验所需参数, 但若实际受测方法改变了 R2 的核心认识或设计关系, 先返回 R2 由用户确认。

#### 完整方法

说明完整方法包含哪些必要部分, 它们如何组合, 训练或进化阶段产生什么, 运行阶段读取什么, 哪些状态会更新, 哪些制品在迁移时保持冻结。完整方法在正式实验中使用稳定名称和配置身份。

#### 方法变体

列出后续实验真正需要的变体, 并说明每个变体相对完整方法只改变什么。方法变体用于公平对照、消融、替换或迁移, 其身份在看到正式结果前固定。

#### 比较基线

基线根据研究问题选择, 通常从以下角色中取必要集合:

- 无方法或简单方法, 用来确定基本增益
- 与本研究解决同一问题的强公开方法, 用来确定总体竞争力
- 资源匹配对照, 用来排除更多模型调用、更多上下文、更多数据或更高预算带来的优势
- 理想信息或理想决策条件, 用作解释差距的上界

每个基线都要写清纳入理由、来源、复现方式、可访问信息、模型能力和预算。消融变体属于完整方法的内部归因, 不能替代最相近工作的外部比较。

主实验负责证明完整方法在公平协议下的总体竞争力。若论文准备声称 SOTA, 必须在相同或经过严格对齐的任务、划分、评价、资源和成功定义下直接比较当前最强方法; 不可比的公开数字只能提供背景。若研究价值体现为效果、成本与迁移能力的联合优势, 应预先定义对应的多目标比较, 而不是只追求单一排行榜数字。

### 2. 固定共同实验协议

共同实验协议覆盖所有正式实验中一旦改变就可能影响研究结论的条件。至少根据项目实际情况固定:

- 数据集或任务集合、训练/验证/测试划分及版本
- 数据泄漏、任务污染和跨任务迁移边界
- 评测单位、主要指标、次要指标、机制指标和资源指标的优先级
- 方法变体、模型角色、准确模型标识和可变参数范围
- 执行框架及其版本, 包括跨框架实验中允许变化与保持不变的部分
- 演化、训练、搜索、推理和人工介入预算的统一口径
- 流程记忆是否冻结、何时更新、跨目标时迁移哪一份制品
- 冒烟测试、预跑和正式运行的区分, 以及预跑结果进入正式统计的条件
- 重试、超时、失败、排除和协议无效的预定规则
- 随机性控制、统计单位、重复次数、聚合方法和不确定性表达
- 令牌、调用次数、运行时间、货币成本和摊销成本中适用指标的计算口径

主要指标直接决定主实验结论; 次要指标补充能力侧面; 机制指标解释过程; 资源指标约束公平性。指标层级在正式运行前固定, 避免结果出现后从大量指标中挑选最有利者。

共同实验协议可以使用实验矩阵表达模型 × 执行框架 × 方法变体 × 数据条件。矩阵用于固定变量和复用共享运行, 实际工程布局由 R4 决定。

### 3. 设计实验组合

实验组合先回答整体问题, 再逐步解释收益来源和成立边界。除主实验外, 其余实验家族只在分点主张需要时加入。

#### 主实验

主实验比较完整方法与强基线, 回答“整套方法在主要任务上是否值得成立”。它至少指定:

- 主要 Benchmark、任务范围和评价单位
- 完整方法、强基线和必要资源匹配对照
- 决定总体结论的主要指标
- 方法新增调用、上下文或计算时必须同时观察的成本
- 需要在主表或主图中直接显现的任务子集、稳定性或失败情况

主实验承担总体效果和竞争位置, 不承担所有组件的因果归因。一个总分更高只能证明完整系统更有效, 不能自动证明每个模块必要、机制解释正确或迁移主张成立。

#### 分点深化实验

分点深化实验回答总分无法直接揭示的核心研究判断。它通常针对论文最独特的主张设计受控条件、干预和中间过程测量, 例如:

- 方法是否减少了某类特定失败, 而不只是提高平均成功率
- 记忆是否被更准确地选择、求证和使用
- 进化过程是否减少局部过拟合、回归或错误写回
- 某个机制是否在预期条件出现, 并在条件撤销时消失

分点深化实验优先观察与主张直接对应的现象。它可以复用主实验运行, 也可以增加专门干预; 关键是产生比总体结果更有辨识力的证据。

#### 消融实验

消融实验回答“完整方法的收益由哪些组件和交互产生”。根据方法结构选择必要形式:

- 删除组件, 检验该组件是否必要
- 用较简单或现有设计替换组件, 检验具体设计选择是否必要
- 同时改变两个关键因素, 检验组件之间是否存在互补或依赖
- 逐步加入组件, 展示能力形成过程

消融条件尽量保持信息访问、模型、数据和预算一致。若删除组件天然减少调用或成本, 同时报告效果变化和资源变化。消融用于组件归因; 核心机制主张还需要与机制直接对应的中间证据。

#### 泛化、迁移与稳健性实验

当主张涉及跨项目、跨任务分布、跨模型、跨执行框架、不同规模或噪声条件时, 需要单独验证其成立范围。

迁移实验明确源条件、冻结制品、目标条件和目标侧允许的适配。跨模型与跨执行框架可以共享实验矩阵, 但应区分单轴变化与多轴同时变化, 使失败能够被定位。稳健性实验只覆盖会真实威胁结论的随机性、输入质量、记忆时效、超参数或任务难度变化。

#### 效率、收敛与规模实验

当方法增加进化轮次、强模型调用、上下文编译、检索或运行时步骤时, 成本既是公平条件, 也是研究结果。根据主张选择:

- 效果—令牌、效果—调用或效果—时间曲线
- 达到同一效果所需的样本、轨迹或进化轮次
- 一次性进化成本、单任务运行成本和跨任务摊销成本
- 随任务数、项目规模、记忆规模或模型规模变化的趋势

主实验先报告关键成本, 独立效率实验再解释收敛速度、成本结构或规模规律。

#### 诊断、案例与误差分析

当关键机制难以被单一指标完整描述时, 使用可追溯的中间产物、失败分类和代表性案例解释结果。案例选择规则在查看结果前确定, 并与总体分布或数量统计结合。诊断证据用于解释方法何时、为何成功或失败, 不能单独替代总体效果证据。

若分点主张需要其他实验家族, 例如人工评价、用户研究或特定安全测试, 可以按同一原则加入: 每个实验必须回答清楚的研究问题, 并能改变某条主张的判定。

### 4. 为每个实验定义执行与交付

每个实验使用稳定编号`E1`、`E2`等, 并写清:

- **证据等级**: 核心、强化或探索
- **服务主张**: 对应哪些分点核心主张
- **研究问题**: 该实验单独回答什么
- **设计与对比**: 哪些方法、条件和共享运行参与, 只改变什么
- **重点观察**: 主要指标、必要成本、机制现象和子集结果的优先顺序
- **预定交付**: R4 必须提供哪些结构化结果、运行引用、图表所需材料和分析产物, 以及结果需要回答到什么精度
- **实际交付**: R3 写“待执行”, R4 回填结果摘要和真实路径

证据等级含义如下:

- **核心**: 缺少该实验时, 中心主张或总体方法有效性无法成立
- **强化**: 扩展可信范围、稳健性或解释深度, 但不承担中心主张的唯一证据
- **探索**: 用于发现新现象和后续假设, 其结果与预定核心结论分开解释

实验编号对应论证单元, 共享运行可以被多个实验编号引用。相同运行和原始结果只保存一次。

### 5. 建立分点主张的证据闭环

实验组合完成后, 回到每条需要经验支持的分点主张, 写明:

- **证据组合**: 哪些实验共同支持该主张, 其中哪个是直接证据, 哪些是补充证据
- **关键替代解释与控制**: 还有什么现实原因可能产生同样结果, 已由哪个对照、预算匹配、消融、干预或不变量控制区分
- **支持、削弱和推翻判据**: 不同结果分别允许得出多强的判断
- **当前判定**: R3 写“待执行”, R4 根据完整证据组合回填

证据强度必须与主张类型匹配:

- 完整方法有效性由主实验直接支持
- 组件必要性由消融或替换支持
- 机制解释由干预和中间现象支持
- 跨目标复用由冻结制品的迁移实验支持
- 成本或收敛优势由统一口径的曲线和资源结果支持

同一实验可以服务多条主张, 同一主张也可以依赖多个实验。主张—证据映射用于检查缺口, 不要求把实验重复改写成逐条运行计划。

### 6. 固定写作所需的结果责任

R3 不预写结果, 但要确保 R4 完成后能够形成论文需要的结果事实。每个实验的`预定交付`根据实际问题覆盖:

- 直接观察到的主要数值、趋势、分布或失败类型
- 观察成立的任务、模型、执行框架、预算和比较条件
- 重复、方差、置信区间或其他适用的不确定性
- 与主要效果同时解释的调用、令牌、时间或其他关键成本
- 由当前对照允许的归因, 以及仍未排除的替代解释
- 支持、削弱或推翻哪条主张, 成立范围在哪里
- 适合进入主表、曲线、消融表、案例图或正文的真实结构化材料

图表形式服务研究问题, 不在 R3 为版面美观提前固定。能够由同一份结构化结果支持多种展示时, R4 保存真实数据, R5 再选择最终版式。

实验组合完成后应能提供四类论文内容中实际适用的部分: 完整方法的总体结果、独特主张的直接证据、关键设计的归因证据, 以及成本、失败和成立边界。若其中某类是论文中心结论所必需的, 却没有对应预定交付, 实验证明尚未完成。

### 7. 与用户确认

重点确认:

- 完整方法、方法变体和比较基线的身份是否准确
- 主实验是否真正回答总体竞争力, 主要指标和成本口径是否合理
- 分点深化实验是否抓住了论文最独特、最值得证明的现象
- 消融是否覆盖关键组件与必要交互
- 跨模型、跨执行框架和其他范围实验是否与主张强度一致
- 核心、强化和探索实验的划分是否符合预算
- 每条分点主张是否已经形成充分且最小的证据组合
- 支持、削弱和推翻判据是否符合用户愿意承担的论证强度

## 4. 产出

生成或更新`实验证明.md`, 严格参照`指导/参考/实验证明.md`, 包含:

- 方法与比较边界
- 共同实验协议
- 实验组合
- 分点主张判定

R3 写入方法身份、正式协议、实验问题、证据等级、重点观察、预定交付和预设判据。R4 在同一文件中回填每个实验的实际交付, 再综合更新每条分点主张的当前判定。

## 5. 确认点

- 用户确认完整方法、方法变体和比较基线
- 用户确认共同实验协议、主要指标、实验矩阵和资源边界
- 用户确认主实验与核心实验组合
- 用户确认分点深化、消融、迁移、效率和诊断实验中实际需要的部分
- 用户确认主张—证据映射及支持、削弱和推翻判据
- 用户确认这套核心证据可以交给 R4 执行

## 6. 完成条件

- 已生成`实验证明.md`
- 文档严格遵守`指导/参考/实验证明.md`的固定一级节
- 完整方法、方法变体与比较基线具有稳定身份
- 主实验明确完整方法的总体比较对象、主要指标和重点成本
- 每个实验都有研究问题、证据等级、服务主张、设计、重点观察和预定交付
- 每个核心实验的预定交付能够形成带条件、统计口径、解释边界和真实来源的结果事实
- 多组件方法具有足以完成组件归因的消融设计
- 泛化、迁移、效率或机制主张均有类型匹配的直接证据
- 每条经验性分点主张都能映射到充分且最小的实验组合
- 每项贡献性设计已经映射到直接证据, 或被明确识别为不承担独立经验主张的实现部分
- 所有影响结论的研究变量已经固定或明确列为待用户决定
- 文档没有越界规定实验仓库的工程实现
- 用户确认可以进入 R4

## 7. 边界

- 主实验确定完整方法的总体竞争力, 消融负责内部归因, 两者职责分开
- 总体提升不能单独证明机制、组件必要性或迁移能力
- SOTA 判断只使用可比协议下的结果
- 额外上下文、额外调用、额外模型能力和额外预算进入公平控制
- 正式结果出现后保持主要指标、划分、重试规则和成功定义稳定
- 实验家族由主张需要决定, 版面惯例本身不构成新增实验的理由
- R3 固定研究含义, R4 决定工程实现

## 8. 常见失败

- **主次倒置**: 一开始逐条消融, 却没有完整方法与强基线的主实验
- **总分包办**: 用一个总体成功率解释所有机制和泛化主张
- **消融冒充外部比较**: 只与自己的删减版本比较, 没有最相近方法
- **组件清单式消融**: 每个模块机械删一次, 却没有回答关键设计选择和交互
- **分点实验不够直接**: 想证明记忆利用或局部过拟合, 只观察最终任务分数
- **迁移名不副实**: 目标条件重新优化了制品, 却声称冻结迁移
- **成本观察滞后**: 方法增加大量调用, 主实验只报告效果
- **实验矩阵膨胀**: 为覆盖所有模型、框架和参数耗尽预算, 却没有优先完成核心证据
- **指标漂移**: 看到结果后才决定哪个指标最重要
- **证明映射缺口**: 实验很多, 仍有分点主张没有直接证据
