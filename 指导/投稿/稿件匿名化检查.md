# 稿件匿名化检查

## 能力目标

依据已核验的 venue 匿名规则, 审计 Tex, PDF, 文件名, 注释, 链接, 补充材料和仓库暴露, 先报告风险, 用户确认后才修复安全的本地内容

## 输入

- 投稿规则核验中的匿名要求
- 论文项目, 最终 PDF 和全部提交材料
- 相关代码, 数据, 演示或补充材料链接
- 用户允许检查和修改的范围

## 审计范围

- 作者, 机构, 致谢, 基金, 个人主页和自引措辞
- Tex 注释, 自定义命令, 文件路径, 文件名和压缩包目录
- PDF 属性, 附件, 书签和嵌入元数据
- 图像元数据, 截图账号, 终端路径和可识别水印
- GitHub, 数据下载, 演示站点, 模型卡, 数据卡和 URL 跳转
- 补充材料, 代码说明, 许可证和仓库 commit 中的身份暴露

## 执行流程

1. 只读审计并生成风险清单, 记录位置, 证据, 官方规则和严重性
2. 区分必须清除, 可能允许, 规则不明确和无法在本地修复的暴露
3. 向用户说明拟修改文件和对可复现性的影响
4. 用户确认后, 修复安全的本地内容, 如作者字段, PDF 元数据, 注释和文件名
5. 重新编译或重打包, 复查最终交付文件
6. 外部服务, 公开仓库, 历史 commit 或缓存暴露仅记录并交由用户处理

## 禁止事项

- 不在未确认前修改稿件或删除材料
- 不改写 Git 历史, 不删除远程仓库, 不操作外部服务
- 不为匿名删除影响论文理解或复现的必要信息, 按官方允许方式替换
- 不凭经验扩大匿名要求, 规则不明时标为待用户决定

## 输出

生成 `稿件匿名化检查.md`, 记录规则来源, 审计范围, 风险, 修复决定, 已修复项和剩余暴露, 必要匿名暴露未清除时是 R8 硬性阻断项, 能力完成后结束
