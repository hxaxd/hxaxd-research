# R2 论文叙事与分点核心主张

## 1. 做什么

R2 根据`研究想法.md`把 Idea 展开成可写论文的叙事, 并把核心主张拆成分点主张。正式产物只有`叙事定稿.md`。

叙事和分点核心主张都来自 R1 的核心主张, 只是展开角度不同: 叙事强调为什么当前问题真实且重要; 分点核心主张强调本文做了什么、为什么可能有效, 以及后续需要哪些证据。

## 2. 输入

- `研究想法.md`
- 领域事实、相近工作和文献来源
- 用户对 R1 对话审查结果的取舍

若缺少`研究想法.md`, 先说明缺口, 由用户决定补 R1 还是接受风险直接继续。

## 3. 执行

### 1. 读取核心主张

从`研究想法.md`读取四段式核心主张:

- 共识
- 问题
- 方法
- 原因

不要静默新增新的主张。若发现 R1 的主张不清、过强或与用户最新说法冲突, 先回到 R1 修正并让用户确认。

回写 R1 时必须严格参照`指导/参考/研究想法.md`的固定节格式, 不得自行增减一级节, 不得把修正理由、对话记录或临时判断写进`研究想法.md`。

### 2. 写叙事

叙事要把核心主张展开成论文故事, 重点回答:

- 当前领域普遍接受或至少可证的背景是什么
- 在什么场景下出现了什么具体问题
- 这个问题为什么重要, 会影响什么能力、性能、可靠性、效率或理解
- 现有工作为什么没有充分解决这个问题
- 本文切入点为什么自然、必要且不夸大

叙事可以引用旧论文说明共识、问题存在和重要性。引用文献时在句子后使用`[文献:E1]`, 并确保`文献证据库.md`中有对应条目。

### 3. 拆分点核心主张

把 R1 的核心主张拆成少量分点主张。每一点应能被后续文献、实验或推理单独支持或推翻。

每个分点主张至少说明:

- 主张内容
- 对应 R1 的哪一块: 共识、问题、方法或原因
- 需要什么证据支持
- 什么情况会削弱或推翻它

不同对应部分的证据责任不同:

- **共识**: 可以只用文献引用和论述证明, 不必进入实验。
- **问题**: 可以用文献引用和论述证明, 也可以设计实验或现象观察来证明。
- **方法**: 因为是本文的新方法, 通常需要实验结果证明它确实有效。
- **原因**: 需要说明为什么方法有效, 通常对应可解释性分析、机制观察、消融、案例或中间过程证据。

分点主张不是论文贡献列表, 也不是章节标题。它是 R3 实验证明和后续写作口径的接口。

### 4. 对齐用户

向用户展示叙事和分点核心主张, 重点确认:

- 叙事是否仍是用户原本想做的研究
- 是否把问题说得过强或过宽
- 分点主张是否覆盖用户认为重要的方法和原因
- 哪些主张暂时没有证据, 后续需要实验或文献补足

用户修改方向时, 先判断是否需要回写`研究想法.md`; 只要改变 R1 的共识、问题、方法或原因, 就先回写 R1。回写时仍必须严格使用`指导/参考/研究想法.md`的固定节格式。

## 4. 产出

正式 R2 产出只有:

- `叙事定稿.md`

`叙事定稿.md`参考`指导/参考/叙事定稿.md`, 只包含:

- 叙事
- 分点核心主张

`文献证据库.md`是全周期文献身份索引, R2 使用新的文献时可以更新, 但它不是 R2 专属产物。

## 5. 确认点

- 用户确认最终叙事和分点核心主张后, 才视为 R2 完成
- 新增、删除或改变核心主张时, 必须由用户确认
- 发现 R1 核心主张需要改变时, 先按参考格式回写`研究想法.md`

## 6. 完成条件

- 已生成或更新`叙事定稿.md`
- 文档只包含叙事和分点核心主张两部分
- 每个分点主张都有证据责任和削弱/推翻条件
- 每个分点主张的证据责任符合其对应部分: 共识可引用论述, 问题可引用论述或实验, 方法要实验, 原因要可解释性工作
- 使用文献的地方已按`[文献:E1]`格式引用, 且对应文献条目存在; 未核验内容已明确标为待核验
- 用户确认当前叙事和主张

## 7. 不应做

- 不在 R2 创建独立核心主张文件
- 不创建独立审查记录
- 不把叙事写成摘要或引言草稿
- 不为了写得顺而改变 R1 的研究方向
- 不让分点主张变成宣传性贡献文案

## 8. 常见失败

- **叙事越界**: 为了显得重要, 把问题扩展到证据无法支撑的范围
- **主张混杂**: 一个分点同时承担问题存在、方法有效和泛化结论
- **证据口径漂移**: 文献只支持局部现象, 叙事却写成普遍共识
- **写作提前**: 还没确认主张, 就开始写论文段落
