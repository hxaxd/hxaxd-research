# R2 论文叙事与分点核心主张

## 为什么 R2 是论文的内容准备阶段

一个 Idea 可以逻辑成立, 却仍然没有形成值得整篇论文围绕的 Story。R2 寻找这项研究真正改变了什么理解, 并把实验、图表和写作都会反复依赖的事实说准。

R2 在同一次协作中完成两个内部阶段:

1. 寻找并确认中心叙事
2. 围绕这条叙事澄清论文事实, 再抽取承重主张

两部分共同写入`叙事定稿.md`, 不增加中间状态文件。第一阶段决定论文为什么值得讲; 第二阶段让这个故事能够被真实方法和证据承载。

## 第一阶段: 找到中心叙事

### 从 Idea 本体出发

先读取`研究想法.md`, 抓住研究对象、目标场景、问题、方法本体、原因链条与明确边界。R1 的四类核心主张用于检查逻辑有没有缺口, 不决定 R2 要写几段、生成几条主张或采用什么顺序。

如果用户仍在改变场景、核心问题、主要方法或原因, 先回到 R1 对齐, 待 Idea 本体稳定后再构造叙事。

### 寻找真正的认识变化

好的中心叙事通常围绕一个精确矛盾展开: 既有理解在某个重要条件下不再充分, 一个被忽略的原因或关系解释了失败, 方法因此获得自然的设计方向。寻找时可以追问:

- 哪个可观察失败最值得解释, 它只在什么条件下出现
- 表面表现背后最可能的技术瓶颈是什么
- 最相近路线已经解决了什么, 又在哪个与本文有关的条件下停止
- 本文发现的关系怎样改变解决问题的方式
- 如果判断成立, 读者以后会怎样重新理解这个问题

从中选择与当前研究有关的问题寻找中心。某些论文的价值来自新机制, 某些来自新的失败解释、能力边界或统一视角; 叙事应服从研究本身。

### 用事实约束吸引力

叙事依赖领域共识、现有工作局限或机制事实时, 只检索当前判断需要的原始来源。重点核验问题是否存在、最相近工作是否已经采用同一切入点、所谓局限是否真由原文支持, 以及方法与问题之间是否有合理联系。

“最吸引人”的叙事在真实边界内抓住影响最大的矛盾。搜索摘要只能提供线索; 决定叙事的工作要读到足以比较方法、机制和实验的位置, 并在使用事实处直接引用。

### 候选只来自真实分歧

材料明显只支持一条叙事时, 直接把它推到最精确。只有中心矛盾、重要性来源或方法切入确实存在不同解释时, 才形成少量候选。候选应说明各自抓住什么矛盾、方法为何自然、哪项事实会击穿它; 标题和措辞变化不算不同候选。

由用户确认中心矛盾、认识变化和方法切入点。若选择改变了 R1 的 Idea 本体, 先同步`研究想法.md`。

## 第二阶段: 形成下游共享的论文模型

中心叙事确定后, 把下游会共同引用的事实和关系澄清到足以支持实验与写作。“论文事实”为 R3 提供受测方法, 为 R5 提供稳定的问题原因和贡献关系。

根据当前论文实际需要说明:

- 研究对象、输入输出、使用场景和评价目标
- 关键失败发生的对象与条件, 以及可观察后果
- 对失败的技术解释, 并区分外部事实、本文观察和待证明机制
- 改变问题理解的核心认识
- 完整方法及其关键设计怎样共同运行
- 最相近工作的真实重合、本文改变的部分和明确边界

只展开理解本文所需的信息; 尚不能确认的内容写明证据状态, 让缺口保持可见。

### 理解贡献性设计

对于真正承载研究增量的设计, 需要理解一条完整因果关系:

```text
它回应什么问题或原因
-> 为什么选择这种设计
-> 运行时实际改变什么
-> 预期出现什么可观察差异
-> 什么证据能够区分该解释
-> 结论停在什么边界
```

这条关系是一种推理视角。多个组件共同实现同一认识时可以合并解释; 常规工程步骤在复现需要时进入方法, 贡献身份由它承担的研究增量决定。

### 抽取承重主张

分点核心主张从已确认叙事中的关键转折、方法依据和能力边界抽取。删除测试可以帮助判断必要性: 删除后叙事仍然完整的内容通常无需成为承重主张。

每条主张表达一个能够被独立支持、收窄或推翻的研究判断, 并说明它在叙事中的作用、所需证据和失效条件。证据形式由主张决定: 外部事实依赖来源, 方法效果依赖实验, 机制解释往往需要干预或中间现象, 形式主张则依赖明确前提与推导。R3 处理其中需要经验支持的部分。

最后从整体检查两件事:

- **覆盖**: 中心叙事有没有未经任何主张承担的关键跳跃
- **最小**: 多条主张是否只是同一判断的改写, 或一条主张是否混入数个不同结论

## 写入与确认

按`指导/参考/叙事定稿.md`把中心叙事、论文事实和分点核心主张写入同一文件。叙事使用连续中文自然组织; 论文事实保存全文必须共享的稳定关系; 分点主张保存论文需要承担的证明责任。模板的一级节用于让下游可靠定位内容, 节内结构服从当前研究。

用户需要确认的是会改变论文含义的选择: 中心矛盾、技术原因、核心认识、完整方法、贡献性设计关系、最相近工作边界和承重主张。措辞精度、证据标记和不改变含义的整理可以直接迭代。

找不到真实中心矛盾、方法无法从认识中推出, 或故事依赖未核验事实时, 直接说明缺少的是问题理解、证据还是方法关系。停止包装不等于否定 Idea; 用户可以补材料、返回 R1 或暂时保留未决状态。

当`叙事定稿.md`能够直接向 R3 提供受测方法与证明责任、向 R5 提供论文事实, 且用户认可其中心判断时, R2 已完成。
