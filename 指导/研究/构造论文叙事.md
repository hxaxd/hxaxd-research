# R2 论文叙事与分点核心主张

## 1. 做什么

R2 根据`研究想法.md`寻找一条真实、精确、能够组织后续研究的中心叙事, 再围绕已确认叙事澄清后续实验和写作共同依赖的论文事实, 并抽取不可缺少的分点核心主张。三类内容最终写入同一份`叙事定稿.md`。

R2 内部包含两个阶段:

1. 寻找并确认中心叙事
2. 围绕已确认叙事固定论文事实并抽取承重主张

两个阶段在同一次协作中完成, 最终共同写入`叙事定稿.md`。R2 是论文内容的主要准备阶段: 它固定下游反复使用的任务、问题、原因、认识、设计关系和边界, 但不提前撰写论文段落。

## 2. 输入

- `研究想法.md`
- 用户对 R1 撞车分析和硬度审查结论的取舍
- 必要的领域事实、最相近工作和文献来源
- 用户已经明确的任务输入输出、主要评价目标、典型失败、方法组成、预期技术优势和适用边界

若缺少`研究想法.md`, 先说明缺口, 由用户决定补 R1 还是接受风险直接继续。

若用户仍在改变研究场景、核心问题、主要方法或原因链条, 说明当前仍属于 R1, 不用叙事包装尚未稳定的 Idea。

## 3. 执行

### 开始前: 读取 Idea 本体

从`研究想法.md`读取研究对象、适用场景、核心问题、主要方法、原因链条和明确不处理的范围。用户原始描述中已经存在的输入输出、评价目标、失败条件、设计关系和边界也要保留, 不因 R1 文档简洁而丢失。

R1 的共识、问题、方法、原因是审查 Idea 是否完整的坐标, 不是 R2 的生成模板。不得按四类分别扩写叙事或凑出对应数量的分点主张。

若 R1 内容不清、过强或与用户最新说法冲突, 先按`指导/参考/研究想法.md`修正并由用户确认。

### 第一阶段: 寻找并确认中心叙事

#### 抓住中心矛盾

叙事不是把背景、问题和方法依次写出来, 而是说明这项研究真正改变了什么认识。寻找时重点判断:

- 现有理解或方法在什么精确条件下失效
- 表面问题背后的真正瓶颈是什么
- 现有工作遗漏或误判了什么关键因素
- 方法为什么从这个认识中自然产生
- 若这件事成立, 会改变什么问题理解、解决方式或能力边界

这些是判断问题, 不是固定段落。若当前材料只能说明“增加一个模块”或“希望提高指标”, 却说不清中心矛盾, 不得用修辞伪造叙事。

#### 补必要事实

叙事依赖领域共识、现有工作局限或其他外部事实时, 进行必要的网络检索和来源核验。检索只服务当前叙事, 不扩张成完整文献综述。

重点核验问题是否真实存在、最相近工作是否已采用相同切入点、“现有工作没有解决”的边界是否准确, 以及方法与问题之间是否存在合理联系。

搜索摘要只能提供线索。涉及对方方法、机制、实验或局限的关键判断, 必须读取足以支持判断的原始来源。使用文献时在相关句子附近直接引用可靠网页; 暂无网页来源时写明可识别的论文标题并标记`链接待补`, 相关判断在核验前标为待核验。

#### 形成候选

若材料明显只支持一条叙事, 直接形成并检查这一条, 不强行制造多个候选。

若同一 Idea 确实存在不同的中心矛盾、重要性来源或切入角度, 可以在对话中形成少量候选。候选之间必须是研究判断不同, 不能只是标题、措辞或段落顺序不同。

每条候选用自然语言说明它抓住的中心矛盾、带来的认识变化、方法为什么自然, 以及什么事实会直接击穿它。不要固定候选数量。

#### 比较并确认

从真实性、精确性、集中程度、方法是否自然、认识变化是否重要、能否被证据击穿几个方面比较候选。

“最吸引人”不是修辞最强, 而是在不超出证据的前提下抓住影响最大的真实矛盾。不同候选存在真实性、重要性或可证明性的取舍时, 直接向用户说明。

向用户展示当前叙事或候选, 由用户确认中心矛盾、认识变化和方法切入点。中心叙事未经确认, 不进入第二阶段。

若用户的选择改变 R1 的研究场景、核心问题、主要方法或原因链条, 先回写`研究想法.md`。

### 第二阶段: 固定论文事实并抽取承重主张

#### 澄清论文事实

中心叙事确认后, 先把下游实验、图表和正文都必须共享的内容说清。根据研究实际情况澄清:

- **任务与目标**: 研究对象、输入、输出、真实使用场景和最重要的评价目标
- **关键失败与条件**: 哪类既有方法在什么条件下出现什么可观察问题, 带来什么后果
- **技术原因**: 该失败为什么发生; 区分已有来源支持的事实、本文观察形成的解释和后续需要证明的机制判断
- **核心认识**: 哪个关系或缺口改变了问题理解, 以及它怎样推出解决方向
- **完整方法**: 方法本体由哪些贡献性设计组成, 它们怎样形成一个完整运行关系
- **相近工作与边界**: 本文继承什么、改变什么, 当前明确主张和不主张什么

这些内容不是摘要草稿, 也不按固定句数填写。它们的责任是让 R3 能够定义受测方法与证据, 让 R5 不必在写作时临时发明任务、原因或贡献关系。

外部事实仍按第一阶段的证据规则核验。当前无法确认的内容直接写清缺口及其所需证据; 不把待实验的机制解释写成既成事实。

#### 建立贡献性设计关系

对每项真正承担研究增量的设计, 写清同一条关系:

```text
目标技术问题或原因
-> 设计选择
-> 实际运行机制
-> 预期改变的可观察性质
-> 后续需要的证据
-> 当前适用边界
```

设计数量由完整方法决定。多个模块若只是共同实现一项认识, 按其真实依赖写成一个关系; 常规工程步骤不单列为贡献性设计。R3 可以在不改变方法本体的前提下进一步固定实现身份和实验变体。

#### 从叙事中抽取

分点核心主张只从已确认的中心叙事中产生。每条主张必须承担不可替代的叙事作用; 删除它后, 叙事中的某个关键转折、联系或方法依据应当断裂。

主张不按共识、问题、方法、原因分配名额, 数量和类型由叙事决定。不要把方法模块、论文贡献、章节标题或实验清单当作分点主张。

#### 写清证明责任

每条主张只表达一个能够被单独支持或推翻的研究判断, 并写清:

- **分点主张**: 要成立的准确判断
- **叙事作用**: 它支撑中心叙事中的哪个转折或联系
- **需要的证据**: 哪类文献、观察、实验、推理或机制证据可以支持它
- **削弱或推翻条件**: 什么事实或结果会使它收窄、失效或被替代解释击穿

证据责任由主张内容决定。外部事实需要文献或数据; 方法有效性通常需要实验; 因果或机制解释通常需要对照、消融、干预或中间过程证据; 只靠逻辑或形式推导成立的主张写清前提, 不强行安排实验。

#### 检查最小性与覆盖

抽取完成后检查:

- 删除某条后叙事是否仍完整成立; 若是, 它通常不够承重
- 叙事是否存在没有对应主张的关键跳跃
- 多条主张是否只是同一句话的不同改写
- 一条主张是否混合了问题、原因、方法效果和泛化结论
- 需要经验支持的主张是否能在 R3 中形成明确证明责任

最小性不是主张越少越好, 而是每条都有不可替代的责任, 同时没有遗漏叙事成立所需的关键判断。

### 写入叙事定稿

将用户确认的中心叙事、论文事实和最终承重主张写入同一份`叙事定稿.md`, 严格参照`指导/参考/叙事定稿.md`。

`叙事`使用连续中文段落写入用户确认的中心叙事。`论文事实`保存任务、失败、原因、核心认识、完整方法、贡献性设计关系、相近工作和边界。`分点核心主张`写入最终承重主张。

用户只调整证据要求、表述精度或主张拆分时, 在第二阶段修正; 改变中心矛盾、认识变化或方法切入点时, 回到第一阶段; 改变 Idea 本体时, 先回到 R1。

### 停止规则

出现以下情况时停止继续包装:

- 找不到真实的中心矛盾
- 候选差异只有措辞, 没有研究判断差异
- 叙事依赖未核验事实、夸大现有工作或隐藏撞车关系
- 方法不能从问题或关键认识中自然推出
- 只能得到模块清单、贡献列表或宽泛目标, 无法抽出承重主张

停止不等于 Idea 一定不可做。说明缺少的是事实、问题理解、机制解释还是叙事本身, 由用户决定补充证据、返回 R1、修改 Idea 或暂时结束。

## 4. 产出

正式 R2 产出只有:

- `叙事定稿.md`

文档只包含叙事、论文事实和分点核心主张。使用的文献直接写在相关事实、叙事或主张附近。

## 5. 确认点

- 中心叙事由用户确认后才能拆分承重主张
- 任务定义、技术原因、核心认识、完整方法或贡献性设计关系存在多种会改变论文内容的理解时, 由用户确认
- 存在多条真实候选时, 由用户决定采用哪条或如何合并
- 新增、删除或改变会影响中心叙事的主张时, 由用户确认
- 改变 R1 的研究场景、核心问题、主要方法或原因链条时, 先回写`研究想法.md`
- 用户确认最终叙事、论文事实和分点核心主张后, R2 才完成

## 6. 完成条件

- 已生成或更新`叙事定稿.md`
- 文档只包含叙事、论文事实和分点核心主张三部分
- 论文事实已经覆盖当前研究适用的任务与目标、关键失败、技术原因、核心认识、完整方法、贡献性设计关系、相近工作和边界
- 已明确区分外部事实、本文机制判断和待实验内容
- 叙事围绕一个由用户确认的中心矛盾和认识变化展开
- 方法能够从问题或关键认识中自然推出
- 没有按 R1 的共识、问题、方法、原因机械填槽
- 每个分点主张都有不可替代的叙事作用、证据责任和削弱或推翻条件
- 每项贡献性设计能够说明它回应的问题、运行机制、可观察性质、证据责任和边界
- 分点主张通过最小性与覆盖检查
- 已有可靠网页来源的文献已在使用位置直接链接; 暂无网页来源的文献已标记`链接待补`, 未核验内容已明确标记

## 7. 不应做

- 不在中心叙事确认前机械拆分主张
- 不固定生成若干共识、问题、方法和原因
- 不把方法模块、论文贡献、章节标题或实验清单当作承重主张
- 不用“首次、重要、显著”代替真实的中心矛盾和认识变化
- 不为了故事吸引力而扩大问题、贬低现有工作或改变用户的 Idea
- 不提前写摘要、Introduction、贡献列表或论文正文
- 不把论文事实写成宣传文案, 也不把尚待证明的机制解释写成既成事实
- 不要求所有主张进入实验

## 8. 常见失败

- **四类填槽**: 按共识、问题、方法、原因分别凑主张, 形式完整但没有中心
- **先拆后拼**: 先生成分点主张, 再试图把它们拼成故事
- **模块冒充叙事**: 用系统包含哪些模块代替为什么必须这样解决问题
- **贡献冒充主张**: 把“提出方法、完成实验、取得提升”写成贡献列表
- **修辞冒充吸引力**: 通过夸大重要性和现有工作缺陷制造冲突
- **候选同质化**: 多个候选只有标题和措辞不同
- **机制断裂**: 问题成立, 方法却不是从问题本质自然推出
- **主张膨胀**: 把所有可能优点都列入主张, 没有最小承重集合
- **叙事越界**: 为了形成漂亮故事, 静默改变 R1
- **写作提前**: 研究叙事未确认便开始组织论文文案
- **事实欠账**: 叙事看起来顺畅, 但任务输入输出、失败条件、技术原因或设计关系要等到 R5 才临时补写
