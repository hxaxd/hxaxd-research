# 实验图表类型推荐

## 任务目标

根据数据结构和论文希望强调的结论, 选择最合适的实验统计图或实验表格, 并给出可复现的代码生成要求

先明确读者问题, 再选择图或表, 不能从“我想画柱状图”反推数据和结论, 也不能因为表格显得完整就塞入所有可用数字

## 输入检查

- 自变量, 因变量, 单位和指标方向
- 方法, 数据集, 种子, 时间或类别等维度
- 是否有重复运行, 方差, 置信区间或配对关系
- 希望突出精确数值, 趋势, 分布, 权衡还是误差类型
- 目标图的版面大小和正文/附录位置
- 若考虑表格, 确认分组层级, 比较方向, 单位, 数字精度和需要保留的原始结果

## 选择原则

- 方法对比: 分组柱状图或标签较长时使用横向条形图
- 双指标权衡: 帕累托前沿图；第三维成本可使用气泡图
- 训练趋势: 带置信区间折线图, 必要时局部放大
- 分类评估: 按类别平衡情况选择 ROC 或 PR 曲线
- 矩阵数据: 热力图
- 分布: 小提琴图或箱线图
- 多变量过多: 分面网格；双量纲谨慎使用双 Y 轴或柱折组合

补充判断:

- 精确比较多方法多数据集: 优先表格, 图只展示总体趋势
- 成对样本前后变化: 配对点图或斜率图
- 排名跨数据集变化: 排名图或临界差异图, 前提是统计方法合适
- 错误组成: 堆叠条形图, 但总量和比例不能混淆
- 参数敏感性: 折线或热力图, 必须显示实际采样点

## 代码生成要求

实验统计图和实验表格属于同一条代码生成路线:

1. 从实验记录引用的结构化结果读取数据, 不从截图或论文正文反向抄数
2. 在脚本中固定数据筛选, 聚合, 排序, 方法名称, 单位, 精度和缺失值处理
3. 记录脚本入口, 输入路径, 代码 commit, 运行命令和输出文件
4. 图与表共享同一结果源时, 应复用同一数据加载和统计逻辑, 避免两套手工口径
5. 输出前核对正文, 图, 表和 `结果分析.md` 中的数值, 方向和实验版本
6. 视觉或排版调整不得改变数据点, 排名, 分组关系和统计含义

## 统计与尺度

- 多次实验有方差时使用误差线或置信区间
- 跨数量级使用对数轴；极端范围但需保留原值时考虑断裂轴；强调相对变化时归一化
- 不为单次运行伪造误差线或显著性标记
- 误差线必须说明含义, 样本量和计算方式
- 坐标轴默认从能诚实呈现差异的范围开始；截断时显式标记并解释
- 百分比, 百分点和归一化分数使用不同单位

## 视觉编码

- 使用色盲友好配色, 并通过线型, 标记或纹理提供冗余编码
- 同一方法在全文保持相同颜色与顺序
- 强调本文方法不能牺牲其他方法可读性
- 标签优先直接标注, 图例不应迫使读者频繁来回查找
- 避免 3D 柱状图, 面积误导, 彩虹色热力图和装饰性背景

## 实验表格规范

当读者需要读取精确数值, 比较多个方法和数据集, 或核对消融组合时, 优先使用表格
LaTeX 表格按以下规则生成:

- 使用 `booktabs` 风格的 `\toprule`, `\midrule` 和 `\bottomrule`；不使用竖线,
  不用密集 `\hline` 分隔每一行
- 表注通常放在表格上方, 并先写 `\caption{...}`, 后写 `\label{...}`；若目标模板
  有明确相反要求, 以当前官方模板为准
- 列标题注明指标, 单位和方向, 例如 `Accuracy (\%) $\uparrow$`, `Latency (ms)
  $\downarrow$`
- 文本列左对齐, 数值列按小数点或固定宽度对齐；同一指标保持一致的小数位数
  需要时使用 `siunitx`, 但不得通过舍入改变并列或排名关系
- 多数据集, 多设置或多指标使用 `\multicolumn` 与 `\cmidrule` 建立分组, 不用
  竖线承担层级
- 一张表只承担一个主要比较问题, 无关设置拆表或放入附录, 不能为了显得充分而混入
  不同协议的数据
- 最佳值, 次优值和目标行只做克制且一致的强调；高亮必须由代码根据明确规则生成,
  不手工挑选有利单元格, 颜色不是必要条件, 并需兼顾黑白打印
- 表注聚焦设置, 协议, 缩写和记号, 结果解释放在正文, 不能把未经证明的因果结论写进
  表注
- 跨栏, 单栏和浮动位置服从目标 venue 的模板及版面, 不把某一固定栏位当作普遍规则

表格生成脚本应能从原始结果重建数值单元格和高亮状态, 若列宽, 换行或跨栏命令需要人工调整, 调整后仍需运行数值一致性检查

## 常见陷阱

- 数据只有三个精确值, 却使用复杂图而不是表格
- 双 Y 轴制造不存在的相关趋势
- 只展示均值, 隐藏方差和失败运行
- 折线连接本质离散且无序的类别
- 热力图颜色范围在不同子图中不一致
- 归一化后没有同时提供原始尺度
- 表格依靠手工复制, 正文更新后仍保留旧数字
- 不同列使用不同精度, 导致微小差异看起来像排名变化
- 用粗体, 底色或排序突出本文方法, 却没有一致的自动判定规则

## 简短示例

若要展示“恢复率与 token 成本权衡”, 优先使用散点/帕累托图: X 轴为平均 token,
Y 轴为恢复率, 每个点为一种方法, 误差线来自重复运行, 若只有两种方法且读者需要精确数值, 表格可能比气泡图更合适

## 输出

每个方案说明选择图或表的核心理由, 对应 Claim, 数据来源和代码入口, 统计图还需说明坐标轴, 单位, 尺度, 统计要素, 配色和线型；表格还需说明分组, 精度, 方向, 强调规则和 LaTeX 结构, 同时说明未选择其他常见形式的原因, 以及当前图表不能支持的结论
