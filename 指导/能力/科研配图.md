# 科研配图

## 1. 做什么

科研配图是一项可在 R2、R4、R5 和 R6 中独立调用的能力, 负责把论文主张、方法关系或真实实验结果转化为实际可使用的科研图件。

本能力建立从论文论证到视觉结构再到实际图件的闭环:

- 从论文论证中找出一张图真正应该表达的结论
- 为不同证据选择正确的视觉结构, 避免把面板机械拼在一起
- 把概念生成、精确标签、真实数据和投稿格式组合成可编辑、可追溯的实际产物

科研图首先是一段视觉论证。工具、风格和模板只能在核心结论、证据层次和面板责任明确后决定。

## 2. 输入

- 这张图准备支持或解释的研究主张
- 图在论文中的作用, 例如问题示意、方法总览、主结果、消融、成本、迁移、失败分析或图形摘要
- 真实数据、日志、样本、图片或方法关系, 视图件类型而定
- 已有稿件、图注、图件源文件和论文术语, 若存在
- 目标载体、最终尺寸、语言和投稿场所要求, 若已确定

若用户只说“给论文画张图”, 先从`叙事定稿.md`、`实验证明.md`或实际稿件推导候选核心结论。存在多种实质不同的视觉主线时, 先让用户选择; 不要把不确定关系直接画成定稿。

## 3. 执行

### 1. 先判断是否应该画

在作图前回答:

- 这张图必须让读者理解或相信什么
- 这个信息用正文、表格还是图更合适
- 它服务哪条分点主张或哪一个论文转折
- 读者看完图后应该能够复述哪一句结论

若图不能让某条主张更容易理解、比较或验证, 通常不应创建。精确数字较多时优先考虑表格; 关系、趋势、结构或证据层次难以用文字表达时再使用图。

在对话中形成简短图件契约:

- 核心结论
- 图件作用
- 图件类别
- 主证据
- 支持证据
- 面板责任
- 真实来源
- 最可能受到的审稿攻击

### 2. 区分三类图件

#### 证据与结果图

包括定量比较、趋势、消融、成本曲线、迁移矩阵、分布、热图、失败统计和含精确指标的多面板结果图。

必须遵守:

- 使用真实结果和确定性绘图代码生成
- 图像生成模型不得生成数值、坐标轴、误差线、表格或统计结论
- 绘图脚本能够从真实数据重新生成最终图件
- 样本、种子、均值、方差、置信区间和统计口径与 R3/R4 保持一致
- 只展示最好结果、手工移动数据点或用视觉范围制造虚假优势均被禁止

当精确数值本身重要时使用表格; 当趋势、差异结构或关系更重要时使用图。

#### 概念与方法图

包括问题示意、方法总览、系统架构、流程、机制、Teaser、图形摘要和论文视觉封面。

可以使用图像生成工具探索视觉构图, 但:

- 方法模块、输入输出、箭头、反馈关系和失败路径必须来自真实研究定义
- 生成图只提供视觉草稿, 不自动成为科学事实
- 精确术语、符号和关键标签优先用 SVG、绘图代码、排版工具或其他确定性方式叠加
- 生成模型不得补充用户没有主张的机制、组件、实验现象或因果关系
- 需要后续修改的图应保留可编辑或可再生成的真实来源

若生成式图像不适合表达精确结构, 直接使用 SVG、TikZ、Mermaid、绘图代码或其他可编辑方案, 不为了视觉复杂度强行使用生成模型。

#### 真实样本与定性图

包括模型真实输出、界面截图、图像样本、案例对比和其他直接来自实验的视觉证据。

必须保留样本与原始结果的对应关系。可以统一裁剪、排版和添加标注, 但不能生成、替换或修改会改变科研含义的内容。筛选代表性样本时, 同时说明它是典型行为、边界情况还是失败案例, 不把人工挑选的个例伪装成总体统计。

### 3. 把整张图组织成论证

先确定主证据, 再确定辅助面板:

- 主面板承担核心结论, 获得最清楚的位置、面积和视觉层次
- 辅助面板分别承担验证、机制、消融、控制、稳健性或失败边界
- 每个面板只回答一个必要问题
- 遮住某个面板仍不影响结论时, 删除或合并它
- 不因版面整齐而让所有面板等大
- 不用多个面板重复展示同一个结论

常见的信息顺序是“系统或设定 → 主结果 → 机制或原因 → 控制与边界”, 但应服从当前论文 Story, 不作为固定模板。

### 4. 建立跨图件视觉语言

同一论文中的颜色、符号、名称和方向应具有稳定语义:

- 同一个方法、Baseline、数据条件或流程阶段在不同图中保持相同颜色
- 主方法可以突出, 但不能通过不公平的颜色、粗细或面积暗示不存在的优势
- 颜色不是唯一编码; 需要时同时使用线型、标记、明暗或纹理
- 避免彩虹色图、装饰性 3D、无解释双轴和无意义渐变
- 概念图与结果图尽量共享方法颜色和术语
- 图中文字、数学符号和稿件正文保持一致

视觉语言由当前论文和稿件中的既有约定共同决定。

### 5. 选择实际实现方式

优先沿用项目已经使用的语言、绘图库、排版系统和目录结构。没有既有方案时, 根据实际输入选择:

- 数值结果: Python、R 或现有实验代码中的确定性绘图库
- 精确流程和架构: SVG、TikZ、Mermaid、Graphviz 或绘图代码
- 视觉化概念草图: 可用图像生成工具快速迭代, 再以确定性标签或可编辑源完成
- 多种材料组合: 使用混合方式, 但保持真实数据层与生成视觉层清楚分离

不要为了某个指导固定 Python、R 或特定图像模型。后端只是一项当前实现选择。

概念图可以先低成本生成草图并小步修改构图, 构图和术语确认后再生成最终尺寸。结果图则应优先保证数据与脚本正确, 再调整版式。

### 6. 生成实际产物

产物写入真实使用位置:

- 结果图的源数据、绘图脚本和中间统计优先留在独立实验仓库
- 稿件需要的 SVG、PDF、PNG、TIFF 或其他最终图件放入实际稿件结构
- 概念图的可编辑源、生成提示或确定性叠加代码只在确实用于再生成时保留
- 用户选定方向后保留最终使用或再生成所需的版本

默认优先导出带可编辑文字的 SVG 或 PDF; 只有实际工具链或投稿规则需要时再使用高分辨率栅格格式。

### 7. 同时完成图注和正文接入

图件不是孤立附件。完成图后直接更新真实稿件中的:

- 图文件路径
- 正文第一次引用位置
- 图注
- 与图一致的结果或方法解释

图注应完成四件事:

1. 这张图展示什么
2. 颜色、线型、误差或箭头怎样阅读
3. 读者应得到什么结论
4. 哪项限制会影响解释, 若有

不要只把面板元素逐项复述一遍。

### 8. 在最终尺寸检查

交付前渲染并查看实际图像, 不能只检查源代码。至少确认:

- 缩放到论文最终宽度后文字、图例、标记和误差线仍可读
- 面板顺序、视觉层次和阅读方向清楚
- 所有数字与真实来源一致
- 每个定量面板都能追溯到数据和绘图脚本
- 生成式图件没有幻觉标签、虚构机制、错误箭头、Logo 或水印
- 颜色在灰度和常见色觉条件下仍能区分
- 术语、缩写和颜色与稿件其他图一致
- 图注能够独立解释图件
- 最终格式、尺寸、分辨率和字体满足当前投稿场所的官方要求

发现视觉问题直接修改真实图件。

## 4. 产出

正式产出是当前研究真正使用的图件及其必要来源:

- 最终图件
- 必要的源数据、绘图脚本、可编辑图源或再生成来源
- 实际稿件中的图注和引用

## 5. 确认点

- 核心结论存在多种实质不同理解时
- 概念图中的关系、模块或机制无法从现有研究文档确定时
- 需要从多种视觉主线中选择 Figure 1 或 Teaser 时
- 需要对真实样本进行可能改变科研含义的编辑时
- 需要改变论文现有统一视觉语言时
- 最终图件准备进入对外稿件时

## 6. 完成条件

- 图件对应明确的论文主张或解释责任
- 每个面板承担独立且必要的作用
- 数据图完全来自真实数据和确定性绘图
- 概念图中的科学关系已经核对, 没有生成式幻觉
- 图件拥有实际需要的可编辑或可再生成来源
- 图注和正文已经接入真实稿件
- 最终尺寸下已经完成视觉检查
- 用户确认核心视觉表达

## 7. 不应做

- 不从喜欢的模板出发反推论文结论
- 不用图像生成模型绘制定量结果、精确数字或统计图
- 不让概念图生成不存在的模块、机制或效果
- 不把装饰性插图冒充科研证据
- 不用面板数量制造内容丰富的错觉
- 不重复展示同一结论
- 不只交付 PNG 而丢失必要的真实来源
- 不在没有查看最终渲染的情况下宣称配图完成

## 8. 常见失败

- **先画后想**: 图很精致, 但说不清支持哪条主张
- **面板拼盘**: 每个实验放一格, 没有主证据和阅读顺序
- **数据生成化**: 让图像模型绘制看似合理但并不存在的数值图
- **概念幻觉**: 图比方法本身多出模块、箭头或因果机制
- **文字灾难**: 生成式图件中的术语和标签无法准确修改
- **视觉漂移**: 同一方法在不同图中不断换颜色和符号
- **只看大图**: 屏幕上清楚, 放进双栏论文后完全不可读
- **图文分离**: 图件已经完成, 稿件却没有正确引用、解释和图注
