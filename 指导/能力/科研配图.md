# 科研配图

## 科研图是一段视觉论证

科研配图把论文事实、方法关系或真实实验结果转化为读者能够直接理解和比较的视觉证据。先明确图要让读者理解或相信什么, 再决定图、表、面板、风格和工具。

这项能力可以在 R2、R4、R5 或 R6 中独立调用。已经进入 R2 的研究从`叙事定稿.md`读取任务、失败、核心认识、设计关系与稳定术语; 结果图同时读取`实验证明.md`和 R4 的真实产物。上游尚未确定的科学关系不能由图件补全。

## 先形成图的核心判断

作图前用一句话说明读者看完后应能复述什么, 再判断它是否真的需要视觉表达。精确数字和多条件查找通常适合表格; 趋势、分布、流程、空间关系、视觉样本或多层证据更适合图。

根据复杂度在当前对话或工作上下文中澄清:

- 图服务哪条论文判断或叙事转折
- 主证据是什么, 还有哪些辅助证据
- 每个面板承担什么不同问题
- 数据、样本或方法关系来自哪里
- 最可能受到的审稿攻击是什么

这些关系在当前工作上下文中指导设计。存在多条实质不同的 Figure 1 或 Teaser 主线时由用户选择; 布局细节可以直接迭代。

概念图的判断通常能还原为“失败或问题 → 核心认识 → 设计变化 → 预期性质”; 结果图则还原为“研究问题 → 比较条件 → 观察 → 允许解释与边界”。只取当前图真正需要的部分。

## 不同图件需要不同真实性边界

### 证据与结果图

定量比较、趋势、消融、成本、迁移、分布、热图和失败统计使用真实结构化结果与确定性绘图代码生成。图像生成模型不生成坐标、数值、误差线、表格或统计结论。

绘图脚本应能从源数据重新生成最终图件, 样本、种子、聚合、不确定性和统计口径与 R3/R4 一致。坐标范围、数据筛选和视觉强调不能制造原结果不存在的优势。表格的信息组织与 TeX 接入见`指导/写作/章节/图表.md`。

### 概念与方法图

问题示意、系统架构、方法流程、Teaser 和图形摘要可以使用图像生成工具探索构图, 但模块、输入输出、箭头、反馈和失败路径必须来自真实研究定义。生成结果只提供视觉草稿, 不自动成为科学事实。

精确标签、符号和结构优先用 SVG、TikZ、Mermaid、Graphviz 或绘图代码完成。生成式图像表达关系不可靠时, 直接选择确定性方案, 以科学准确性和可编辑性决定视觉复杂度。

### 真实样本与定性图

模型输出、界面截图、图像样本和案例对比保留与原始结果的对应关系。可以统一裁剪、排版和标注, 但不能生成、替换或修改会改变科研含义的内容。样本应说明是典型行为、边界还是失败, 人工挑选个例不能替代总体统计。

## 用面板建立阅读顺序

整张图先确定主面板, 再加入解释主结论所需的机制、控制、消融、稳健性或失败面板。主面板获得最清楚的位置和面积, 面板大小服从证据重量。

遮住一个面板后核心判断完全不变, 说明它可能应删除或合并。多个面板重复同一趋势也不会让证据更强。

常见阅读顺序是“设定或系统 → 主结果 → 机制解释 → 控制与边界”。方法图、结果图和图形摘要根据各自的研究关系形成节奏, 箭头和布局直接表达这些关系。

## 建立论文内部的视觉语义

同一个方法、基线、数据条件和流程阶段在不同图中保持稳定名称、颜色与符号。关键类别同时使用颜色与线型、标记、明暗或纹理编码。视觉元素围绕数据关系和阅读任务选择。

主方法可以突出, 突出程度与真实比较关系一致。概念图和结果图尽量共享术语与方法颜色, 让读者沿用同一套视觉语义。

## 选择实现并保留真实来源

优先沿用项目已有绘图库、排版系统与目录结构。没有既有方案时按材料选择:

- 数值结果: Python、R 或实验仓库中的确定性绘图库
- 精确流程与架构: SVG、TikZ、Mermaid、Graphviz 或绘图代码
- 概念构图探索: 图像生成工具, 再以可编辑结构和确定性标签收束
- 多种材料组合: 分离真实数据层与生成视觉层

结果图的源数据、脚本和中间统计通常留在实验仓库; 稿件使用的最终 SVG、PDF、PNG 或 TIFF 放入真实稿件结构。概念图只保留实际再生成需要的可编辑源、提示或叠加代码。格式和版本数量服从当前工具链与投稿要求。

## 图件必须进入真实稿件

完成图后同步更新图文件路径、正文第一次引用、图注与结果/方法解释。图注说明图展示什么、视觉编码怎样读取、主要观察是什么、哪项边界影响解释; 正文解释为什么此处需要它以及它怎样改变判断, 不逐项复述面板。

最后在论文实际尺寸渲染查看, 核对阅读方向、视觉层次、术语、数字、误差、颜色、图注和最终格式。定量面板应能回到数据与脚本, 生成式草图不能残留幻觉标签、错误箭头、Logo 或水印。发现问题直接修订真实图源和稿件。

当图的核心判断清楚、科学关系和数据可追溯、每个面板具有独立责任、真实稿件已经接入, 并且最终尺寸可读时, 图件才算交付。
