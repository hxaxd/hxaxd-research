% 已有方法面临的技术问题, 围绕本文解决的技术问题展开, 技术问题包括局限和技术原因

### 版本三: 已有流程上增加新模块

% 我们的创新是引入循环卷积, 用于高效学习轮廓上的特征, 如图 1 所示

### 版本一: 技术方法 + 技术流程 + 实验证据 ([Deep Snake](https://openaccess.thecvf.com/content_CVPR_2020/html/Peng_Deep_Snake_for_Real-Time_Instance_Segmentation_CVPR_2020_paper.html))

```tex
我们的方法在 Cityscapes [8], KINS [35], SBD [16] 和 COCO [24] 数据集上取得了具有竞争力的表现, 同时能够以每秒 32.3 帧的速度处理 $512 \times 512$ 图像, 满足实时实例分割的效率要求
以下两个原因使基于学习的 snake 同时具备较高的速度和准确性: 第一, 我们的方法能够处理物体定位阶段产生的误差, 因而可以使用轻量级检测器; 第二, 轮廓表示的参数量少于像素表示, 并且不需要掩码上采样等昂贵的后处理

综上, 本文具有以下贡献:
\begin{itemize}
    \item 提出一种用于实时实例分割的学习式 snake 算法, 并引入循环卷积在轮廓上学习特征
    \item 提出由初始轮廓生成和轮廓变形组成的两阶段实例分割流程, 两个阶段都能够处理初始物体定位中的误差
    \item 在 Cityscapes, KINS, SBD 和 COCO 数据集上取得当前最佳表现, 对于 $512 \times 512$ 图像, 算法以每秒 32.3 帧运行, 能够用于实时应用
\end{itemize}
```

### 版本二: 完整流程 + 新表示 + 优化方法 + 多任务验证 ([DUSt3R](https://openaccess.thecvf.com/content/CVPR2024/html/Wang_DUSt3R_Geometric_3D_Vision_Made_Easy_CVPR_2024_paper.html))

```tex
综上, 本文具有四项贡献
第一, 我们提出首个从未标定且位姿未知的图像出发, 统一单目和双目三维重建的完整端到端流程
第二, 我们将点图表示引入多视图立体重建, 使网络能够在规范坐标系中预测三维形状, 同时保留像素与场景之间的隐式关系, 从而摆脱常规透视相机表述中的多项约束
第三, 我们提出一种在多视图三维重建中对点图进行全局对齐的优化方法, 并能够直接恢复传统运动恢复结构和多视图立体重建流程中的各种中间结果
第四, 我们在多项三维视觉任务上验证了所提方法, 统一模型在单目深度估计, 多视图深度估计和多视图相机位姿估计上取得当前最佳表现
```
